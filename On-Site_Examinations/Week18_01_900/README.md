### 題目：[UVa 900 - Brick Wall Patterns](https://onlinejudge.org/external/9/900.pdf)

#### 解題思路
鋪設一堵寬度為 $n$ 的牆，磚塊有兩種規格：寬度 1 的直磚，或寬度 2 的橫磚
- 目標：計算總共有幾種鋪設牆面的方式
- 核心特性：當我們要鋪設長度為 $n$ 的牆時，最後一塊磚只可能有兩種情況：
    1. 最後一塊是寬度 1 的直磚，則前面長度為 $n-1$
    2. 最後一塊是寬度 2 的橫磚，則前面長度為 $n-2$
- 核心策略：因此，總方法數 $f(n) = f(n-1) + f(n-2)$。這完全符合費氏數列的定義

#### 關鍵邏輯：
1. 狀態定義：
    - dp[i] 表示寬度為 $i$ 的牆面共有幾種排列方式
2. 狀態轉移：
    - dp[i] = dp[i-1] + dp[i-2]
    - 基礎情況：dp[1] = 1（只有一種：一塊寬1磚）、dp[2] = 2（兩種：兩塊寬1磚，或一塊寬2磚）
3. 預先計算：
    - 由於題目規定的最大寬度 $n \le 50$，我們可以預先算好所有結果存在陣列中，之後每次查詢只需 $O(1)$ 的時間

#### 模擬運算(以 $n=4$ 為例)：
1. dp[1] = 1
2. dp[2] = 2
3. dp[3] = dp[2] + dp[1] = 2 + 1 = 3
4. dp[4] = dp[3] + dp[2] = 3 + 2 = 5
- 結果為 5 種鋪設方式
- Time Complexity：$O(N)$，預處理過程僅需一次線性迴圈
- Space Complexity：$O(N)$，用於存儲已計算好的結果

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 數值型態溢位：$n=50$ 時，費氏數列的值已經相當大，超過了 int 的範圍 (約 20 億)，必須使用 long long 來儲存結果，否則會發生溢位錯誤 (Overflow)
    - 預處理優化：當題目涉及多次查詢時，將結果「預先算好」再進行查詢是提升效能的最佳作法
    - 邊界條件判定：題目以 0 作為結束條件，這在 UVa 題目中非常常見，務必在迴圈條件中正確處理

#### 總結 (Takeaway)
- 鋪磚問題計數 SOP：
    1. 透過推導找出遞迴關係 (本題為 $f(n) = f(n-1) + f(n-2)$)
    2. 確定範圍並宣告適當的資料型態 (long long 預防溢位)
    3. 利用預計算或動態規劃存儲答案
    4. 讀取輸入並查表輸出
