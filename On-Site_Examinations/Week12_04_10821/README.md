### 題目：[UVa 10821 - Constucting BST](https://onlinejudge.org/index.php?option=com_onlinejudge&Itemid=8&page=show_problem&problem=1762)

#### 解題思路
給定兩個整數 $N$（節點數量）與 $H$（最大高度）
- 目標：建構出一棵高度不超過 $H$ 的二元搜尋樹，並輸出其前序走訪序列。如果存在多種方案，輸出 字典序最小 的那一個
- 字典序最小原則：為了讓序列開頭的數字儘可能小，我們在選擇根節點時應當選取符合高度限制下 最小的可能數字
- 核心策略：根節點選得越小，左子樹就越小，右子樹就越大。但右子樹的大小受限於高度 $H-1$。因此，根節點的選擇必須保證「剩下的節點數量能夠被塞進高度為 $H-1$ 的右子樹中」

#### 關鍵邏輯：
1. 最大容量判定：
    - 一棵高度為 $H$ 的二元樹最多能容納 $2^H - 1$ 個節點
    - 如果 $N > 2^H - 1$，則無法建構，輸出 Impossible
2. 貪心選擇根節點:
    - 對於當前區間 [left, right]，節點數為 numNodes
    - 右子樹的最大容納能力為 $MaxRight = 2^{H-1} - 1$
    - 為了讓根節點儘量小，我們要推給右子樹儘量多的負擔。根節點位置 root 應為：left + max(0, numNodes - 1 - MaxRight)
    - 這樣能確保扣除根節點與右子樹後，左邊留下的都是最小的數字
3. 遞迴建構：
    - 輸出 root（前序：根 $\rightarrow$ 左 $\rightarrow$ 右）
    - 遞迴處理左子樹 [left, root-1]，高度減 1
    - 遞迴處理右子樹 [root+1, right]，高度減 1

#### 模擬運算($N=4, H=3$ 為例)：
- 第一層：numNodes = 4, H = 3
    - 右子樹最大容量 $2^{3-1}-1 = 3$
    - 根節點：$1 + \max(0, 4 - 1 - 3) = 1$
    - 輸出 1，左子樹 [], 右子樹 [2, 3, 4]
- 第二層 (右側)：numNodes = 3, H = 2
    - 右子樹最大容量 $2^{2-1}-1 = 1$
    - 根節點：$2 + \max(0, 3 - 1 - 1) = 3$
    - 輸出 3，左子樹 [2], 右子樹 [4]
- 結果：輸出 1 3 2 4
- Time Complexity：$O(N)$。每個節點被選取並輸出一次
- Space Complexity：$O(H)$。遞迴深度受限於樹的高度

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 貪心策略的應用：理解了「字典序最小」在 BST 中意味著要儘量讓左子樹變小
    - 位元溢出預防：在使用 1 << H 計算 $2^H$ 時，若 $H \ge 31$ 會導致整數溢出。程式碼中透過 if (h < 20) 或使用 long long 來安全處理 10,000 內的節點規模
    - 格式控制：輸出空格時需注意結尾不留空白，本例中使用 caseNum == -1 或簡單的標記變數來處理
#### 總結 (Takeaway)
- 高度限制下的 BST 建構 SOP：
    1. 檢查 $N \le 2^H - 1$ 是否成立
    2. 計算右子樹剩餘高度能承載的最大節點數
    3. 從左邊界開始，跳過「必須留給右子樹」以外的所有節點，選出最小根節點
    4. 按「根 $\rightarrow$ 左 $\rightarrow$ 右」順序遞迴
