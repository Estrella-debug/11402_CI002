### 題目：[UVa 10107 - What is the Median?](https://onlinejudge.org/index.php?option=com_onlinejudge&Itemid=8&page=show_problem&problem=1048)

#### 解題思路
讀取一連串的整數，每讀入一個新數字，就必須輸出目前所有已讀入數字的 中位數
- 中位數定義：
    1. 將數列從小到大排序
    2. 如果總數是 奇數：中位數是正中間的那一個
    3. 如果總數是 偶數：中位數是中間兩個數字的平均值

#### 關鍵邏輯：
1. 動態儲存 (Dynamic Storage)：使用 vector<long long> v 來儲存讀入的數字
2. 即時排序 (Real-time Sorting)：每加入一個新成員 v.push_back(n)，就呼叫 sort(v.begin(), v.end()) 確保數列處於有序狀態
3. 位置判定：
    - 奇數長度：索引值為 size / 2
    - 偶數長度：取索引值 size / 2 與 size / 2 - 1 兩數相加除以 2
4. 大數處理：由於數值可能較大，使用 long long 避免計算平均值時發生溢位
#### 模擬運算(1, 3, 4, 60 為例)：
- 加入 1：排序後 [1]。長度 1 (奇)。輸出 1
- 加入 3：排序後 [1, 3]。長度 2 (偶)。輸出 (1+3)/2 = 2
- 加入 4：排序後 [1, 3, 4]。長度 3 (奇)。輸出 3
- 加入 60：排序後 [1, 3, 4, 60]。長度 4 (偶)。輸出 (3+4)/2 = 3
- Time Complexity：$O(N^2 \log N)$。因為每次插入都重新排序一次（$N$ 次 $N \log N$）
- Space Complexity：$O(N)$。需要儲存所有讀入的數字

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 排序的代價：雖然直接 sort 最簡單，但當資料量極大時會變慢。程式碼中的註解提到可以使用 lower_bound 找到位置後用 insert 插入，能將複雜度優化至 $O(N^2)$，或者用兩個 priority_queue 優化至 $O(N \log N)$
    - C++ 的整數除法會自動捨去小數點，這剛好符合題目對偶數長度中位數的要求
    - 資料型態範圍：使用 long long 是一個好習慣，尤其是在處理可能涉及大量加法的題目時

#### 總結 (Takeaway)
- 動態中位數問題 SOP：
    1. 準備一個可動態增長的容器
    2. 確保容器內容維持有序
    3. 根據當前總數的奇偶性，計算中間索引並輸出結果
- 中位數反映了資料的集中趨勢，掌握其動態計算方式是資料分析的基礎
