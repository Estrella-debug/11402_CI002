### 題目：[UVa 679 - Dropping Balls](https://onlinejudge.org/index.php?option=com_onlinejudge&Itemid=8&page=show_problem&problem=620)

#### 解題思路
有一棵深度為 $D$ 的滿二元樹，每個節點都有一個開關（初始為 False，指向左方）。當一顆球落在節點上時：
1. 若開關為 False，球往 左子節點 掉落，開關變為 True
2. 若開關為 True，球往 右子節點 掉落，開關變為 False
- 目標：計算第 $I$ 顆球最後落下的葉節點編號
- 核心挑戰：若直接模擬 $I$ 顆球的掉落過程，當 $I$ 很大時會導致超時（TLE）
- 關鍵策略：觀察球的分配規律。第 1 顆球往左、第 2 顆往右、第 3 顆往左... 這種規律完美對應了 二進位的最低有效位 (LSB)

#### 關鍵邏輯：
1. 球數歸零化 (n--)：
    - 將第 $n$ 顆球轉換為從 $0$ 開始計算，方便進行位元運算
2. 位元決定方向:
    - 對於每一層的選擇，球的編號 $n$ 的第 $i$ 個位元決定了它是該層的第幾顆「經過該分支」的球
    - 若 $n$ 的第 $i$ 位是 0（偶數次到達該節點），則往左走
    - 若 $n$ 的第 $i$ 位是 1（奇數次到達該節點），則往右走
3. 節點編號計算 (ans <<= 1)：
    - 在層次遍歷編號中，左子節點是 parent * 2，右子節點是 parent * 2 + 1
    - ans <<= 1 相當於 * 2
    - if (n & (1LL << i)) ans |= 1 相當於判斷若該位元為 1，則加上 1（即往右走）
4. 高效運算：
    - 捨棄循環模擬，直接根據球的編號 $n$ 的二進位形式，在 $O(D)$ 時間內直接「預測」出最終落點

#### 模擬運算($D=3, I=2$ 為例)：
- 初始化：n = 2-1 = 1, d = 3-1 = 2, ans = 1
- 第 1 層 (i=0)：
    - 檢查 n 的第 0 位（$1 \& 1 = 1$）
    - ans = 1 << 1 = 2
    - 位元為 1，ans |= 1 $\rightarrow$ ans = 3
- 第 2 層 (i=1)：
    - 檢查 n 的第 1 位（$1 \& 2 = 0$）
    - ans = 3 << 1 = 6
    - 位元為 0，不變
- 結果：輸出 6
- Time Complexity：$O(D)$。每一組測試資料只需遍歷樹的深度次
- Space Complexity：$O(1)$

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 二進位規律：發現了開關狀態的切換本質上就是二進位數值的累加
    - 位元運算的威力：ans <<= 1 與 ans |= 1 的組合比加法與乘法更具備硬體層面的效率
    - 大數處理：使用 1LL 確保位移運算在 64 位元整數下進行，避免溢出
    - IO 優化：ios_base::sync_with_stdio(0) 在處理大量測資（$T$ 很大）時非常有感
#### 總結 (Takeaway)
- 規律性二元樹問題 SOP：
    1. 觀察路徑選擇是否具有週期性
    2. 將序號從 0 開始以適配位元操作
    3. 利用二進位位元（Bit）來決定每一層的左右分支
    4. 利用層次編號公式（*2 與 *2+1）更新目前位置
