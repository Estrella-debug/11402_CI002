### 題目：[UVa 11321 - Sort! Sort!! and Sort!!!](https://onlinejudge.org/index.php?option=onlinejudge&page=show_problem&problem=2296)

#### 解題思路
給定 $N$ 個整數以及一個除數 $M$，請依照下列規則對這 $N$ 個數進行排序
- 排序優先級（由高到低）：
    1. 餘數大小：除以 $M$ 後餘數較小的排在前面
    2. 奇偶差異：若餘數相同，奇數排在偶數前面
    3. 同為奇數：若餘數相同且都是奇數，較大的奇數排在前面
    4. 同為偶數：若餘數相同且都是偶數，較小的偶數排在前面

#### 關鍵邏輯：
1. Lambda 運算式與閉包：
    - 程式中使用 [M](const int& a, const int& b) { ... }。這是一個 Lambda 函式，透過 [M] 將外部變數 $M$ 傳入比較邏輯中
2. 階層式比較判斷：
    - 第一層：比較 a % M 與 b % M
    - 第二層：利用布林值 a_odd 與 b_odd 判定。若一奇一偶，回傳 a_odd（即奇數優先）
    - 第三層：同奇時用 a > b，同偶時用 a < b
3. 負數取餘數的陷阱：
    - 在 C++ 中，負數取餘數（如 -5 % 3）會得到 -2。這套程式碼利用了這個特性，因為 -2 < -1 < 0 < 1，依然符合題目要求的升序規則
4. 輸出入優化：
    - ios::sync_with_stdio(false), cin.tie(nullptr); 能大幅加快讀取效率，這在處理多達 10,000 筆資料時非常重要

#### 模擬運算( $M=3$ 為例)：
- 比較 15 與 12：
    - $15 \pmod 3 = 0, 12 \pmod 3 = 0$。餘數相同
    - 15 與 12 皆為偶數
    - 規則：小偶數在前 $\rightarrow 12$ 在前
- 比較 13 與 15：
    - $13 \pmod 3 = 1, 15 \pmod 3 = 0$
    - 規則：餘數小在前 $\rightarrow 15$ 在前
- Time Complexity：$O(N \log N)$。核心操作是標準排序，比較函式內的運算為 $O(1)$
- Space Complexity：$O(N)$。需空間儲存這 $N$ 個整數

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 多條件排序的嚴謹性：撰寫 sort 的比較函式時，必須確保邏輯覆蓋所有可能的情況，且滿足「嚴格弱序 (Strict Weak Ordering)」
    - 奇數判斷技巧：在處理負數時，使用 x % 2 != 0 來判斷奇數比 x % 2 == 1 更安全（因為負奇數餘數是 -1）
    - 輸入結束標記：題目要求最後輸出 0 0 作為結束符號，這是一個容易被遺忘的小細節
#### 總結 (Takeaway)
- 複雜規則排序問題 SOP：
    1. 確立排序的優先級階層
    2. 撰寫自定義比較函式，逐層使用 if 或 return 進行分流
    3. 特別注意「特殊數值」（如負數）在運算中的行為
    4. 對於大數據量，務必使用 Fast I/O 優化
