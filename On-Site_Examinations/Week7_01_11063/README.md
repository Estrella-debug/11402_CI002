### 題目：[UVa 11063 - B2-Sequence](https://onlinejudge.org/index.php?option=com_onlinejudge&Itemid=8&category=22&page=show_problem&problem=2004)

#### 解題思路
判斷一個數列 $\{b_1, b_2, b_3, \dots, b_n\}$ 是否為 B2-Sequence
- 定義規則：
    1. 數列必須是正整數：$1 \le b_1 < b_2 < b_3 < \dots$
    2. 數列必須嚴格遞增
    3. 任兩數之和必須唯一：所有的兩數之和 $b_i + b_j$（其中 $i \le j$）都不能重複出現
- 範例：1 2 4 7 是一組 B2-Sequence，因為它的兩數之和集合為 {2, 3, 5, 8, 4, 6, 9, 8, 11, 14}，其中 8 重複了，所以它不是 B2-Sequence（注意：此處僅為邏輯示範，實際需嚴格比對）

#### 關鍵邏輯：
1. 初步檢查 (Basic Constraints)：
    - 讀取數字時，先檢查是否所有數字都大於等於 1
    - 檢查數列是否符合 $num[i] > num[i-1]$ 的嚴格遞增條件
2. 兩數之和去重 (Sum Uniqueness)：
    - 使用 std::set<int> sums 來記錄所有產生的和
    - 使用巢狀迴圈遍歷所有組合 $(i, j)$，計算 sum = num[i] + num[j]
    - 檢查衝突：如果 sums.count(sum) > 0，代表這個「和」以前出現過，不符合 B2-Sequence 定義，立即標記並跳出迴圈
3. 輸出格式：題目要求每組結果後都要多印一個換行符號（\n\n），這在自動評測系統中非常重要

#### 模擬運算(1 3 5 為例)：
- 檢查遞增：$1 < 3 < 5$，通過
- 兩數之和計算：
    - $1+1=2$
    - $1+3=4$
    - $1+5=6$
    - $3+3=6$ $\rightarrow$ 衝突 6 已存在於 set 中
- 結果：It is not a B2-Sequence.
- Time Complexity：$O(n^2 \log(n^2))$。兩層迴圈產生 $n^2$ 個和，Set 的操作成本為 $\log$
- Space Complexity：$O(n^2)$。需要空間儲存所有不重複的和

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 嚴格定義的重要性：題目中隱含了 $b_1 \ge 1$ 的條件，如果忽略了「正整數」這個細節，會導致某些測資出錯
    - Set 的高效去重：在處理「是否出現過」的問題時，std::set 是最直覺且穩定的工具
    - 格式細節：UVa 題目對 Case #x: 的格式以及空行要求非常嚴格，程式碼中的 \n \n 處理正是為了對應這些細節

#### 總結 (Takeaway)
- 數列定義判定 SOP：
    1. 逐一檢查基礎限制（正數、遞增）
    2. 利用 Set 或雜湊表記錄運算結果（如兩數之和）
    3. 一旦發現重複，立即終止運算並輸出結果
- 這種「兩兩組合」的檢查方式是離散數學題目中常見的暴力優化解法
