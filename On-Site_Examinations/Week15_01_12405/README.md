### 題目：[UVa 12405 – Scarecrow](https://onlinejudge.org/external/124/12405.pdf)

#### 解題思路
給定一個長度為 $n$ 的田地，其中有些位置需要種植作物 (.)，有些則是荒地 (#)。一個稻草人可以覆蓋放置位置以及其左右兩側（共 3 個連續位置）
- 目標：用最少數量的稻草人覆蓋所有作物
- 核心特性：這是一個區間覆蓋問題。為了讓稻草人效益最大化，每當我們遇到一個需要覆蓋的作物時，我們應該將稻草人放在最右邊的位置，這樣能覆蓋最廣的範圍
- 核心策略：貪心法。當遇到第一個未被覆蓋的作物時，就在其右側擺放稻草人，並跳過已被覆蓋的區域

#### 關鍵邏輯：
1. 狀態維護 (r)：
    - r 紀錄當前稻草人影響範圍的右邊界。如果現在的索引 i 大於 r，代表該位置尚未被任何稻草人保護
2. 貪心策略:
    - 當 s[i] == '.' 且 i > r 時，代表發現一個「必須照顧」的作物
    - 我們將稻草人放在 i+1 的位置，它會影響 i 到 i+2 的範圍，因此將 r 更新為 i+2，並將 ans 加 1
3. 線性掃描：
    - 只要從左到右遍歷一次，就能保證找到最佳解

#### 模擬運算(以s = ".#...", n=5為例)：
1. i=0 (.)：0 > -1，發現作物！放置稻草人，r = 0+2 = 2，ans = 1
2. i=1 (#)：1 <= 2，跳過
3. i=2 (.)：2 <= 2，該位置已被前一個稻草人覆蓋，跳過
4. i=3 (.)：3 > 2，發現作物！放置稻草人，r = 3+2 = 5，ans = 2
5. i=4 (.)：4 <= 5，跳過
- 最終結果：ans = 2
- Time Complexity：$O(n)$，只需要掃描一遍田地
- Space Complexity：$O(n)$，用於儲存田地狀態字串
#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 貪心法的關鍵判斷：這題的核心在於「遇到第一個沒被照顧到的，就馬上補一個稻草人」，且位置擺得越靠右越好
    - 變數重置：在 while 迴圈內，必須對 ans 和 r 等變數進行初始化，否則會影響下一筆測資的計算
    - 格式輸出：UVa 題目通常要求輸出 Case X: Y 的格式，這是解題時最容易因疏忽被判錯誤 (Wrong Answer) 的地方
#### 總結 (Takeaway)
- 區間覆蓋貪心 SOP：
    1. 依照順序處理目標（從左到右）
    2. 當遇到未滿足的需求時，進行一次操作，並預測該操作能覆蓋的最遠邊界
    3. 跳過所有已被覆蓋的範圍
    4. 重複直到所有需求處理完畢
