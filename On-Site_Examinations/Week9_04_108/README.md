### 題目：[UVa 108 - Maximum Sum](https://onlinejudge.org/index.php?option=com_onlinejudge&Itemid=8&category=3&page=show_problem&problem=44)

#### 解題思路
在一個 $N \times N$ 的整數矩陣中，找到一個子矩陣，使得該子矩陣內的所有元素總和最大
- 目標：輸出該最大總和
- 核心挑戰：如果使用暴力法枚舉所有可能的子矩陣（左上角與右下角座標），複雜度會高達 $O(N^6)$。本解法透過將二維問題壓縮為一維，將複雜度降至 $O(N^3)$

#### 關鍵邏輯：
1. Row Prefix Sum：
    - 在讀取資料時，立即計算每一行的前綴和：a[i][j] += a[i][j-1]
    - 這樣做的好處是：我們可以在 $O(1)$ 時間內，透過 a[k][j] - a[k][i] 算出第 $k$ 行中從第 $i+1$ 到第 $j$ 列的元素和
2. Matrix Compression：
    - 我們固定左右邊界（列 $i$ 到列 $j$）
    - 將這兩列之間的每一行看作一個「複合元素」。此時，二維矩陣就變成了一條長度為 $N$ 的一維數列
3. Kadane's Algorithm：
    - 對壓縮後的一維數列執行 Kadane 演算法：
        - 累加當前和 total += row_sum
        - 隨時更新最大值 ans = max(ans, total)
        - 關鍵優化：如果 total < 0，代表前面的累加對後續只會起負面作用，因此將 total 歸零重算

#### 模擬運算($N=3$ 且固定列 1 到列 2為例)：
- 第 1 行列 1~2 和：5
- 第 2 行列 1~2 和：-2
- 第 3 行列 1~2 和：3
- 轉換為一維數列：[5, -2, 3]
- 執行：
    - 讀到 5：total = 5, ans = 5
    - 讀到 -2：total = 3, ans = 5
    - 讀到 3：total = 6, ans = 6
- 結果：這兩列之間的最大子矩陣和為 6
- Time Complexity：$O(N^3)$。兩層迴圈枚舉列邊界 ($N^2$)，內層一圈處理行累加 ($N$)
- Space Complexity：$O(N^2)$。需要空間儲存原始矩陣與前綴和

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 降維打擊：將高維度問題壓縮回低維度是 DP 中非常強大的技巧
    - 前綴和的靈活性：前綴和不只能處理一維區間，在二維矩陣中先做「列壓縮」也能發揮奇效
    - 初始值設定：ans 初始化為 0。注意：若矩陣內全是負數，題目可能要求輸出其中的最大單個負數，此時 ans 需初始化為極小值。但本題通常假設至少選一個元素或輸出 0
#### 總結 (Takeaway)
- 二維最大區間和問題 SOP：
    1. 預處理每一行的前綴和
    2. 枚舉所有可能的「列區間」（左邊界與右邊界）
    3. 將該區間內的每一行合併，簡化為一維數列
    4. 對該數列執行 Kadane's Algorithm
