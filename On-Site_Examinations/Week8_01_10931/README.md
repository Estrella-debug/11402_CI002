### 題目：[UVa 10931 - Parity](https://onlinejudge.org/index.php?option=com_onlinejudge&Itemid=8&category=21&page=show_problem&problem=1872)

#### 解題思路
給定一個正整數 $I$，求出其二進位表示法，並計算該二進位中出現「1」的總次數
1. 輸出格式：必須印出二進位字串 $B$ 以及計數結果 $cnt$
2. 核心策略：利用位元運算逐一取出二進位的每一位，直到數字變為 0 為止

#### 關鍵邏輯：
1. 位元提取 (Bit Extraction)：
    - 使用 n & 1：透過與運算（AND）取得 $n$ 的最低位元。若最低位為 1，結果為 1；若為 0，結果為 0
    - cnt += (n & 1)：直接累加提取出的位元值，即可算出「1」的總數
2. 字串建構 (Binary String)：
    - B += '0' + (n & 1)：將得到的位元（0 或 1）轉為字元 '0' 或 '1' 並存入字串中
    - 注意：這樣存入的順序會是「從低位到高位」（即反向的二進位）
3. 右移更新：
    - 使用 n >>= 1：將數字向右移 1 位，相當於除以 2，以便下次迴圈提取更高一位的位元
4. 順序修正：
    - 由於存入字串時是從最後一位開始，最後必須呼叫 reverse(B.begin(), B.end()) 將其翻轉回正確的二進位順序
#### 模擬運算($I = 21$ 為例)：
- 初始：$n = 21$
- 迴圈 1：21 & 1 = 1。cnt = 1, B = "1"。右移 $\rightarrow$ $n = 10$
- 迴圈 2：10 & 1 = 0。cnt = 1, B = "10"。右移 $\rightarrow$ $n = 5$
- 迴圈 3：5 & 1 = 1。cnt = 2, B = "101"。右移 $\rightarrow$ $n = 2$
- 迴圈 4：2 & 1 = 0。cnt = 2, B = "1010"。右移 $\rightarrow$ $n = 1$
- 迴圈 5：1 & 1 = 1。cnt = 3, B = "10101"。右移 $\rightarrow$ $n = 0$ (結束)
- 翻轉後：B = "10101"，cnt = 3
- Time Complexity：$O(\log_2 I)$。位元數與數值呈對數關係
- Space Complexity：$O(\log_2 I)$。需字串空間儲存二進位

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 位元運算 vs 算術運算：n & 1 效能優於 n % 2，n >>= 1 效能優於 n /= 2。雖然現代編譯器會自動優化，但手寫位元運算能體現對底層邏輯的理解
    - ASCII 轉換技巧：透過 '0' + (0 or 1) 快速完成數字與字元的轉換
    - 容器翻轉：了解 std::reverse 是處理「後續處理需反轉」資料的最快方式

#### 總結 (Takeaway)
- Parity 統計問題 SOP：
    1. 建立迴圈直到數字變為 0
    2. 利用 & 1 取得當前位元並累加
    3. 將位元存入字串並右移數字
    4. 翻轉字串以符合正確的進位讀取順序
