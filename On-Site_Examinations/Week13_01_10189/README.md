### 題目：[UVa 10189 - Minesweeper](https://onlinejudge.org/external/101/10189.pdf)

#### 解題思路
給定一個 $n \times m$ 的網格，其中一些格子藏有地雷 (*)
- 目標：將所有非地雷的格子替換為其周圍 8 個格子內的地雷總數
- 核心挑戰：如何精確且簡潔地檢查每個格子周圍的 8 個鄰居，並處理網格邊緣的邊界條件

#### 關鍵邏輯：
1. 方向陣列：
    - 為了避免寫 8 個獨立的 if 判斷，使用 dx 和 dy 陣列儲存 8 個方向的位移量
    - 例如 dx = {-1, -1, -1, 0, 0, 1, 1, 1} 搭配 dy 可以遍歷所有鄰居
2. Boundary Check:
    - 在訪問鄰居 (nx, ny) 前，必須確保 nx >= 0 && nx < n && ny >= 0 && ny < m，這能防止程式訪問到陣列之外的非法記憶體
3. 原地輸出 vs. 複製結果：
    - 使用 result 陣列儲存計數結果，確保讀取 field 時保持地雷位置資訊不被覆蓋，邏輯更為清晰
4. 字元轉數字：
    - 將計數 count 轉換為字元的方法：count + '0'。這會自動將整數轉為對應的 ASCII 字元
#### 模擬運算：
- Time Complexity：$O(n \times m)$。每個格子僅需遍歷一次，且鄰居數是固定的 8 個常數
- Space Complexity：$O(n \times m)$，用於儲存網格資料

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 方向陣列的威力：這是處理二維網格問題的標準模板，能大幅提升程式碼的可讀性與維護性
    - 格式要求：UVa 題目對輸出格式要求嚴格，例如「兩次測試資料間需要空行」，這裡透過 if (field_num > 1) 邏輯來處理
    - IO 效能：使用 ios::sync_with_stdio(false) 能顯著加快大規模輸入的讀取速度
#### 總結 (Takeaway)
- 踩地雷/網格周邊統計 SOP：
    1. 建立方向陣列（8 個方向）
    2. 讀入網格，並複製一份用於儲存結果
    3. 兩層迴圈遍歷所有格子
    4. 若非地雷，執行 8 個方向的鄰居檢查
    5. 執行邊界檢測後進行累加計數
