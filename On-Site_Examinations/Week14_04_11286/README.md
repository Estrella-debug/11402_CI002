### 題目：[UVa 11286 - Conformity](https://onlinejudge.org/external/112/11286.pdf)

#### 解題思路
每一位學生會選修 5 門課程，選課組合的順序並不重要
- 目標：找出出現頻率最高的選課組合，並回傳「所有選擇此熱門組合的學生總數」
- 核心特性：由於順序無關，我們需要先對每位學生的 5 門課程進行排序，使其成為唯一的「標準表示法」
- 核心策略：利用 std::map<string, int> 將標準化的組合對應到其出現次數

#### 關鍵邏輯：
1. 組合標準化：
    - 對每位學生的 5 門課進行 sort()，接著串接成一個字串
2. 頻率統計:
    - 使用 map<string, int> 統計每一種組合出現的次數。map 會自動處理字串鍵值，非常方便
3. 計算最高頻率與總人數：
    - 第一次遍歷 map 找出最高頻率 max_num
    - 第二次遍歷 map，將所有頻率等於 max_num 的組合之計數值累加到 total 中

#### 模擬運算(以3 位學生為例)：
- 學生 1：選 {500, 100, 200, 400, 300} $\rightarrow$ 排序後變成 100200300400500，map 紀錄 {100200300400500: 1}
- 學生 2：選 {100, 200, 300, 400, 500} $\rightarrow$ 排序後變成 100200300400500，map 紀錄 {100200300400500: 2}
- 學生 3：選 {111, 222, 333, 444, 555} $\rightarrow$ 排序後變成 111222333444555，map 紀錄 {111222333444555: 1}
- 結論：最高頻率為 2，對應組合為 100200300400500，total = 2
- Time Complexity：$O(N \times K \log K)$，其中 $N$ 為學生數，$K=5$ 為課程數（排序耗時）
- Space Complexity：$O(N \times K)$，用於存儲 map 中的組合字串
#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - Map 的便利性：在處理需要將字串或複雜型別對應到計數的問題時，map 或 unordered_map 是首選
    - 組合無序問題：處理「集合相等」問題時，預先排序 (Sort) 是將其轉化為「字串相等」最簡單的手段
    - 清空與重置：在多組測資環境下，記得使用 course.clear() 重置地圖，否則數據會跨測試案例累加
#### 總結 (Takeaway)
- 集合組合統計 SOP：
    1. 將一組資料轉化為固定順序（排序）
    2. 將排序後的資料轉化為單一字串或 ID 作為 Key
    3. 使用 Map 統計每個 Key 的出現次數
    4. 掃描 Map 找出最大值及其對應的總人數
