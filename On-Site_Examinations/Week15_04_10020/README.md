### 題目：[UVa 10020 - Minimal coverage](https://onlinejudge.org/external/100/10020.pdf)

#### 解題思路
給定一個目標區間 $[0, M]$ 和一堆線段 $[L_i, R_i]$，要求從中選出最少數量的線段，使得它們的聯集包含 $[0, M]$
- 目標：最小化所選線段的數量
- 核心策略：貪心法。在當前已覆蓋的區間 $[0, L]$ 之基礎上，我們總是要選擇所有「起點 $\le L$」且「終點 $R$ 最遠」的線段，這樣才能確保每一步的覆蓋擴展效率最高，從而使總數最少

#### 關鍵邏輯：
1. Sorting：
    - 將所有線段依照左端點 $L$ 從小到大排序。這是貪心策略能夠順利執行指標掃描的前提
2. Greedy Selection：
    - 維護一個當前已覆蓋的右邊界 $L$（初始為 0）
    - 在所有「能與當前 $L$ 接上」（即 $seg[i].L \le L$）的線段中，挑選一個右端點 $R$ 最大的線段
3. 更新狀態：
    - 選定最佳線段後，更新 $L = mxR$
    - 若在某個步驟中，所有符合條件的線段都無法延伸目前的覆蓋範圍，則代表 $[0, M]$ 無法被完全覆蓋

#### 模擬運算(以$[0, 5]$ 為例，線段為 $\{[0, 2], [0, 3], [1, 4], [3, 6]\}$)：
1. 初始 $L=0$：
    - 可選線段有 $\{[0, 2], [0, 3]\}$
    - 選取 $R$ 最大的 $[0, 3]$，更新 $L=3$
2. 目前 $L=3$：
    - 可選線段有 $\{[1, 4], [3, 6]\}$
    - 選取 $R$ 最大的 $[3, 6]$，更新 $L=6$
3. 完成：$L=6 \ge M=5$，成功覆蓋
- Time Complexity：$O(N \log N)$（排序耗時 $O(N \log N)$，遍歷 $O(N)$）
- Space Complexity：$O(N)$，儲存所有線段與答案
#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 貪心的陷阱：很多人會優先考慮線段長度，但這題的關鍵不在長度，而在於「哪條線段能讓已覆蓋的右邊界延伸得最遠」
    - 資料邊界處理：對於 0 0 的終止符號，以及 L >= M 的判定，是處理這類輸入輸出的基本功
    - 輸出格式：UVa 題目的空行要求非常嚴格（每組測資後要輸出一個空行），務必在 while 迴圈結束前正確處理
#### 總結 (Takeaway)
- 最小覆蓋問題 SOP：
    1. 排序：依照左端點升序排列
    2. 掃描：在 $start \le current\_L$ 的條件下，尋找 $end$ 最大的那個線段
    3. 更新：將 $current\_L$ 更新為該線段的 $end$
    4. 檢查：若無法繼續延伸且未達目標，則回傳失敗；若達標，則回傳所選線段集合
