### 題目：[UVa 11450 - Wedding Shopping](https://onlinejudge.org/external/114/11450.pdf)

#### 解題思路
我們需要為婚禮購買 $C$ 件衣物，每種衣物有多種款式（不同價格）可選
- 規則：必須且只能為每一種衣物購買一件，且總花費不能超過預算 $M$
- 目標：在預算內找到「花費最多」的組合，若無法買齊則輸出 no solution
- 核心策略：使用 記憶化搜尋 (Memoization)。這是一個典型的「多階段決策問題」，每個階段（購買某種衣物）都會影響下一階段的可用預算

#### 關鍵邏輯：
1. 狀態定義：
    - money：目前剩餘的預算
    - g：目前正在購買第幾種衣物（階段）
    - dp 表紀錄了在「剩餘預算」與「已買衣物數量」下的最大消費總和
2. 記憶化處理:
    - 使用 dp 陣列儲存重複的狀態計算結果。若 dp[money][g] != -1，直接回傳，避免指數級的遞迴運算
3. 遞迴轉移：
    - 對於每一種衣物，嘗試該類別下所有可選的價格：shop(money - price[g][i], g + 1)
    - 如果預算變成負數（money < 0），回傳一個極小值（如 -1e9）表示此路不通
4. Base Case:
    - 當 g == C（所有衣物都買完了），回傳已花費的金額（M - money）

#### 模擬運算(以預算 $M=10, C=2$ 為例)：
- 階段 0：嘗試購買第一種衣物，花費 $p_i$
- 階段 1：根據剩餘預算，嘗試購買第二種衣物
- 分支剪枝：如果某次購買導致剩餘預算 $< 0$，該遞迴分支立即回傳負數，在 max 比較中會被淘汰
- Time Complexity：$O(M \times C \times K)$，其中 $K$ 為平均每個類別的款式數量
- Space Complexity：$O(M \times C)$，即 dp 陣列的大小

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 狀態重置的重要性：每次處理新的一筆測試資料（Test Case）時，必須使用 memset 將 dp 表重置為 -1，否則會污染結果
    - 背包問題的變形：這與標準的 0/1 背包不同之處在於「每個階段必須選一個」，這使得邏輯更像是一個有限制的分層選擇問題
    - 記憶化與遞迴的搭配：這種寫法非常優雅，將原本複雜的組合搜尋轉化為填表問題
#### 總結 (Takeaway)
- 階段性選擇問題 SOP：
    1. 定義階段 g（買到第幾樣）與狀態 money（剩下多少錢）
    2. 定義遞迴函式，包含 Base Case（買完）與失敗條件（錢不夠）
    3. 使用 dp 陣列存儲計算過的結果
    4. 主函式記得重置 DP 狀態
