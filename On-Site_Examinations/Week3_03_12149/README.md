### 題目：[UVa 12149 - Feynman](https://onlinejudge.org/index.php?option=com_onlinejudge&Itemid=8&category=24&page=show_problem&problem=3301)

#### 解題思路
這題的目標是數出一個 n * n 的大正方形（square）裡面，總共藏了多少個大大小小的正方形。我們可以從最簡單的例子開始找規律（rule）：
- 如果是 1 * 1 的方格，只有 1 個正方形 ($1^2$)
- 如果是 2 * 2 的方格，有 4 個小的 1 * 1，加上 1 個大的 2 * 2，總共 5 個 ($2^2 + 1^2$)
- 如果是 3 * 3 的方格，會有 9 個 1 * 1、4 個 2 * 2、1 個 3 * 3，總共 14 個 ($3^2 + 2^2 + 1^2$)

#### 關鍵邏輯：
1. 數學規律 (Math Rule)：我們發現總數的公式（formula）其實就是把數字的平方加起來：1*1 + 2*2 + 3*3 + ... + n*n。
2. 結束條件 (Stop Condition)：題目說當輸入的 n 是 0 的時候就要結束程式。所以在 while 迴圈（loop）裡面，加上 if (n == 0) break; 是必須的。
3. 累加器 (Accumulator)：每次讀到一個新的 n，就要準備一個變數 sum = 0 來把每次的 i * i 加起來。

#### 模擬運算：
- 第一步：i = 1，加進第一個正方形：sum = 0 + (1 * 1) = 1。
- 第二步：i = 2，加進 2 * 2 的正方形數量：sum = 1 + (2 * 2) = 5。
- 第三步：i = 3，加進 1 * 1 的正方形數量：sum = 5 + (3 * 3) = 14。
- 結果：迴圈結束，印出 14。
- Time Complexity (時間複雜度)：O(n)。迴圈會執行 n 次。
- Space Complexity (空間複雜度)：O(1)。我們只用了一個變數 sum。

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 變數歸零 (Reset Variable)：sum = 0 一定要寫在 while 迴圈的「裡面」。如果寫在外面，上一次算完的答案會被留下來，造成下一輪計算錯誤。
    - 更快的公式解 (Faster Formula)：雖然用 for 迴圈很簡單，但其實有一個公式可以直接算出答案，不用跑迴圈,如果 n 非常大，直接套用這個公式計算會超級快！

#### 總結 (Takeaway)
- 處理圖形數學題的 SOP：
    1. 在紙上畫出 n=1、n=2、n=3 的情況。
    2. 把答案寫下來，找出數字變化的規律（pattern）。
    3. 用迴圈（loop）把這個規律寫成程式碼。
- 只要冷靜找規律，很多看似複雜的圖形題都能變成簡單的加減乘除。
