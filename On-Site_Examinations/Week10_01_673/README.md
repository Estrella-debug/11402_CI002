### 題目：[UVa 673 - Parentheses Balance](https://onlinejudge.org/index.php?option=com_onlinejudge&Itemid=8&category=8&page=show_problem&problem=614)

#### 解題思路
給定一個包含小括號 () 與中括號 [] 的字串，判斷其是否為正確的平衡組合
- 合法條件：
    1. 每個左括號必須有對應的右括號
    2. 括號必須正確嵌套（例如 ([)] 是不合法的，因為 [ 必須在 ) 之前先被 ] 閉合）
- 核心策略：利用堆疊 「後進先出 (LIFO)」 的特性。遇到左括號就存入，遇到右括號就檢查堆疊最上層是否為對應的左括號

#### 關鍵邏輯：
1. 左括號處理：
    - 遇到 ( 或 [，直接使用 S.emplace(c) 放入堆疊
2. 右括號與配對檢查：
    - 匹配成功：遇到 ] 且堆疊頂端是 [，或是遇到 ) 且堆疊頂端是 (。此時代表配對成功，將左括號從堆疊彈出 (S.pop())
    - 匹配失敗：如果堆疊為空，或者頂端不是對應的左括號，則將該右括號也放入堆疊中。這是一個巧妙的作法，因為一旦堆疊中留下了右括號，最後堆疊就不可能為空，從而判定為 No
3. 輸入緩衝區處理：
    - 使用 cin >> T 後接 cin.ignore()，是為了清除換行符，避免 getline 讀到空的測試資料
4. 結果判定：
    - 遍歷完字串後，若 S.empty() 為真，代表所有括號都完美配對，輸出 Yes

#### 模擬運算(([])為例)：
- 讀到 (：堆疊 [(]
- 讀到 [：堆疊 [(, []
- 讀到 ]：頂端是 [，匹配成功，彈出。堆疊 [(]
- 讀到 )：頂端是 (，匹配成功，彈出。堆疊 [] (空)
- 結果：Yes
- Time Complexity：$O(n)$。對字串中的每個字元僅進行一次常數時間的操作
- Space Complexity：$O(n)$。最差情況下（全為左括號）需儲存整個字串

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - getline 的重要性：題目中提到可能會有空字串（Empty string is balance），一般的 cin >> str 會跳過空字串，必須使用 getline 才能正確處理
    - emplace vs push：S.emplace(c) 在效能上略優於 push，因為它直接在容器內構造元素
    - IO 優化：使用了 ios::sync_with_stdio(false) 提高在處理大量測資時的效率
#### 總結 (Takeaway)
- 括號匹配問題 SOP：
    1. 建立一個字元堆疊
    2. 左括號入棧
    3. 右括號比對棧頂：成功則彈出，失敗則入棧（或直接回傳失敗）
    4. 最後檢查堆疊是否全空
