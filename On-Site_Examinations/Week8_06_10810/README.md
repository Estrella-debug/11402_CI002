### 題目：[UVa 10810 - Ultra-QuickSort](https://onlinejudge.org/index.php?option=onlinejudge&page=show_problem&problem=2296)

#### 解題思路
給定一個長度為 $n$ 的數列，只允許透過「交換相鄰元素」的方式進行排序
- 目標：計算最少需要交換幾次才能完成排序
- 數學本質：在排序理論中，「相鄰交換」的最少次數等於數列中的 逆序數對 (Inversions) 總數
- 效能挑戰：如果使用氣泡排序 ($O(n^2)$) 會因為資料量高達 500,000 筆而超時（TLE）。因此必須改用 合併排序 (Merge Sort) 的分治法邏輯

#### 關鍵邏輯：
1. Divide and Conquer：
    - 將數列不斷對半拆解，直到每個部分只有一個數字
    - 在「Merge」的過程中，同時計算逆序數對
2. The Core Formula：
    - 當我們在合併左、右兩個子陣列時，若發現「右邊子陣列的數字 input[idxr] 小於左邊子陣列的數字 input[idxl]」，代表發生了逆序
    - 關鍵公式：counter += (mid - idxl + 1);
    - 原理：因為左、右子陣列都已各自排序，若右邊的數比左邊目前的數小，代表它也比左邊剩下還沒被處理的所有數字都小
3. 大數處理：
    - 交換次數可能非常龐大（超過 $2 \times 10^9$），因此 counter 必須宣告為 long long int 以防溢位
4. 空間分配：
    - 使用一個全域的輔助陣列 ans[] 來存放合併結果，避免在遞迴中頻繁宣告記憶體導致效能下降

#### 模擬運算( [3, 1, 2] 為例)：
- 拆分：[3] 與 [1, 2]
- 合併右半部 [1, 2]：沒有逆序，counter = 0
- T最後合併 [3] 與 [1, 2]：
    - 比較 3 與 1：1 較小，發生逆序！counter += (0 - 0 + 1)，counter = 1
    - 比較 3 與 2：2 較小，發生逆序！counter += (0 - 0 + 1)，counter = 2
- 結果：總交換次數為 2
- Time Complexity：$O(n \log n)$。與合併排序相同，是處理大量逆序數對問題的最優解
- Space Complexity：$O(n)$。需要額外的輔助陣列儲存排序過程

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 演算法的轉化：學會將看似簡單的「排序交換」問題連結到「逆序數對」，並用更高效的演算法解決
    - 遞迴深度與空間：在 C++ 中處理 50 萬筆數據時，陣列應盡量開在全域（靜態區）或堆積（Heap），避免棧溢位
    - 長整數的必要性：很多初學者會卡在 int 溢位，養成計算累加值時先評估數值範圍的習慣非常重要
#### 總結 (Takeaway)
- 逆序數對/交換次數問題 SOP：
    1. 確認題目是否等同於尋找逆序對
    2. 實作合併排序邏輯
    3. 在右方元素先被移入結果陣列時，累加「左方剩餘元素數量」
    4. 使用 long long 確保大數運算的正確性
