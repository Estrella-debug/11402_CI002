### 題目：[UVa 495 - Fibonacci Freeze](https://onlinejudge.org/external/4/495.pdf)

#### 解題思路
計算費氏數列的第 $n$ 項
- 目標：輸出 $F(n)$ 的精確值，其中 $n \le 5000$
- 核心特性：當 $n$ 達到 5000 時，結果將是一個擁有超過一千位數字的龐大數值，傳統程式語言內建的資料型態無法儲存
- 核心策略：使用「字串」來儲存大數，並透過「模擬小學加法」的方式（從低位到高位、處理進位）來實作大數相加

#### 關鍵邏輯：
1. 大數加法：
    - 將兩個字串轉為數字逐位相加，由最後一位開始往前處理
    - 使用 carry 變數處理進位
    - 最後將結果反轉 (reverse) 以符合正確的數值顯示順序
2. 預計算：
    - 建立一個長度為 5001 的 vector<string> 陣列，在程式啟動時將所有答案預先算出。這樣每次處理輸入時只需 $O(1)$ 的時間即可查表輸出

#### 模擬運算(以 $F(0) \to F(3)$ 為例)：
1. fib[0] = "0", fib[1] = "1"
2. fib[2] = "1" + "0" = "1"
3. fib[3] = "1" + "1" = "2"
4. fib[4] = "2" + "1" = "3"
- Time Complexity：$O(N^2)$，因為 $N$ 個數值，每個數字平均位數為 $O(N)$，加法需要 $O(N)$
- Space Complexity：$O(N^2)$，用於儲存所有位數的字串結果

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 處理大數的必要性：在數論或競賽題目中，當看到「數字極大」或「輸出長度驚人」的題目，第一反應應是評估是否需要實作大數運算
    - 字串處理技巧：a[i--] - '0' 是將字元轉為數字的精簡寫法，reverse() 則是處理加法運算後字串順序的標準工具
    - IO 優化：使用 ios_base::sync_with_stdio(false) 能顯著提升大量文字輸出的效率，這對於 $N=5000$ 且測資眾多的題目是必要的優化

#### 總結 (Takeaway)
- 大數費氏數列 SOP：
    1. 實作一個處理兩個字串加法的函式 stringAdd，確保考慮進位
    2. 使用 vector<string> 作為記憶體表，預先運算至最大限制 $N=5000$
    3. 讀取測資時直接查表輸出
