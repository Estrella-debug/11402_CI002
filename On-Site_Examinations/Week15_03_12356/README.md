### 題目：[UVa 12356 - Army Buddies](https://onlinejudge.org/external/123/12356.pdf)

#### 解題思路
有一排 $S$ 名士兵排成一列。當有 $B$ 個任務發生時，每項任務會指定一個區間 [L, R]，該區間內的所有士兵都會陣亡。每陣亡一組後，我們需要輸出原本在陣亡區間左側的最鄰近士兵，以及右側的最鄰近士兵
- 目標：在多次區間刪除下，即時輸出左右鄰居編號
- 核心特性：若使用 vector 進行刪除，每次刪除都要 $O(S)$ 的時間移動元素，總時間複雜度會高達 $O(S \times B)$，必然會超時
- 核心策略：利用「陣列模擬雙向鏈結串列」，維護每個士兵的 left 和 right 指針。刪除區間 [L, R] 時，只需將 L 左側士兵的 right 指向 R 右側士兵，以及將 R 右側士兵的 left 指向 L 左側士兵，實現 $O(1)$ 的刪除操作

#### 關鍵邏輯：
1. 結構初始化：
    - left[i] 儲存士兵 $i$ 左邊鄰居的編號
    - right[i] 儲存士兵 $i$ 右邊鄰居的編號
2. 動態更新：
    - 收到陣亡區間 [L, R] 後：
    - 左鄰居的更新：right[left[L]] = right[R]
    - 右鄰居的更新：left[right[R]] = left[L]
3. 邊界判斷：
    - 若某側已經沒有人，陣列儲存 0，並輸出 *

#### 模擬運算(以5 名士兵，陣亡 [2, 3]為例)：
1. 初始狀態：
    - left = [0, 0, 1, 2, 3, 4]
    - right = [0, 2, 3, 4, 5, 0]
2. 執行陣亡 [2, 3]：
    - 士兵 2 左鄰居是 1，士兵 3 右鄰居是 4
    - 更新：right[1] = 4, left[4] = 1
    - 輸出：1 4
- Time Complexity：$O(S + B)$。初始化 $O(S)$，每次刪除操作 $O(1)$
- Space Complexity：$O(S)$，儲存左右指標陣列
#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - Simulated Linked List：在處理需要頻繁刪除中間元素的題目時，不要直接操作 vector。使用兩個陣列來模擬指標，既擁有指標的高效（刪除 $O(1)$），又擁有陣列的記憶體空間連續性
    - I/O 效能：ios_base::sync_with_stdio(false)，否則標準輸出 cout 會成為導致 TLE (Time Limit Exceeded) 的元兇
    - Sentinel Value: 使用 0 代表邊界是個簡化邏輯的好方法，但在操作時要小心避免存取到 index 0
#### 總結 (Takeaway)
- 陣列模擬指標 SOP：
    1. 建立 left 與 right 兩個陣列，分別記錄每個元素的左右鄰居
    2. 刪除區間 [L, R] 時，更新邊界：right[left[L]] = right[R] 且 left[right[R]] = left[L]
    3. 若更新後指標指向哨兵（如 0），則輸出 *
