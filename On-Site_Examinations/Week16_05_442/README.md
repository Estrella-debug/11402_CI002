### 題目：[UVa 442 - Matrix Chain Multiplication](https://onlinejudge.org/external/4/442.pdf)

#### 解題思路
給定若干矩陣的名稱與維度，並輸入包含括號的乘法運算式。矩陣相乘規則為：若矩陣 $A$ 維度為 $r_1 \times c_1$，矩陣 $B$ 維度為 $r_2 \times c_2$，則 $A$ 與 $B$ 可相乘的條件為 $c_1 = r_2$，乘法運算次數為 $r_1 \times c_1 \times c_2$，且結果矩陣維度為 $r_1 \times c_2$
- 目標：計算運算式的總運算次數，或判定為不可相乘
- 核心特性：運算式中括號明確定義了計算順序，這正是使用堆疊處理運算式的絕佳場景

#### 關鍵邏輯：
1. 矩陣對照表：使用 map<char, Matrix> 儲存矩陣名稱與對應的維度
2. 堆疊處理運算式：
    - 遍歷運算式中的每個字元：
        - 若為 字母：表示一個矩陣，直接存入堆疊
        - 若為 右括號 )：代表一次相乘運算，從堆疊中彈出最後兩個矩陣
        - 檢查規則：若 m1.cols != m2.rows，則運算不合法，標記為 error
        - 累加成本：total_mult += m1.rows * m1.cols * m2.cols，並將結果矩陣 {m1.rows, m2.cols} 推回堆疊
    - 若為 左括號 (：忽略即可（因為右括號已經定義了運算時機）

#### 模擬運算((AB)C 為例)：
假設 $A(10 \times 20), B(20 \times 30), C(30 \times 40)$：
1. 讀取 (：跳過
2. 讀取 A、B：推入堆疊 [A, B]
3. 讀取 )：彈出 B, A。$A.cols(20) == B.rows(20)$，合法。成本累加 $10 \times 20 \times 30 = 6000$。推入結果矩陣 D(10 \times 30)，堆疊：[D]
4. 讀取 C：堆疊：[D, C]
5. 讀取 )：彈出 C, D。$D.cols(30) == C.rows(30)$，合法。成本累加 $6000 + (10 \times 30 \times 40 = 12000) = 18000$
- 結果：總成本 18000
- Time Complexity：$O(L)$，其中 $L$ 為運算式字串長度
- Space Complexity：$O(N + L)$，用於儲存矩陣定義與堆疊

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - Expression Parsing：括號決定了計算優先級，堆疊是解決此類問題的最標準資料結構
    - 邏輯順序的重要性：彈出堆疊時，順序為 m2 = top(); pop(); m1 = top(); pop();，因為堆疊特性為後進先出，m2 才是右邊的矩陣，這點務必留意
    - 錯誤狀態維護：在運算過程中一旦發現維度不符，應立即 break 跳出迴圈，並透過 error 變數在輸出時正確回報 error

#### 總結 (Takeaway)
- 矩陣連乘運算 SOP：
    1. 建立矩陣名稱至維度的映射表
    2. 使用堆疊存放處理中的矩陣物件
    3. 遇到右括號即彈出兩個矩陣並執行乘法驗證與運算
    4. 將計算後的新矩陣推回堆疊，累積運算成本
