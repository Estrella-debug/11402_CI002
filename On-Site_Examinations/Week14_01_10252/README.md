### 題目：[UVa 10252 - Common Permutation](https://onlinejudge.org/external/102/10252.pdf)

#### 解題思路
給定兩個字串 $a$ 和 $b$，找出它們的「共同排列」
- 定義：找出兩個字串中皆出現過的字元，若字元 $x$ 在 $a$ 中出現 $n$ 次，在 $b$ 中出現 $m$ 次，則該字元 $x$ 應該在結果中出現 $\min(n, m)$ 次
- 目標：輸出結果必須按照字母順序排列
- 核心策略：利用字元陣列（頻率表）統計各字元出現次數，再進行比較

#### 關鍵邏輯：
1. 頻率統計：
    - 利用 count(a.begin(), a.end(), 97+i)，其中 97 是 ASCII 的 'a'，遍歷 i 從 0 到 25 來統計 'a' 到 'z' 在字串中出現的次數
2. 交集計算:
    - i對於每個字母，其在共同序列中出現的次數為兩個字串各自出現次數的最小值：min(countA, countB)
3. 字典序輸出：
    - 由於我們是從 i = 0 ('a') 到 25 ('z') 依序檢查並輸出，這保證了最終輸出的結果自然符合字典序，無需額外排序

#### 模擬運算(以$a=$ "aba", $b=$ "baa" 為例)：
- 統計 a：'a': 2, 'b': 1
- 統計 b：'a': 2, 'b': 1
- 比較：
    - 'a': min(2, 2) = 2
    - 'b': min(1, 1) = 1
- 輸出：輸出兩個 'a'，接著輸出一個 'b'。結果為 "aab"
- Time Complexity：$O(26 \times (N+M))$，其中 $N$ 和 $M$ 為字串長度。因 26 為常數，實際可視為 $O(N+M)$
- Space Complexity：$O(1)$，固定使用大小為 26 的陣列
#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - ASCII 映射：理解了如何透過 97 + i 將數字映射回字母
    - getline 的使用：在 UVa 等題目中，輸入字串可能包含空格，使用 getline(cin, s) 比 cin >> s 更能正確讀取完整的一行
    - 計數器重置：每次 while 迴圈開始時，都要確保頻率相關變數被清空，避免上一次的數據干擾當前計算
#### 總結 (Takeaway)
- 字元交集問題 SOP：
    1. 建立長度為 26 的頻率陣列
    2. 對兩個輸入字串分別統計
    3. 兩兩對應取最小值
    4. 依序遍歷陣列並輸出對應次數的字元
