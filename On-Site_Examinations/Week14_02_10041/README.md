### 題目：[UVa 10041 – Vito’s Family](https://onlinejudge.org/external/100/10041.pdf)

#### 解題思路
Vito 有許多親戚散佈在數線上的不同位置，他希望在這些親戚中選擇一個居住地（或者選擇一個位置），使得所有親戚到該位置的總距離之和最小
- 目標：最小化 $\sum |s_i - x|$，其中 $x$ 為選擇的位置
- 核心特性：根據數學性質，當目標是使「絕對距離總和」最小化時，這個點 $x$ 必然是所有資料的 中位數 (Median)
- 核心策略：將資料排序後，取中間位置的數值即可達到總距離最小

#### 關鍵邏輯：
1. 資料排序：
    - 為了找到中位數，必須先將所有親戚的位置 $s_i$ 由小到大排序
2. 定位中位數:
    - 對於包含 $r$ 個親戚的陣列，中位數位於 s[r/2]
    - 如果 $r$ 為奇數，中位數是唯一的；如果 $r$ 為偶數，選取兩中間值中的任意一個皆可得到相同的最小總距離
3. 計算總和：
    - 計算所有親戚位置與中位數的距離差的絕對值之和

#### 模擬運算(以親戚位置 {2, 4, 9}為例)：
- 排序：{2, 4, 9}
- 選取中位數：r=3，r/2 = 1，故中位數為 s[1] = 4
- 計算距離總和：
    - |2 - 4| = 2
    - |4 - 4| = 0
    - |9 - 4| = 5
    - 總和：2 + 0 + 5 = 7
- Time Complexity：$O(R \log R)$，主要花費在排序上
- Space Complexity：$O(R)$，用於儲存親戚位置
#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 中位數 vs. 平均數：很多人直覺會想用平均數，但平均數會讓「平方距離總和」最小化，而「絕對距離總和」最小化必須使用中位數。這是一個非常實用的數學常識
    - 排序的重要性：只要一個排序，問題就從複雜的搜尋變為簡單的 $O(1)$ 選取
    - 資料規模：使用 ios_base::sync_with_stdio(0) 可以確保在大數據量下讀寫依然快速，這是競賽程式碼的標準配備
#### 總結 (Takeaway)
- 找尋「距離總和最小點」SOP：
    1. 將所有位置放入陣列
    2. 對陣列進行升序排序
    3. 直接存取 array[size/2] 取得中位數
    4. 計算所有點到中位數的距離差並加總
