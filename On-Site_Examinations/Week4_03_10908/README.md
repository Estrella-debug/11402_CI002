### 題目：[UVa 10908 - Largest Square](https://onlinejudge.org/index.php?option=com_onlinejudge&Itemid=8&category=21&page=show_problem&problem=1849)

#### 解題思路
這題的目標是在一個充滿字元的網格（grid）中，給定一個中心點 $(r, c)$，找出以該點為中心且所有字元都相同的「最大正方形」邊長。

#### 關鍵邏輯：
1. 向外擴散 (Expansion)：從中心點開始，半徑 $k$ 從 0 開始遞增（$k=0, 1, 2, ...$）。
2. 邊界檢查 (Boundary Check)：
- 檢查正方形的四個邊界是否超出網格範圍：$r-k, r+k, c-k, c+k$。
- 檢查該範圍內的所有字元是否都與中心點的字元 centerChar 相同。
3. 邊長公式 (Side Formula)：如果半徑為 $k$ 的範圍內字元都合法，則正方形的邊長為 $2k + 1$。
4. 停止條件 (Stop Condition)：一旦發現超出範圍或字元不同，就停止擴散，前一個合法的邊長即為答案。

#### 模擬運算：
- $k = 0$：檢查 $(r, c)$ 本身，邊長為 $2(0)+1 = 1$。
- $k = 1$：檢查從 $(r-1, c-1)$ 到 $(r+1, c+1)$ 的 $3 \times 3$ 區域。
- $k = 2$：檢查從 $(r-2, c-2)$ 到 $(r+2, c+2)$ 的 $5 \times 5$ 區域。
- Time Complexity (時間複雜度)：最壞情況下為 $O(T \times Q \times M \times N)$。雖然看起來很大，但因為實務上很快就會遇到不同字元而 break，所以能通過測試。
- Space Complexity (空間複雜度)：$O(M \times N)$。我們需要一個 vector<string> 來儲存整個地圖。

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 雙重迴圈跳脫：在檢查字元時，使用 bool possible 搭配 break 可以快速跳出多層迴圈，節省執行時間。
    - 座標對應：在處理二維陣列時，要小心 $r$（列）對應地圖的第一維，而 $c$（欄）對應第二維，順序不可弄反。
    - 輸入格式：題目要求輸出時先印出網格的大小（$M, N, Q$），這點在寫程式時要特別留意輸出規範。

#### 總結 (Takeaway)
- 區域搜尋問題 SOP：
    1. 確立中心點並讀取基準值。
    2. 設計迴圈逐步擴大搜尋半徑。
    3. 嚴格檢查邊界條件（不可小於 0 或超過 $M, N$）。
- 這種「由內而外」的暴力搜尋法在網格範圍不大時非常直覺且有效。
