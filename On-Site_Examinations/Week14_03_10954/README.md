### 題目：[UVa 10954 - Add All](https://onlinejudge.org/external/109/10954.pdf)

#### 解題思路
給定 $n$ 個數字，每次可以選擇任意兩個數字相加，產生的「成本」即為這兩個數的和。目標是將這 $n$ 個數字最終合併成一個數，並使得過程中所有合併成本的總和最小
- 目標：最小化所有中間合併步驟的和
- 核心特性：這是一個貪心問題。為了讓總成本最小，我們應該在每次合併時，優先選擇當前數列中 最小的兩個數字 進行相加
- 核心策略：使用 最小堆疊 (Min-Heap)，確保每次取出的都是目前剩餘數字中最小的兩個

#### 關鍵邏輯：
1. 最小堆疊維護：
    - 使用 priority_queue<int, vector<int>, greater<int>> 建立最小堆疊
    - 這能保證我們在 $O(\log N)$ 的時間內取出最小值，並在 $O(\log N)$ 的時間內將新產生的數字插回
2. 合併流程:
    - 只要佇列中還有超過 1 個數字，就執行以下操作：
        1. 取出最小的兩個數 temp1 與 temp2
        2. 計算合併後的和 sum = temp1 + temp2
        3. 將 sum 累加到總成本 result 中
        4. 將 sum 放回優先佇列，繼續下一輪合併
3. 為什麼貪心有效：
    - 每次合併的結果都會被之後的步驟再次加總。因此，越小的數字應該越早被合併，這樣它被重複加總的次數才會越少，總成本才會最低

#### 模擬運算(以輸入 {1, 2, 3}為例)：
- 初始佇列：[1, 2, 3]
- 第一步合併：取出 1 和 2，和為 3。總成本 result = 3
    - 將 3 放回，佇列狀態：[3, 3]
- 第二步合併：取出 3 和 3，和為 6。總成本 result = 3 + 6 = 9
    - 將 6 放回，佇列狀態：[6]
- 結果：輸出 9
- Time Complexity：$O(N \log N)$。每個數字會被取出和放入堆疊數次，堆疊操作為 $\log N$
- Space Complexity：$O(N)$，用於儲存數字的堆疊
#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 貪心原則：理解了對於此類「多次加總」的問題，優先處理最小項是降低成本的不二法門
    - STL 語法：熟練了 priority_queue 在 C++ 中配合 greater<int> 實作最小堆疊的寫法
    - IO 優化：當測試資料量大時，ios::sync_with_stdio(false) 能顯著避免因輸出入效率導致的 TLE (Time Limit Exceeded)
#### 總結 (Takeaway)
- 最小合併成本問題 SOP：
    1. 建立Min-Heap
    2. 將所有數字加入堆疊
    3. 不斷取出最小的兩個數相加，直到剩下一項
    4. 累加每次相加的和即為答案
