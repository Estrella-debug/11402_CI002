### 題目：[UVa 10935 - Throwing cards away I](https://onlinejudge.org/index.php?option=com_onlinejudge&Itemid=8&page=show_problem&problem=1876)

#### 解題思路
給定一疊編號 $1$ 到 $n$ 的牌（1 在最上方），重複執行以下動作直到只剩下一張牌：
1. 丟棄：將最上面的牌丟掉
2. 移動：將現在最上面的牌移到這疊牌的最下方
- 目標：依序輸出被丟棄的卡牌編號，以及最後留下的那張牌
- 核心策略：利用 std::queue 模擬這疊牌。丟棄動作對應 pop()，而移動動作則對應將 front() 重新 push() 到隊尾後再次 pop()

#### 關鍵邏輯：
1. Initialization：
    - 將 $1 \sim n$ 的數字依序推入 queue 中
2. 循環處理規則：
    - 總共會執行 $n-1$ 次丟棄動作
    - 步驟 A：輸出並移除 q.front()（丟棄）
    - 步驟 B：將新的 q.front() 取出並推入隊尾（q.push(q.front())），接著將其從原位置移除
3. Formatting：
    - 丟棄的卡片之間需要逗號與空格間隔（例如  1, 2, 3）
    - 程式透過 if (i != 0) cout << ','; 巧妙地處理了第一個元素前不加逗號的邏輯
4. 邊界情況：
    - 當 $n=1$ 時，迴圈不會執行，直接輸出 Discarded cards:（後方無數字）並顯示唯一的剩餘卡片

#### 模擬運算($n=4$為例)：
- 初始狀態：q = [1, 2, 3, 4]
- 第一次操作：
    - 丟棄 1，q = [2, 3, 4]
    - 將 2 移到最後，q = [3, 4, 2]
- 第二次操作：
    - 丟棄 3，q = [4, 2]
    - 將 4 移到最後，q = [2, 4]
- 第三次操作：
    - 丟棄 2，q = [4]
    - 將 4 移到最後（此時只有一個元素，位置不變）
- 結果：丟棄 1, 3, 2，剩餘 4
- Time Complexity：$O(n)$。每張牌最多被移動或丟棄常數次
- Space Complexity：$O(n)$。需要一個佇列儲存 $n$ 張牌

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 循環邏輯的簡化：透過佇列的 pop 與 push 組合，可以非常輕鬆地模擬「圓環」或「牌堆底層」的移動，而不需要手動處理陣列索引
    - 輸出格式控制：這題的空格與逗號位置非常嚴格，練習了如何在迴圈中精準控制符號的輸出時機
    - 記憶體效率：由於 $n$ 的範圍通常不大，使用 std::queue 雖然比手寫環形陣列稍微多一點開銷，但程式碼的開發速度與可讀性大幅提升
#### 總結 (Takeaway)
- 模擬卡牌/序列變動問題 SOP：
    1. 將物件存入 queue
    2. 執行「彈出（移除）」動作
    3. 若有「移至末尾」需求，先 push(front()) 再 pop()
    4. 重複直到達到終止條件（如 size == 1）
