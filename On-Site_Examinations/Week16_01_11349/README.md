### 題目：[UVa 11349 - Symmetric Matrix](https://onlinejudge.org/external/113/11349.pdf)

#### 解題思路
判斷一個矩陣是否滿足以下兩個條件：
1. 非負性：矩陣中所有的元素都必須大於等於 0
2. 中心對稱性：對於矩陣中任何位置 $(i, j)$ 的元素，必須滿足 matrix[i][j] == matrix[N-1-i][N-1-j]
- 目標：回傳是否為對稱矩陣
- 核心特性：這是一個關於矩陣索引映射的問題。中心對稱點的對應關係是將原本的索引反轉

#### 關鍵邏輯：
1. 資料類型與範圍：
    - 題目輸入值可能非常大，因此必須使用 long long 來儲存矩陣元素
2. 非負性檢查：
    - 在讀取矩陣資料時，同時檢查是否有小於 0 的數值。若有，直接標記為非對稱
3. 對稱性檢查：
    - 遍歷矩陣上半部分或全部，比對 a[i][j] 與其對稱點 a[x+1-i][x+1-j]（此處程式碼採用 1-indexed 寫法）
    - 若兩者數值不相等，則該矩陣不對稱

#### 模擬運算(3x3 矩陣為例)：
- 對稱點關係：
    - (1,1) 對應 (3,3)
    - (1,2) 對應 (3,2)
    - (1,3) 對應 (3,1)
    - (2,2) 對應 (2,2) (中心點)
- 若矩陣為 [[1, 2, 3], [4, 5, 4], [3, 2, 1]]：
    - (1,1)=1, (3,3)=1 (相等)
    - (1,2)=2, (3,2)=2 (相等)
    - (1,3)=3, (3,1)=3 (相等)
    - (2,2)=5 (中心對稱自己，相等)
- 結果：Symmetric
- Time Complexity：$O(N^2)$，需要遍歷矩陣中的每個元素
- Space Complexity：$O(N^2)$，用於儲存矩陣
#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 邊界條件 N=0：題目輸入格式中 N 可能為 0 或負數，雖然本題輸入會給定正確範圍，但在處理矩陣時，初始化大小（如 memset）時務必確保空間足夠
    - 資料型態 (Data Type)：題目提到輸入可能是 long long 範圍的數字，若宣告成 int 會導致數值溢出，導致判斷錯誤
    - 對稱索引計算：對於 1-indexed 的矩陣，對稱點公式為 x+1-i 和 x+1-j，這是必須精確掌握的數學關係
#### 總結 (Takeaway)
- 矩陣對稱檢查 SOP：
    1. 讀取矩陣的同時檢查所有元素是否大於等於 0
    2. 使用雙重迴圈遍歷矩陣
    3. 比較 matrix[i][j] 與 matrix[N+1-i][N+1-j]
    4. 若有任一條件不符，立即標記並中斷檢查
