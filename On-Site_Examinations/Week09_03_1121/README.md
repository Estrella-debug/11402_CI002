### 題目：[UVa 1121 - Subsequence](https://onlinejudge.org/index.php?option=com_onlinejudge&Itemid=8&category=246&page=show_problem&problem=3562)

#### 解題思路
給定一個含有 $N$ 個正整數的序列以及一個正整數 $S$
- 目標：找出一個最短的連續子序列，使得其數字之和 大於或等於 $S$
- 輸出要求：
    1. 輸出該最短子序列的長度
    2. 若找不到任何符合條件的子序列，則輸出 0
- 核心策略：利用「滑動視窗」來動態調整區間範圍，避免暴力法 $O(N^2)$ 的計算，將效率提升至 $O(N)$

#### 關鍵邏輯：
1. Right Pointer：
    - 遍歷數列，將右指針 right 不斷向右移動，並將數值累加到 current_sum
2. Right Pointer：
    - 當 current_sum >= S 時，代表目前的視窗已經符合條件
    - 嘗試最短化：我們開始移動左指針 left 來縮小視窗，直到總和小於 $S$ 為止
    - 在收縮的過程中，不斷更新最小長度 min_len = min(min_len, right - left + 1)
3. 單調性應用：
    - 由於數列中的數字皆為 正整數，總和會隨視窗擴大而增加、隨視窗縮小而減少。這種特性保證了滑動視窗邏輯的正確性
4. 初始化與結果判定：
    - min_len 初始化為 N + 1，用來判定最後是否有找到解答

#### 模擬運算(nums = [2, 3, 1, 2, 4, 3], S = 7為例)：
- 右指針移動到 2, 3, 1, 2：current_sum = 8 (大於 7)
    - 計算長度：4
    - 縮小左側：移除 2，current_sum = 6 (小於 7)
- 右指針移動到 4：current_sum = 6 + 4 = 10
    - 縮小左側：移除 3, 1，視窗變為 [2, 4]，current_sum = 6
    - 期間最短長度更新為 2
- 右指針移動到 3：current_sum = 6 + 3 = 9
    - 縮小左側：移除 2，視窗變為 [4, 3]，current_sum = 7
    - 最短長度保持為 2
- Time Complexity：$O(N)$。雖然有巢狀迴圈，但 left 與 right 指針各自最多只走過整個數列一次
- Space Complexity：$O(N)$。用於儲存輸入的數列

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 視窗優化：學會將 $O(N^2)$ 的區間問題優化為 $O(N)$
    - 型態選擇：對於總和 $S$，雖然題目說是 $10^9$ 以內，但保險起見使用 long long 處理 current_sum 可以避免潛在的溢位風險
    - 邊界處理：當完全沒有解時，min_len 會維持初始值 N+1，這時輸出 0 是題目常見的陷阱
#### 總結 (Takeaway)
- 連續子序列求和問題 SOP：
    1. 設定左右兩個指針
    2. 右指針負責「尋找可行解」
    3. 左指針負責「尋找最優解」
    4. 重複步驟 2 與 3 直到結束
