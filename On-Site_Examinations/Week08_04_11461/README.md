### 題目：[UVa 11461 - Square Numbers](https://onlinejudge.org/index.php?option=com_onlinejudge&Itemid=8&category=26&page=show_problem&problem=2456)

#### 解題思路
給定一個區間 $[a, b]$，計算該區間內有多少個 完全平方數
- 完全平方數定義：一個整數可以表示為另一個整數的平方
- 核心策略：一個整數 $x$ 如果是完全平方數，代表它的平方根 $\sqrt{x}$ 也是一個整數。因此，我們只需要找出範圍內第一個與最後一個平方根整數，兩者相減即可得知總數

#### 關鍵邏輯：
1. 開根號定位：
    - n1 = sqrt(a)：求出下界的平方根
    - n2 = sqrt(b)：求出上界的平方根
2. 修正起始點：
    - 因為 sqrt() 回傳的是整數（無條件捨去），如果 $a$ 本身不是完全平方數（例如 $a=5, \sqrt{5} \approx 2.23 \rightarrow n1=2$），那麼第一個符合條件的平方數應該從 $3^2$ 開始算，所以需要執行 if (n1*n1 != a) n1++
3. 計算總數：
    - 區間內整數平方根的個數公式為：n2 - n1 + 1
4. 效能優化：
    - 使用 ios_base::sync_with_stdio(0); cin.tie(0); 關閉 C++ 與 C 輸出入流的同步，提升大量資料讀取的效率

#### 模擬運算( $a=3, b=17$ 為例)：
- Step 1：$n1 = \sqrt{3} = 1$
- Step 2：檢查 $1 \times 1 = 1 \ne 3$，故 $n1$ 變為 2（代表第一個平方數是 $2^2=4$）
- Step 3：$n2 = \sqrt{17} = 4$
- Step 4：計算 $4 - 2 + 1 = 3$
- 驗證：區間內的平方數為 $\{4, 9, 16\}$，共 3 個。正確
- Time Complexity：$O(1)$。對於每組測資，只需執行兩次 sqrt 運算，效率極高
- Space Complexity：$O(1)$

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 數學法 vs 暴力法：比起從 $a$ 掃描到 $b$ 並逐一判斷是否為平方數（$O(N)$），利用平方根直接計算（$O(1)$）在處理大區間時具有絕對優勢
    - 浮點數轉整數：在 C++ 中將 double 指定給 int 會自動捨去小數點，這與 floor() 效果相同
    - IO 優化：在 UVa 等線上評測系統中，處理大量測資時 sync_with_stdio(0) 能顯著縮短運行時間
#### 總結 (Takeaway)
- 區間平方數問題 SOP：
    1. 對區間的首尾分別取平方根
    2. 判斷首項平方根是否需要補 1（確保起始點在區間內）
    3. 利用「連續整數個數」公式計算結果
