### 題目：[UVa 11157 – Dynamic Frog ](https://onlinejudge.org/external/111/11157.pdf)

#### 解題思路
青蛙要從河的一端跳到另一端再跳回來。河中有兩種石頭：大石頭 (Big, B) 和小石頭 (Small, S)。大石頭可以被多次踩踏，小石頭只能被踩一次
- 目標：最小化青蛙在往返過程中，單次跳躍的最大距離 (Minimax)
- 核心特性：為了使最大距離最小，我們必須謹慎分配小石頭。跳去的時候踩一部分小石頭，回來時踩剩下的小石頭，以此分攤跳躍間距
- 核心策略：貪心法。在處理小石頭時，採用「輪流踩」的方式（第一次跳過去踩這顆，第二次跳回來踩那顆），這樣能讓相鄰跳躍點的距離縮到最小

#### 關鍵邏輯：
1. 分類石頭：
    - 將大石頭 (B) 與小石頭 (S) 分開儲存。大石頭是往返跳躍的「必經錨點」，小石頭則是分擔距離的「輔助點」
2. 貪心分攤距離：
    - 當面對連續多個小石頭時，為了最小化最大跳躍距離，我們讓往返路徑分別各使用其中的一部分石頭
    - 對於大石頭，往返兩次路徑都必須經過它，因此 max(mx, B_pos - last_pos) 會被更新兩次
3. 邊界條件：
    - 河岸起點 (0) 與終點 (D) 對於青蛙而言視同固定錨點，其間距必須納入 mx 計算

#### 模擬運算：
1. 往返路徑分離：想像兩隻青蛙，一隻往東，一隻往西，它們在遇到小石頭時輪流佔用，而大石頭則共享
2. 距離更新：每次跳躍更新 mx = max(mx, current_pos - prev_pos)
3. 最終檢核：最後一段路程（從最後一個石頭到河岸終點 D）也要納入最大跳躍距離的比較
- Time Complexity：$O(N)$，只需遍歷石頭陣列一次
- Space Complexity：$O(N)$，用於儲存石頭的位置

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 貪心的維度思考：這題看起來像路徑搜尋，但實質上是「間距優化」。將「往返」簡化為「兩條不重疊的路徑」是解題的關鍵洞察
    - 資料結構處理：使用 vector 配合 reverse 後的 pop_back 可以很方便地模擬佇列或堆疊操作，達到從左到右遍歷的效果
    - 浮點數與溢位：雖然本題數據規模較小，但在處理距離計算時，務必確保資料型態足夠容納最大距離

#### 總結 (Takeaway)
- 跳躍間距最佳化 SOP：
    1. 將錨點（大石頭、岸邊）與可變點（小石頭）分類
    2. 確保往返路徑都經過所有錨點，並將可變點平均分配給往返路徑
    3. 遍歷整個路徑，持續追蹤並更新單次跳躍的距離最大值
    4. 最終結果即為該路徑中最大間距的極小值
