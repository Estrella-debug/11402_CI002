### 題目：[UVa 11332 - Summing Digits](https://onlinejudge.org/index.php?option=com_onlinejudge&Itemid=8&category=25&page=show_problem&problem=2307)

#### 解題思路
給定一個正整數 $n$，定義一個函數 $f(n)$ 為 $n$ 的各個位數之和
- 目標：重複執行 $f(n)$，直到結果變成一個小於 10 的數字
- 範例：若 $n = 472$，則：
    1. $f(472) = 4 + 7 + 2 = 13$
    2. $f(13) = 1 + 3 = 4$
    3. 最終結果為 4

#### 關鍵邏輯：
1. 數位求和：
    - 使用 while(x) 迴圈配合 % 10 取得最後一位數，並用 /= 10 移除最後一位
    - 這能精準計算出當前數字所有位數的總和
2. 遞迴終止條件：
    - 如果求和結果 ret < 10，代表已經變成個位數，直接回傳
3. 遞迴呼叫：
    - 如果結果仍大於或等於 10，則將 ret 當作新的輸入，再次呼叫 solve(ret)
4. 輸入終止條件：
    - 使用 while (cin >> n && n) 讀取輸入，直到讀到 0 時停止

#### 模擬運算( $n = 999,999,999$ 為例)：
- 第一層 solve(999999999)：
    - 位數和 $ret = 81$|
    - 因為 $81 \ge 10$，進入下一層
- 第二層 solve(81)：
    - 位數和 $ret = 8 + 1 = 9$
    - 因為 $9 < 10$，符合終止條件，回傳 9
- Time Complexity：$O(\log_{10} n)$。數位求和的次數與數字的位數成正比，且每次遞迴都會使數字急遽變小
- Space Complexity：$O(1)$。雖然使用了遞迴，但遞迴深度極淺（例如 $2^{31}-1$ 的位數和最大也才數十），佔用的棧空間微乎其微

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 遞迴的直觀性：這類「重複執行某操作直到滿足條件」的問題，用遞迴實作非常簡潔且符合人類直覺
    - 大數處理：雖然 n 的範圍可能很大，但計算一次位數和後，數字會迅速縮小，這降低了處理難度
    - 數學捷徑 (Fun Fact)：這題其實是在求 $n \pmod 9$。除了 0 以外，數位根的結果與 $n$ 除以 9 的餘數有高度相關（若餘 0 則結果為 9）
#### 總結 (Takeaway)
- 數位處理問題 SOP：
    1. 建立基礎的位數求和邏輯
    2. 判斷目前結果是否符合最終要求
    3. 若不符合，將結果重新導回處理邏輯中
