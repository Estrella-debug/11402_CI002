### 題目：[UVa 10405 - Longest Common Subsequence](https://onlinejudge.org/external/104/10405.pdf)

#### 解題思路
給定兩個字串 $S$ 與 $T$，找出它們的最長共同子序列長度（不必連續，但必須保持相對順序）
- 目標：求出 LCS 長度
- 核心特性：
    1. 若兩個字串結尾字元相同 S[i] == T[j]，則 LCS 長度為 LCS(i-1, j-1) + 1
    2. 若不同，則 LCS 長度為 max(LCS(i-1, j), LCS(i, j-1))，即捨棄其中一個字元後的最優解
- 核心策略：使用二維 DP 表格 dp[i][j] 來記錄前 $i$ 個字元與前 $j$ 個字元的最長共同子序列長度

#### 關鍵邏輯：
1. 狀態定義：
    - dp[i][j] 表示字串 $S$ 的前 $i$ 個字元與字串 $T$ 的前 $j$ 個字元的最長共同子序列長度
2. Transition Formula:
    - if (s[i-1] == t[j-1])：dp[i][j] = dp[i-1][j-1] + 1
    - else：dp[i][j] = max(dp[i-1][j], dp[i][j-1])
3. 邊界條件：
    - 當 $i=0$ 或 $j=0$ 時，dp[i][j] = 0（空字串與任何字串的 LCS 皆為 0）。在 C++ 全域變數宣告中，陣列預設初始化為 0，剛好符合此需求
4. 輸入處理:
    - 使用 getline(cin, s) 確保能讀取含有空白字元的字串，並在處理完每組資料後，需注意若有多組測試資料，DP 表是否需要清空（本題輸入較為簡潔，僅需注意全域變數狀態）

#### 模擬運算(以 ABC 與 AC 為例)：
- Initialization：dp 全部為 0
- 比對：
    - i=1 (A), j=1 (A)：相同，dp[1][1] = dp[0][0] + 1 = 1
    - i=2 (B), j=1 (A)：不同，dp[2][1] = max(dp[1][1], dp[2][0]) = 1
    - i=3 (C), j=2 (C)：相同，dp[3][2] = dp[2][1] + 1 = 2
- 結果：dp[3][2] 為 2
- Time Complexity：$O(N \times M)$，其中 $N$ 和 $M$ 分別為兩個字串的長度
- Space Complexity：$O(N \times M)$，用於儲存 DP 表

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 陣列清空問題：若測試資料多組且共享全域 dp 陣列，務必在每次計算前使用 memset(dp, 0, sizeof(dp))，否則舊數據會影響新計算
    - 空間優化：LCS 問題其實只需要「上一行」的狀態，因此可將空間複雜度從 $O(N \times M)$ 優化至 $O(\min(N, M))$
    - getline 的陷阱：在某些系統下 getline 可能會讀到換行符，需確保讀取行為符合題目要求
#### 總結 (Takeaway)
- LCS 問題 SOP：
    1. 建立 (N+1) x (M+1) 的 DP 二維陣列
    2. 將邊界（第 0 行與第 0 列）設為 0
    3. 兩層迴圈遍歷，應用轉移方程式
    4. 最終答案存於 dp[N][M]
