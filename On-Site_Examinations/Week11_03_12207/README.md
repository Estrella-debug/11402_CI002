### 題目：[UVa 12207 - Media General](https://onlinejudge.org/index.php?option=com_onlinejudge&Itemid=8&page=show_problem&problem=3359)

#### 解題思路
在一個醫院中，病人按編號 $1, 2, \dots, P$ 排隊看診。系統支援兩種指令：

 1. N (Next)：叫下一位病人看診。看診完後，該病人會重新排到隊伍的最末端
 2. E (Expedite)：某位病人有緊急狀況，需要插隊到隊伍的最前面
- 目標：依序輸出所有 N 指令所對應的病人編號
- 核心挑戰：總人數 $P$ 高達 $10^9$，但指令數 $C$ 只有 $1000$。我們不能真的建立一個長度為 $10^9$ 的佇列
- 關鍵策略：觀察發現，只有被「叫到號」或「被插隊」的人會影響順序。因此我們只需要取 $\min(P, C)$ 個人放入 std::deque 即可處理所有邏輯

#### 關鍵邏輯：
1. Small Deque Trick：
    - 由於指令總數 $C$ 最多只有 $1000$，即使 $P$ 很大，排在 $1000$ 名以後的人永遠不會被叫到（除非發生插隊）
    - 因此初始化時只將 $1$ 到 $\min(P, C)$ 的編號推入 deque
2. 叫號處理 (N 指令):
    - 輸出 li.front()
    - 模擬循環：將 front() 移至 back()，並執行 pop_front()
3. 插隊處理 (E 指令)：
    - 讀入插隊者編號 number
    - 第一步：將 number 推至隊首 push_front(number)
    - 第二步：遍歷隊伍剩餘部分，如果發現重複的 number，必須將舊的位置刪除（使用 li.erase(t1)），確保每個人在隊伍中只出現一次
4. 容器選擇：
    - 使用 std::deque 而非 std::queue，是因為我們需要頻繁地在「前端插入」以及「中間刪除（透過迭代器）」

#### 模擬運算($P=10, C=3$, 指令 N, E 5, N為例)：
- 初始狀態：li = [1, 2, 3] (假設 $\min(P, C)=3$)
- 指令 N：
    - 輸出 1
    - 移到末尾：li = [2, 3, 1]
- 指令 E 5：
    - 插入前端：li = [5, 2, 3, 1]
    - 檢查重複：隊伍中無其他 5，不刪除
- 指令 N：
    - 輸出 5
    - 移到末尾：li = [2, 3, 1, 5]
- Time Complexity：每條指令最高為 $O(C)$。總複雜度 $O(C^2)$，在 $C=1000$ 的限制下可輕鬆通過
- Space Complexity：$O(C)$

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 離散化思維：當數據範圍 ($10^9$) 遠大於操作次數時，只處理「感興趣的範圍」是解題關鍵
    - Deque 的靈活性：std::deque 支援 push_front 和 erase 迭代器操作，比 std::queue 更適合處理插隊邏輯
    - 迭代器失效：在使用 erase 之後，舊的迭代器會失效，但在本題中因為 erase 後立刻 break，所以避開了這個陷阱
#### 總結 (Takeaway)
- 超大規模序列調度問題 SOP：
    1. 評估操作次數與數據範圍的關係
    2. 將範圍縮小至 $O(C)$
    3. 選用支援雙端操作的 deque
    4. 插隊時先插在前面，再掃描並移除後方的重複項
