### 題目：[UVa 514 - Rails](https://onlinejudge.org/index.php?option=com_onlinejudge&Itemid=8&page=show_problem&problem=455)

#### 解題思路
有一列火車編號為 $1, 2, \dots, N$，按順序進入一個 A 字型的調度站。調度站是一個「死胡同」，進去的車廂必須後進先出（LIFO）
- 目標：判定給定的出站序列 $c$ 是否可以透過調度站的緩衝達成
- 核心策略：使用一個堆疊（stack<int> cars）模擬調度站。嘗試將編號 $1$ 到 $N$ 的車廂依序推入堆疊，若堆疊頂端的車廂剛好是我們目前需要的出站車廂，就將其彈出
#### 關鍵邏輯：
1. Pushing：
    - 變數 j 代表目前在 A 軌道準備進入調度站的火車編號
    - 當我們需要出站車廂 c 時，如果它還沒在堆疊頂端，我們就持續將 j 增加並推入堆疊
2. Popping：
    - 檢查 cars.top() == c：
        - 如果匹配成功，代表該車廂可以順利出站，將其從堆疊移除
        - 如果推入了所有車廂（$j=N$）後，堆疊頂端依然不是 c，則代表該序列無法達成
   
3. 多層迴圈與終止：
    - 第一層：讀取 $N$（車廂數量）
    - 第二層（marshal）：讀取不同的出站序列建議。若序列第一個數字為 0，則結束該組 $N$ 的測試
4. 結果判定：
    - 如果最後所有車廂都能按要求出站，堆疊應該會是空的（cars.size() == 0），輸出 Yes

#### 模擬運算($N=3$, 期望序列 2 1 3 為例)：
- 讀取 2：
    - 調度站為空，推入 1, 推入 2
    - 堆疊頂端是 2，匹配！彈出 2。此時堆疊：[1]
- 讀取 1：
    - 堆疊頂端是 1，匹配！彈出 1。此時堆疊：[]
- 讀取 3：
    - 堆疊為空，目前進站編號 j=2。推入 3
    - 堆疊頂端是 3，匹配！彈出 3。此時堆疊：[]
- 結果：堆疊全空，輸出 Yes
- Time Complexity：$O(N)$。每節車廂最多進棧一次、出棧一次
- Space Complexity：$O(N)$。需要一個堆疊儲存車廂

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 模擬法的威力：這類物理限制問題，直接用對應的資料結構（堆疊）進行模擬通常是最直觀的解法
    - 輸入格式陷阱：UVa 514 的輸入格式較為繁瑣（多組 $N$，每組 $N$ 內有多組序列，且有不同的 0 作為結束標記），正確使用 scanf 或 cin 並處理好結束標記是過關關鍵
    - 重置狀態：在每一組序列測試前，必須確保堆疊是空的（while(cars.size() > 0) cars.pop();）
#### 總結 (Takeaway)
- 鐵軌調度/堆疊合法性問題 SOP：
    1. 建立一個堆疊作為緩衝
    2. 依序嘗試將原始序列推入堆疊
    3. 每推入一個就檢查堆疊頂部是否與目標序列目前的元素匹配
    4. 若匹配則持續彈出
    5. 最終檢查堆疊是否清空
