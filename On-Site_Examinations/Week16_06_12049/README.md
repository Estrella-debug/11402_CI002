### 題目：[UVa 12049 - Just Prune The List](https://onlinejudge.org/external/120/12049.pdf)

#### 解題思路
給定兩個數字列表，計算最少需要刪除多少個元素，才能使這兩個列表包含完全相同的元素
- 目標：計算刪除次數總和
- 核心特性：這是一個關於「多重集 (Multiset)」差異的問題。若某個數字 $x$ 在序列 A 出現 $c_A$ 次，在序列 B 出現 $c_B$ 次，則為了達成相同，我們必須刪除 $|c_A - c_B|$ 個該數字
- 核心策略：使用 std::map<int, int> 統計兩個序列中每個數字的出現頻率，並計算兩者頻率差值的絕對總和

#### 關鍵邏輯：
1. Frequency Mapping：分別建立兩個 map 來儲存序列 A 與序列 B 中每個數字的出現頻率
2. Calculating Differences：
    - 遍歷第一個 map：
        - 若該數字在第二個 map 中也存在，計算兩者出現次數的絕對差 abs(val1 - val2)，此即為該數字需要刪除的量
    - 遍歷第二個 map：
        - 若該數字只在第二個 map 出現，同樣將其出現次數全刪除

#### 模擬運算(A={1, 2, 2}, B={2, 3}為例)：
1. 統計頻率：
    - count1: {1: 1, 2: 2}
    - count2: {2: 1, 3: 1}
2. 差異計算：
    - 檢查數字 1：只在 count1 出現，需刪除 1 次。deletions = 1
    - 檢查數字 2：兩者皆出現，刪除 abs(2 - 1) = 1 次。deletions = 1 + 1 = 2
    - 檢查數字 3：只在 count2 出現，需刪除 1 次。deletions = 2 + 1 = 3
- 結果：總計刪除 3 次
- Time Complexity：$O((N+M) \log (\min(N, M)))$，其中 $N, M$ 為兩序列長度
- Space Complexity：$O(N+M)$，用於儲存兩個 map

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 絕對值的重要性：題目要求刪除元素直到兩序列相等，這暗示了「多出來的」和「缺失的」都需要被處理，使用 abs() 處理頻率差是最高效的手段
    - map 的遍歷：在 C++17 中，可以使用結構化綁定 for (auto const& [key, val] : count) 來簡化存取 map 的代碼，提升可讀性
    - 效能考量：若數據範圍非常大，std::map 的 $O(\log N)$ 查找可能會稍慢。如果允許，使用 std::unordered_map 可以將複雜度優化至平均 $O(N+M)$

#### 總結 (Takeaway)
- 多重集差異計算 SOP：
    1. 使用兩個 map 分別統計兩個序列的元素頻率
    2. 遍歷其中一個 map，處理共同元素（計算頻率差）及孤立元素（全數刪除）
    3. 再次遍歷另一個 map，處理剩餘的孤立元素
    4. 將所有刪除量累加即為答案
