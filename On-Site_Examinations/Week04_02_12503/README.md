### 題目：[UVa 12503 - Robot Instructions](https://onlinejudge.org/index.php?option=com_onlinejudge&Itemid=8&category=24&page=show_problem&problem=3947)

#### 解題思路
這題的核心在於模擬一個機器人在一維座標軸上的移動。機器人會收到三種指令：向左、向右、或是「重複之前的某個指令」。為了正確執行「重複」指令，我們必須把每一步真正的移動方向記錄下來。

#### 關鍵邏輯：
1. 指令分類：
    - LEFT：位置 $-1$。
    - RIGHT：位置 $+1$。
    - SAME AS i：這不是新動作，而是去查第 $i$ 步做過什麼，然後照著做。
2. 歷史紀錄 (History)：使用一個 vector<int> 來儲存每一動發生的位移量。這樣當遇到 SAME AS 時，可以快速找到過去的資料。
3. 座標更新：每讀取並解析完一個指令，就將該步的位移量累加到總位置 pos。

#### 模擬運算：
- LEFT：history[0] = -1, pos = -1
- RIGHT：history[1] = 1, pos = 0
- SAME AS 1：查閱 history[0] 為 $-1$，所以 history[2] = -1, pos = -1
- 最後輸出 pos = -1。
- 時間複雜度：$O(n)$，每個測資有 $n$ 個指令，每個指令處理速度極快。
- 空間複雜度：$O(n)$，我們需要一個陣列或向量來存儲這 $n$ 步的紀錄。

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 索引對應 (1-based vs 0-based)：題目給的指令是 SAME AS 1（從 1 開始），但在程式碼陣列中是 history[0]，所以存取時要記得 index - 1。
    - 字串處理技巧：利用 cin >> s 可以輕鬆切分空格。當讀到 SAME 時，再額外用 cin 讀掉無意義的單字 AS 並取得目標數字。
    - 動態記憶體：使用 vector 可以根據每筆測資的指令數 $n$ 來彈性分配空間。

#### 總結 (Takeaway)
- 指令模擬問題 SOP：
    1. 建立一個容器（如 vector）記錄所有發生過的動作。
    2. 妥善處理字串輸入，區分固定指令與參數指令。
    3. 注意題目給的編號與程式碼索引之間的落差（通常差 1。
- 這種「記錄歷史」的做法是解決所有「重複動作」類題目的標準解法。
