### 題目：[UVa 11364 - Parking](https://onlinejudge.org/index.php?option=com_onlinejudge&Itemid=8&category=25&page=show_problem&problem=2349)

#### 解題思路
Michael 在一條直線上有多個商店要逛。他想停好車後，走路去所有的商店再回到車位
- 目標：找到一個停車位置，使得總走路距離最小化
- 邏輯轉化：
    - 假設他把車停在最左邊的商店與最右邊的商店之間
    - 他必須從車位走到最左邊，再走到最右邊（或反之），最後回到車位
    - 無論車停在哪裡，只要在最左與最右商店的範圍內，他都必須完整走過這段距離 兩次
    - 因此，最小距離就是 (最大座標 - 最小座標) × 2

#### 關鍵邏輯：
1. Min/Max Tracking：
    - 初始化 mn = 100（根據題目範圍 0-99）與 mx = 0
    - 在讀入每個商店座標 x 時，不斷更新目前看到的最大值與最小值
2. 公式化簡：
    - 不需要使用二心搜尋或複雜的優化，直接套用幾何結論：distance = (mx - mn) * 2
3. 多組測資處理：
    - 先讀取測試組數 T，接著用迴圈處理每一組資料

#### 模擬運算(座標 [10, 20, 50, 30]為例)：
- 讀取數據：
    - x = 10: mn = 10, mx = 10
    - x = 20: mn = 10, mx = 20
    - x = 50: mn = 10, mx = 50
    - x = 30: mn = 10, mx = 50
- 計算：(50 - 10) * 2 = 80
- 結果：輸出 80
- Time Complexity：$O(n)$。每組測資只需掃描一次所有座標
- Space Complexity：$O(1)$。只需儲存極值，不需要存下所有座標

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 看穿問題本質：題目描述 Michael 想找「最優停車位」，但數學上只要停車位在最寬範圍內，結果都一樣。這種「腦筋急轉彎」式的簡化能大幅減少程式複雜度
    - 邊值初始化：學會根據題目給定的範圍（0-99）來設定 mn 和 mx 的初始值
    - 效率意識：這類線性掃描的問題，不需要排序（$O(n \log n)$），直接找極值（$O(n)$）才是最快的方法
#### 總結 (Takeaway)
- 直線區間問題 SOP：
    1. 找出集合中的最大值與最小值
    2. 確認移動路徑是否為往返（通常是距離的兩倍）
    3. 利用一次迴圈同步更新多個極值
