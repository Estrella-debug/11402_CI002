### 題目：[UVa 11559 - Event Planning](https://onlinejudge.org/index.php?option=com_onlinejudge&Itemid=8&category=27&page=show_problem&problem=2595)

#### 解題思路
有一群朋友（$N$ 人）計畫要去旅行，預算上限為 $B$。現在有 $H$ 間旅館，每間旅館提供 $W$ 個星期的住宿資訊
- 目標：在符合預算且旅館床位足夠（至少 $N$ 個空位）的前提下，找出最低的旅行總花費
- 輸出規則：
    1. 樹木名稱須按 字典順序 (Alphabetical order) 排列
    2. 百分比須精確到小數點後 四位

#### 關鍵邏輯：
1. 極值初始化：
    - min_cost 初始化為 B + 1（即超過預算的數值），這是一個常見的技巧，用來確保任何符合條件的合法價格都能在第一次更新時被錄入
2. 兩層迴圈篩選：
    - 外層迴圈：遍歷每一間旅館，讀取該旅館的每人單價 price
    - 內層迴圈：遍歷該旅館的每一個星期，檢查床位數 beds 是否大於等於人數 $N$
3. 合法性驗證：
    - 只要有一個星期滿足人數需求，該旅館即標記為 possible = true
    - 計算總費用：total_cost = N * price
    - 判斷總費用是否在預算 $B$ 之內，若通過則使用 min(min_cost, total_cost) 更新最小成本
4. 結果判定：
    - 最終檢查 min_cost 是否仍大於預算。若是，代表無解

#### 模擬運算(3 人、預算 1000、旅館單價 200為例)：
- 旅館 A (200元/人)：
    - 第一週剩 2 床 (不足)
    - 第二週剩 5 床 (足夠！)
    - 總費用：$3 \times 200 = 600$
    - 更新 min_cost = 600 (因為 $600 \le 1000$)
- 結果：輸出 600
- Time Complexity：$O(H \times W)$。需要掃描所有旅館的所有週次
- Space Complexity：$O(1)$。不需要額外空間儲存所有數據，邊讀邊處理即可

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 即時處理數據：這題不需要將所有的床位資訊存入 vector 或 array，讀取後立即判斷，能節省記憶體並簡化邏輯
    - 多測資處理：熟練使用了 while (cin >> N >> B >> H >> W) 來處理 UVa 常見的「直到檔案結尾 (EOF)」輸入格式
    - 邏輯順序：先檢查床位，再算總價，最後比預算。清晰的階層式過濾能避免不必要的運算
#### 總結 (Takeaway)
- 模擬與篩選問題 SOP：
    1. 設定一個初始極大值
    2. 透過迴圈逐項檢查所有候選方案
    3. 建立「篩選門檻」（本題為床位數與總預算）
    4. 在滿足所有門檻後，更新最優解
