### 題目：[ UVa 10359 - Tiling](https://onlinejudge.org/external/103/10359.pdf)

#### 解題思路
我們需要鋪設寬度為 $N$、高度為 2 的矩形區域。可使用的磚塊有：
- $2 \times 1$ 的直磚
- $2 \times 2$ 的方磚
- $1 \times 2$ 的橫磚（兩塊橫磚堆疊）
- 目標：計算鋪設該區域的方法總數
- 核心特性：當我們要鋪設長度為 $N$ 的區域時，最後一塊區域的鋪設方式：
    1. 用一塊 $2 \times 1$ 直磚填滿最後一列，剩餘 $N-1$ 區域
    2. 用一塊 $2 \times 2$ 方磚填滿最後兩列，剩餘 $N-2$ 區域
    3. 用兩塊 $1 \times 2$ 橫磚填滿最後兩列，剩餘 $N-2$ 區域
- 核心策略：因此，狀態轉移方程為 $f(N) = f(N-1) + 2 \times f(N-2)$

#### 關鍵邏輯：
1. 遞迴關係推導：
    - $f(N) = f(N-1) + 2f(N-2)$
    - 基礎情況：$f(0) = 1, f(1) = 1$
2. 大數處理：
    - 由於 $N$ 可達 250，結果將會遠遠超過 long long 的儲存範圍（約 $10^{75}$ 以上），因此必須實作字串大數加法來模擬運算
3. 預計算：
    - 建立一張 DP 表，在程式啟動時先將 0 到 250 的結果全部算出，以符合競賽頻繁查詢的需求

#### 模擬運算 (以 $N=3$ 為例)：
1. $f(0) = 1$
2. $f(1) = 1$
3. $f(2) = f(1) + 2 \times f(0) = 1 + 2(1) = 3$
4. $f(3) = f(2) + 2 \times f(1) = 3 + 2(1) = 5$
- 結果：$N=3$ 時有 5 種鋪法
- Time Complexity：$O(N^2)$，主要取決於 $N$ 次大數加法，每次加法複雜度與位數成正比
- Space Complexity：$O(N^2)$，需儲存 $N$ 個大數

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 大數加法實作：這是處理 $N > 100$ 的經典計數問題必備技能。核心在於利用 string 倒序運算並處理進位 (Carry)
    - 遞迴定義的敏感度：這題容易與常見的費氏數列 (Fibonacci) 混淆。多了「2倍」的係數，是因為 $2 \times 2$ 方磚與兩塊 $1 \times 2$ 橫磚視為兩種不同的鋪法
    - 記憶體配置：使用 vector<string> 預先建表，避免每次查詢重複計算，是兼顧效能與寫法的優良選擇

#### 總結 (Takeaway)
- 大數遞迴鋪磚 SOP：
    1. 推導出正確的遞迴關係（注意係數）
    2. 實作一個基礎的 stringAdd 函式處理進位
    3. 執行預建表 (Pre-computation) 將 250 以內的答案全部存入陣列
    4. 讀取輸入直接進行查表輸出
