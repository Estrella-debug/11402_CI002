### 題目：[UVa 1062 - Containers](https://onlinejudge.org/index.php?option=com_onlinejudge&Itemid=8&page=show_problem&problem=3503)

#### 解題思路
一艘貨輪載著標有 $A \sim Z$ 的貨櫃依序抵達。貨櫃必須暫存在碼頭的堆棧中，且只有最上方的貨櫃可以隨時被取走
- 限制條件：為了方便後續按字母順序取貨，你不能將較大的字母（如 B）壓在較小的字母（如 A）上面，否則小的貨櫃會被卡住
- 目標：計算最少需要開闢多少個堆棧（Stacks），才能容納所有抵達的貨櫃
- 核心策略：使用 貪心法 (Greedy)。每來一個貨櫃，優先把它放在「目前頂端字母大於或等於它、且與它最接近」的堆棧上。如果所有堆棧頂端的字母都比它小，則必須開闢一個新堆棧

#### 關鍵邏輯：
1. 動態堆棧管理：
    - 使用 vector<stack<char>> stacks 來動態儲存多個堆疊
2. Best Fit：
    - 對於每一個進來的貨櫃 c，遍歷所有現有的堆疊
    - 條件：stacks[i].top() >= c
    - 優化：在所有符合條件的堆疊中，選擇頂端字母最小的那一個（即最接近 c 的），這樣可以保留其他頂端較大的堆疊給未來更後面的字母使用
3. 決策分支：
    - 放入舊堆疊：若找到 bestStack，執行 push(c) 更新該堆疊頂端
    - 建立新堆疊：若完全找不到可放置的位置，則 push_back 一個新的堆疊
4. 結果判定：
    - 處理完字串後，stacks.size() 即為所需的最少堆棧數

#### 模擬運算(ACBA為例)：
- 讀到 A：建立堆疊 1，頂端為 A。 stacks: [[A]]
- 讀到 C：C > A，無法放入。建立堆疊 2。 stacks: [[A], [C]]
- 讀到 B：
    - 堆疊 1 頂端 A < B (不行)
    - 堆疊 2 頂端 C > B (可以！)
    - 更新堆疊 2 頂端為 B。 stacks: [[A], [B]]
- 讀到 A：
    - 堆疊 1 頂端 A >= A (可以！)
    - 堆疊 2 頂端 B >= A (也可以！)
    - 貪心選擇最接近的堆疊 1。 stacks: [[A], [B]]
- 結果：輸出 2
- Time Complexity：$O(N \times S)$，其中 $N$ 是字串長度，$S$ 是最終堆疊的數量。由於 $S \le 26$（字母種類），效能非常優異
- Space Complexity：$O(N)$，用於儲存所有的貨櫃字元

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - Greedy 的正確性：這題與「最長遞增子序列 (LIS)」的 Patience Sorting 演算法非常相似，理解了為什麼「放回最接近的堆疊」能達到全域最優
    - 字元比較：直接利用 ASCII 碼進行 char 的大小比較，直觀且高效
    - 輸入終止：處理 while 迴圈中的 end 字串作為結束標記
#### 總結 (Takeaway)
- 貨櫃堆放/貪心問題 SOP：
    1. 準備一個容器存放多個堆疊
    2. 循序處理輸入元素
    3. 嘗試在現有堆疊中找尋「最小且大於等於當前元素」的頂端
    4. 找得到就更新，找不到就新增
    5. 堆疊總數即為解答
