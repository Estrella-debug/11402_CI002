### 題目：[62. Unique Paths](https://leetcode.com/problems/unique-paths/description/)

#### 解題思路
機器人位於一個 $m \times n$ 網格的左上角，目標是移動到右下角。機器人每次只能 向下 或 向右 移動一步
- 目標：計算到達右下角共有多少條唯一的路徑
- 核心特性：
    1. 到達某個格子 $(i, j)$ 的方法，只能是從其 上方格子 $(i-1, j)$ 或 左方格子 $(i, j-1)$ 走過來的
    2. 因此，到達 $(i, j)$ 的總路徑數 = 到達上方的路徑數 + 到達左方的路徑數
- 核心策略：使用動態規劃。雖然直觀上會想用二維陣列 $dp[m][n]$，但由於更新當前格子只需參考「左方」與「上方」的數據，我們可以進一步壓縮空間至一維陣列
#### 關鍵邏輯：
1. 狀態定義與初始化：
    - vector<int> dp(n, 1)：建立一個長度為列數 $n$ 的一維陣列，全部初始化為 1
    - 這代表第一行（Row 0）的所有格子都只有一種到達方式（一路向右走）
2. Transition Formula:
    - 當對於每一行（從第 1 行到第 $m-1$ 行）
    - 這裡的奧妙：更新前的 dp[j] 儲存的是上一行的舊資料（即「上方」），而 dp[j-1] 則是剛剛才更新過的當前行資料（即「左方」）
3. 邊界處理：
    - 第 0 列的路徑數永遠是 1（一路向下走），因此內層迴圈從 j = 1 開始，自動保留 dp[0] 為 1

#### 模擬運算( $3 \times 3$ 網格為例)：
- 初始化 (Row 0)：dp = [1, 1, 1]
- 處理 Row 1：
    - $j=1$: dp[1] = dp[1](上) + dp[0](左) = 1 + 1 = 2
    - $j=2$: dp[2] = dp[2](上) + dp[1](左) = 1 + 2 = 3
    - dp 更新為 [1, 2, 3]
- 處理 Row 2：
    - $j=1$: dp[1] = 2(上) + 1(左) = 3
    - $j=2$: dp[2] = 3(上) + 3(左) = 6
    - dp 更新為 [1, 3, 6]
- 結果：輸出 6
- Time Complexity：$O(m \times n)$
- Space Complexity：$O(n)$。僅需儲存一列的大小，大幅優於二維陣列的 $O(m \times n)$

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 空間壓縮技巧 (Space Optimization)：這是 DP 進階技巧中非常實用的一環，理解了如何利用更新順序（由左至右、由上至下）來重複利用記憶體
    - 排列組合的連結：這題在本質上也可以用數學組合公式 $C^{(m+n-2)}_{(m-1)}$ 來解，但在處理大數時，DP 遞迴加法能有效避免乘法造成的整數溢出問題
    - 對稱性：網格 $3 \times 7$ 與 $7 \times 3$ 的結果完全相同，這驗證了路徑選擇在長寬交換下的對稱性
#### 總結 (Takeaway)
- 網格型 DP 問題 SOP：
    1. 確認移動方向限制（只能下與右）
    2. 寫出轉移方程（當前 = 上 + 左）
    3. 判斷是否可以進行空間優化（一維陣列替代二維）
    4. 小心處理邊界（第一行與第一列的初始化）
