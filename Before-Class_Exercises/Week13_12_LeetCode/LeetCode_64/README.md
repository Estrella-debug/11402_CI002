### 題目：[64. Minimum Path Sum](https://leetcode.com/problems/minimum-path-sum/description/)

#### 解題思路
給定一個包含非負整數的 $m \times n$ 網格 grid，請找出一條從左上角到右下角的路徑，使得路徑上的數字總和為最小
- 規則限制：每次只能 向下 或 向右 移動一步
- 核心特性：
    1. 到達某個格子 $(i, j)$ 的代價，取決於該格子的值加上「從上方走過來」或「從左方走過來」的最小代價
    2. 這具有 最佳子結構 (Optimal Substructure)：整條路徑的最優解包含其子路徑的最優解
- 核心策略：原地（In-place）更新動態規劃。為了節省空間，我們直接在原始的 grid 陣列上更新數值，將其轉換為到達該點的累計最小路徑和

#### 關鍵邏輯：
1. 狀態定義：
    - grid[i][j] 更新後代表從 $(0,0)$ 到 $(i,j)$ 的最小累計路徑和
2. Transition Formula:
    - 一般情況（有左也有上）
    - 特殊邊界
3. 原地更新：
    - 由於更新 grid[i][j] 只需要用到左邊或上邊的值，我們可以不使用額外空間，直接修改傳入的二維陣列

#### 模擬運算：
- Time Complexity：$O(m \times n)$。需遍歷網格中所有元素
- Space Complexity：$O(1)$。不考慮輸入陣列本身佔用的空間，因為我們是原地修改

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 空間利用：原地修改（In-place update）是節省記憶體的極致手段。若不允許修改輸入，則可建立一個一維或二維陣列來存儲 DP 狀態
    - 邊界優先：先考慮第一行和第一列，能讓中間的轉移邏輯更清晰，避免大量的 `if-else`
    - 非負整數的意義：題目強調非負整數，這保證了累積路徑和一定是單調不減的，簡化了最優化問題
#### 總結 (Takeaway)
- 網格最優化路徑問題 SOP：
    1. 確認移動方向限制（下與右）
    2. 確定起點與終點
    3. 建立轉移方程（當前值 + min(上方值, 左方值)）
    4. 依序填寫 DP 表
