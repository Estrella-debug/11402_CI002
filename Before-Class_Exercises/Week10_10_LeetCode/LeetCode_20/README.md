### 題目：[20. Valid Parentheses](https://leetcode.com/problems/valid-parentheses/description/)

#### 解題思路
給定一個僅包含 (、)、{、}、[、] 的字串，判斷該字串是否有效
- 有效條件：
    1. 左括號必須用相同類型的右括號閉合
    2. 左括號必須以正確的順序閉合
    3. 每個右括號都有一個對應的相同類型的左括號
- 核心策略：遍歷字串，遇到左括號就先「存起來」，遇到右括號就找「最近存入的左括號」來配對。這正好符合堆疊的運作邏輯

#### 關鍵邏輯：
1. Pushing：
    - 當遇到開口括號 (、{、[ 時，將其推入堆疊 st
2. Matching & Popping：
    - 當遇到閉合括號時：
        - 檢查空堆疊：如果堆疊已空卻還有右括號，代表沒有左括號可配對，回傳 false
        - 檢查類型：取出堆疊最頂端的元素（即最近進入的左括號），檢查是否與當前右括號類型一致。若不一致，回傳 false
        - 移除配對：若匹配成功，執行 st.pop() 移除該左括號
3. Final Check：
    - 遍歷結束後，檢查堆疊是否為空。若仍有剩餘元素（如只有左括號），代表未完全閉合，回傳 false

#### 模擬運算(s = "([{}])"為例)：
- 讀到 (：堆疊 [(]
- 讀到 [：堆疊 [(, []
- 讀到 {：堆疊 [(, [, {]
- 讀到 }：與頂端 { 匹配。彈出後堆疊 [(, []
- 讀到 ]：與頂端 [ 匹配。彈出後堆疊 [(]
- 讀到 )：與頂端 ( 匹配。彈出後堆疊 []
- 結果：堆疊為空，回傳 True
- Time Complexity：$O(n)$。只需遍歷字串一次，堆疊操作均為 $O(1)$
- Space Complexity：$O(n)$。最差情況下（全為左括號）需儲存所有字元

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - LIFO 的應用：理解了為什麼「對稱結構」或「嵌套結構」首選堆疊。最近出現的左括號必須最先被閉合
    - 邊界處理：學會考慮「只有右括號」與「只有左括號」這兩種極端失敗情況
    - 程式碼可讀性：雖然本程式碼使用多個 if 判斷，但在處理更多括號類型時，也可以考慮使用 map 或 switch 來增加擴充性

#### 總結 (Takeaway)
- 括號/符號匹配問題 SOP：
    1. 遇到「開啟類」符號：入棧
    2. 遇到「閉合類」符號：
        - 檢查棧是否為空
        - 檢查棧頂是否匹配
        - 彈出棧頂
    3. 結束時檢查棧是否清空
