### 題目：[179. Largest Number](https://leetcode.com/problems/largest-number/)

#### 解題思路
給定一組非負整數 nums，重新排列它們，使它們組合成一個「最大」的數字。
- 範例：輸入 [10, 2]，可以組成 102 或 210，其中 210 最大。
- 注意：因為結果可能非常大，所以必須以 字串 (string) 的形式回傳。

#### 關鍵邏輯：
1. 數字轉字串：為了方便組合與比較，我們先把 vector<int> 轉換成 vector<string>。
2. 特殊情況處理 (Edge Case)：如果排序後第一個字元是 "0"（例如 [0, 0]），代表整組數字都是 0，我們直接回傳 "0"，而不是 "00"。

#### 模擬運算：
- 第一步：全部轉成字串 ["3", "30", "34", "5", "9"]。
- 第二步：執行自定義排序。
- 第三步：依序串接字串，得到 "9534330"。
- 最後結果：持續進行直到所有數字定位。
- Time Complexity (時間複雜度)：$O(n \log n \cdot k)$。其中 $n$ 是數字個數，$k$ 是字串平均長度（排序需要 $n \log n$ 次比較，每次字串接合與比較需要 $O(k)$）。
- Space Complexity (空間複雜度)：$O(n \cdot k)$。需要額外空間儲存字串陣列。

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - Lambda 運算式：在 sort 函數中使用 [] 語法直接定義比較邏輯，可以讓程式碼更簡潔。
    - 字串接合的威力：透過 a + b 與 b + a 的比較，完美解決了不同位數數字誰該放前面的邏輯難題。
    - 全為 0 的處理：這是最容易漏掉的測試案例（Case），一定要檢查結果是否有多餘的領先零。

#### 總結 (Takeaway)
- 組合最大數問題 SOP：
    1. 將所有物件轉為字串。
    2. 定義比較邏輯：測試兩個物件「互相接合」後的結果。
    3. 串接排序後的結果，並處理特殊開頭（如領先零）。
- 這種自定義排序的技巧在處理「優先權」或「最優組合」問題時非常強大。
