### 題目：[LeetCode: 56. - Merge Intervals](https://leetcode.com/problems/merge-intervals/description/)

#### 解題思路
給定一個區間陣列 intervals，將所有重疊的區間合併，並回傳一個包含所有互不重疊區間的陣列
- 目標：合併所有重疊區間
- 核心特性：當區間按照「左邊界」排序後，重疊的情況僅會發生在相鄰的區間之間。這讓我們只需維護當前正在合併的區間，並依序檢查下一個區間是否重疊

#### 關鍵邏輯：
1. Sorting：
    - 使用 sort(intervals.begin(), intervals.end())。C++ 的 sort 對 vector<vector<int>> 的排序規則預設為：先比較第一個元素（左邊界），若相同則比較第二個元素。這確保了我們總是按區間起始位置依序處理
2. Merging Condition:
    - 假設當前合併中的區間為 [l, r]，下一個區間為 [i[0], i[1]]：
        - 若 i[0] > r：兩者不重疊。我們將 [l, r] 存入答案，並更新 l 與 r 為新的區間
        - 若 i[0] <= r：兩者重疊。更新當前合併區間的右邊界為 max(r, i[1])

#### 模擬運算([[1,3],[2,6],[8,10]]為例)：
- 排序後：[[1,3],[2,6],[8,10]]
- 起始狀態：l=1, r=3
- 遍歷 [2,6]：
    - 2 <= 3 (重疊)，更新 r = max(3, 6) = 6。當前區間變為 [1,6]
- 遍歷 [8,10]：
    - 8 > 6 (不重疊)，將 [1,6] 存入答案。更新 l=8, r=10
- 收尾：將最後留在手上的 [8,10] 存入答案
- Time Complexity：$O(N \log N)$，主要耗時在於排序
- Space Complexity：$O(N)$（視排序演算法空間需求而定），用於儲存答案

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 排序的重要性：沒有排序，我們必須進行 $O(N^2)$ 的暴力比對。排序後，區間問題通常能降至 $O(N \log N)$
    - 貪心策略：我們不需要窮舉所有可能的合併組合，只要依序處理相鄰區間並更新邊界，即為最佳解
    - 邊界值維護：在迴圈結束後，別忘了處理最後剩下的區間，這是區間問題最容易漏掉的「最後一哩路」
#### 總結 (Takeaway)
- 區間重疊處理 SOP：
    1. 依據左邊界進行排序
    2. 利用兩個變數 l, r 追蹤當前合併範圍
    3. 線性掃描：不重疊則切分、重疊則更新右邊界
    4. 迴圈結束後記得加入最後一項
