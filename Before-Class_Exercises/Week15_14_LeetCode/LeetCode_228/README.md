### 題目：[LeetCode: 228. - Summary Ranges](https://leetcode.com/problems/summary-ranges/description/)

#### 解題思路
給定一個已排序且不重複的整數陣列 nums，找出涵蓋陣列中所有數字的最小排序範圍列表
- 目標：將連續的數字合併為 "a->b" 的形式，若數字不連續則單獨顯示為 "a"
- 核心特性：由於輸入已經排序，我們只需要判斷 nums[i] + 1 == nums[i+1] 是否成立，即可知道數字是否連續
- 核心策略：使用雙指針或單次迴圈掃描，當發現斷層時，將當前區間記錄下來

#### 關鍵邏輯：
1. 區間起始點記錄：
    - 在迴圈中，用 start 變數記錄當前連續區間的開頭
2. 連續偵測:
    - 使用 while 迴圈向前探測，若下一個數字剛好等於 當前值 + 1，則將指標 i 向後推進
3. 區間格式化：
    - 當 start 與 nums[i] 相等時，代表該區間只有一個數字；反之則為 "start->end" 格式
4. 邊界保護：
    - i + 1 < n 的條件確保了我們不會在處理最後一個元素時存取到非法記憶體

#### 模擬運算(nums = [0, 1, 2, 4, 5, 7]為例)：
- 開始：i=0, start=0
- 連續偵測：0+1==1 (True), 1+1==2 (True), 2+1!=4 (False)。指標 i 停在 2
    - start(0) != nums[2](2)，存入 "0->2"
- 下一區間：i=3, start=4
    - 4+1==5 (True), 5+1!=7 (False)。指標 i 停在 4
    - start(4) != nums[4](5)，存入 "4->5"
- 最後一項：i=5, start=7
    - 無下一個元素，start(7) == nums[5](7)，存入 "7"
- 結果：["0->2", "4->5", "7"]
- Time Complexity：$O(N)$，只需遍歷陣列一次
- Space Complexity：$O(1)$（不計輸出空間）

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 雙指針思維：雖然是用單個 for 迴圈，但內部的 while 本質上就是一個移動的指針，這種方式比維護兩個變數更為精簡
    - 類型轉換：在 C++ 中，善用 to_string() 將數字轉為字串進行組合，是處理 LeetCode 字串回傳格式的必備技巧
    - 溢出隱憂：雖然本題範圍在 int 內，但在 nums[i] + 1 計算時，若 nums[i] 為 INT_MAX 可能會導致溢出。在更嚴謹的場景下應使用 long long 進行比較
#### 總結 (Takeaway)
- 線性序列分組 SOP：
    1. 標記區間起始點 start
    2. 利用迴圈向後掃描直到條件中斷
    3. 根據 start 與結束指標的差值決定輸出格式
    4. 更新迴圈指標，繼續下一個區間的處理
