### 題目：[LeetCode: 455. Assign Cookies](https://leetcode.com/problems/assign-cookies/description/)

#### 解題思路
假設你是一位很棒的家長，想給孩子們一些餅乾。每個孩子 $i$ 都有一個胃口值 $g_i$，也就是孩子對餅乾大小的最小要求。每塊餅乾 $j$ 都有一個大小 $s_j$。如果 $s_j \ge g_i$，我們就可以將這塊餅乾分給這個孩子
- 目標：最大化被滿足的孩子數量
- 核心特性：為了不浪費大餅乾，同時又能滿足胃口最小的孩子，我們應該採用「精準匹配」的策略
- 核心策略：貪心法。先將餅乾和孩子的胃口都由小到大排序，接著用最小的餅乾去滿足胃口最小的孩子，這樣能保留較大的餅乾給後面胃口較大的孩子

#### 關鍵邏輯：
1. Sorting：
    - 對 g (孩子胃口) 與 s (餅乾大小) 進行升序排序
2. Two-Pointer Matching：
    - child_ptr 指向當前胃口最小的孩子
    - cookie_ptr 指向當前最小的餅乾
    - 每次比較：若餅乾足以滿足孩子，則 child_ptr 後移（該孩子已滿足）；無論是否滿足，cookie_ptr 皆向後移動（代表該餅乾已被使用或被視為太小而廢棄）

#### 模擬運算(g = [1, 2], s = [1, 2, 3]為例)：
1. 排序後：g = [1, 2], s = [1, 2, 3]
2. 匹配開始：
    - child_ptr=0 (胃口1), cookie_ptr=0 (餅乾1)：1 >= 1，滿足！child_ptr=1, cookie_ptr=1
    - child_ptr=1 (胃口2), cookie_ptr=1 (餅乾2)：2 >= 2，滿足！child_ptr=2, cookie_ptr=2
3. 結果：child_ptr 為 2，表示滿足了 2 個孩子
- Time Complexity：$O(N \log N + M \log M)$，主要耗時在排序
- Space Complexity：$O(1)$ 或 $O(N+M)$，取決於排序演算法對空間的需求

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 貪心的決策：在資源配置問題中，從「最小」開始分配通常是確保效率的最佳手段。這確保了我們不會把大資源浪費在低需求者身上
    - 雙指標的簡潔性：這段程式碼最優美的地方在於 cookie_ptr 不管結果如何都會遞增，這省去了額外的 if 判斷分支，邏輯非常直觀
    - 不要過度優化：初學者常會想用複雜的資料結構（如優先佇列）來解，但其實只要簡單的排序加雙指標就能達成 $O(N \log N)$，這類題目教導我們「最簡單的方法往往最有效」
#### 總結 (Takeaway)
- 資源與需求匹配 SOP：
    1. 將需求與資源分別由小到大排序
    2. 使用雙指標依序比對
    3. 若資源足以滿足需求，則指標同時移動
    4. 若資源不足以滿足當前需求，只移動資源指標，尋找更大的資源
