### 題目：[LeetCode: 55. Jump Game](https://leetcode.com/problems/jump-game/description//)

#### 解題思路
給定一個非負整數陣列 nums，每個數字代表你在該位置可以向前跳躍的最大長度。判斷是否能從第一個位置到達最後一個位置
- 目標：回傳布林值 (true/false)
- 核心特性：我們不需要知道「怎麼跳」才能到達終點，只需要確認「有沒有能力」到達終點。這代表我們只需要關注在每一個位置，所能延伸的最遠距離為何
- 核心策略：貪心法。在掃描過程中，隨時更新並維護一個 max_reach（最遠可達索引），只要當前遍歷的索引 i 超過了這個 max_reach，就代表前方已經無路可走

#### 關鍵邏輯：
1. 最大可達距離維護：
    - 對於每個位置 i，其所能觸及的最遠點為 i + nums[i]
    - 我們不斷更新 max_reach = max(max_reach, i + nums[i])
2. Stuck Condition：
    - 如果當前位置 i > max_reach，說明我們連當前的位置都無法到達，因此失敗
3. 提早結束優化：
    - 若 max_reach >= n - 1，表示終點已經在我們的覆蓋範圍內，可以直接回傳 true，無需繼續遍歷

#### 模擬運算(nums = [2, 3, 1, 1, 4]為例)：
1. 初始：max_reach = 0
2. i=0：max_reach = max(0, 0+2) = 2
3. i=1：max_reach = max(2, 1+3) = 4
    - 此時 max_reach(4) >= n-1(4)，達成條件，提早回傳 true
- 結果：需要移除 1 個區間
- Time Complexity：$O(N \log N)$，主要耗時在排序
- Space Complexity：$O(1)$，僅需常數級別的額外空間

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 貪心 vs. DP：這題原本可以用動態規劃 (DP) 解，但會達到 $O(N^2)$ 或 $O(N)$ 但空間較高的複雜度。貪心法的 $O(N)$ 時間與 $O(1)$ 空間是處理此類問題的最優解
    - 不要陷入細節：初學者容易糾結於「應該跳幾步」，但貪心法的精髓在於只關心「最遠可以到哪裡」，跳過的細節不需要計算。
    - 邊界優化：提前檢查 max_reach >= n - 1 可以大幅節省處理長陣列時的時間，這是競賽中常見的優化技巧
#### 總結 (Takeaway)
- 跳躍遊戲貪心 SOP：
    1. 初始化 max_reach 為 0
    2. 遍歷每個位置 i
    3. 若 i > max_reach 則判定失敗
    4. 更新 max_reach 為 max(max_reach, i + nums[i])
    5. 若 max_reach 到達終點，則判定成功
