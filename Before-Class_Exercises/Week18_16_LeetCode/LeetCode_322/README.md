### 題目：[LeetCode: 322. Coin Change](https://leetcode.com/problems/coin-change/description/)

#### 解題思路
給定一個硬幣面額陣列 coins 和一個總金額 amount，計算湊出該金額所需的最少硬幣數量。若無法湊出，則回傳 -1
- 目標：最小化硬幣總數
- 核心特性：這是一個最佳化問題，且具有「重疊子問題」與「最佳子結構」特性。湊出金額 $i$ 的最少硬幣數，取決於「湊出 $i - \text{coin}$ 的最少硬幣數 + 1」
- 核心策略：使用一個大小為 amount + 1 的 DP 陣列，dp[i] 儲存湊齊金額 i 所需的最少硬幣數

#### 關鍵邏輯：
1. 初始化：
    - 將 dp 陣列初始化為 amount + 1（這是一個大於任何可能解的值，代表「無法湊出」的初始狀態）
    - 設定 dp[0] = 0，因為湊齊 0 元不需要任何硬幣。

3. 邊界判定：
    - 最終若 dp[amount] 仍大於 amount，代表該金額無法透過給定硬幣湊出，返回 -1

#### 模擬運算(coins = [1, 2, 5], amount = 6為例)：
1. dp[0] = 0
2. dp[1] = dp[1-1] + 1 = 1
3. dp[2] = min(dp[2-1]+1, dp[2-2]+1) = 1
4. dp[3] = min(dp[3-1]+1, dp[3-2]+1) = 2
5. dp[4] = min(dp[4-1]+1, dp[4-2]+1) = 2
6. dp[5] = min(dp[5-1]+1, dp[5-2]+1, dp[5-5]+1) = 1
7. dp[6] = min(dp[6-1]+1, dp[6-2]+1, dp[6-5]+1) = 2
- 結果：dp[6] = 2 (5+1)
- Time Complexity：$O(A \times C)$，其中 $A$ 為金額， $C$ 為硬幣種類數量
- Space Complexity：$O(A)$，需要長度為金額的 DP 陣列

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - DP 陣列的初值設定：初始化為 amount + 1 是常見技巧，因為任何金額最多只需 amount 個 1 元硬幣，所以 amount + 1 絕對是一個無法達成的無效值
    - 雙層迴圈順序：此問題中，先遍歷金額或先遍歷硬幣順序皆可，因為我們是在求「最少數量」，而非「組合數」
    - 記憶體效率：相較於遞迴加上 memoization，直接使用陣列進行 iterative DP 的執行速度更快，且不會有 Stack Overflow 的風險
#### 總結 (Takeaway)
- 硬幣找零 DP SOP：
    1. 建立一個長度為 amount + 1 的陣列，填入無效大值
    2. 設定 dp[0] = 0 作為遞迴或迭代的起點
    3. 執行雙層迴圈，根據狀態轉移方程更新陣列中每個金額的最優解
    4. 最終檢查目標金額的 DP 值
