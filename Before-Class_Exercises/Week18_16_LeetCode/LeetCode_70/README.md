### 題目：[LeetCode: 70. Climbing Stairs](https://leetcode.com/problems/climbing-stairs/description/)

#### 解題思路
假設你在爬樓梯，一次可以爬 1 階或 2 階。問爬到第 $n$ 階總共有幾種不同的走法？
- 目標：回傳到達頂層的方法數
- 核心特性：到達第 $n$ 階的方法，取決於你是從第 $n-1$ 階爬 1 步上來的，還是從第 $n-2$ 階爬 2 步上來的
- 核心策略：狀態轉移方程為 $f(n) = f(n-1) + f(n-2)$。這與費氏數列完全相同，透過由底向上的迭代 (Bottom-up)，我們可以避免重複運算

#### 關鍵邏輯：
1. Base Cases：
    - $n=1$: 只有 1 種方法
    - $n=2$: 有 2 種方法 (1+1 或 2)
2. State Transition：
    - 每一階的方法數均為前兩階方法數的總和
3. Space Optimization：
    - 原本需要 $O(N)$ 的陣列來存儲所有階層的方法數，但因為計算當前階數只需要「前兩階」的資料，我們可以使用兩個變數 first 與 second 來不斷更新，將空間複雜度優化至 $O(1)$

#### 模擬運算( $n=5$ 為例)：
1. 第 1 階：1 種
2. 第 2 階：2 種
3. 第 3 階：$1 + 2 = 3$ 種
4. 第 4 階：$2 + 3 = 5$ 種
5. 第 5 階：$3 + 5 = 8$ 種
- Time Complexity：$O(N)$，只需一次迴圈
- Space Complexity：$O(1)$，只使用固定數量的整數變數

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 空間優化技巧：很多動態規劃問題不需要儲存整個表格，只要能觀察出狀態只依賴於前幾個狀態，就能進行「滾動空間」優化
    - 邊界處理：對於 $n=1$ 或 $n=2$ 等極端小輸入，必須在迴圈開始前處理完畢，否則容易產生邏輯錯誤
    - 遞迴與迭代的選擇：初學者常用遞迴 ($O(2^N)$) 實作，但會導致大量冗餘計算。在實際開發中，應儘量採用迭代方法以確保效能
#### 總結 (Takeaway)
- 費氏數列式 DP SOP：
    1. 定義基礎狀態 ($n=1, n=2$ 的值)
    2. 找出狀態轉移方程 (當前 = 前一個 + 前兩個)
    3. 設定變數進行滾動更新
    4. 完成迴圈並輸出結果
