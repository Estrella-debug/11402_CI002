### 題目：[724. Find Pivot Index](https://leetcode.com/problems/find-pivot-index/description/)

#### 解題思路
尋找陣列中的「中心索引 (Pivot Index)」
- 中心索引定義：該索引左側所有數字之和，等於其右側所有數字之和
- 規則：
    1. 如果中心索引在最左邊，左側和為 0
    2. 如果中心索引在最右邊，右側和為 0
    3. 若有多個中心索引，回傳 最左邊 的那一個
    4. 若不存在，回傳 -1

#### 關鍵邏輯：
1. 預算總和 (Precompute Total Sum)：先利用 std::accumulate 計算出整個陣列的總和 totalSum
2. 動態維護左側和 (Maintain Left Sum)：
    - 遍歷陣列，使用一個變數 leftSum 紀錄目前索引左邊的所有數字之和
3. 數學等式判定：
    - 對於任何位置 i，右側和可以表示為：rightSum = totalSum - leftSum - nums[i]
    - 我們只需要檢查 leftSum == totalSum - leftSum - nums[i] 是否成立
    - 關鍵點：這個等式不需要額外的迴圈來計算右側和，大大提升了效能
4. 即時更新：如果當前索引不是中心，則將 nums[i] 加到 leftSum 中，繼續檢查下一個位置

#### 模擬運算(nums = [1, 7, 3, 6, 5, 6]為例)：
- Total Sum = 28.
- i = 0 (1)：left = 0, right = 28 - 0 - 1 = 27。不相等。left 變 1
- i = 1 (7)：left = 1, right = 28 - 1 - 7 = 20。不相等。left 變 8
- i = 2 (3)：left = 8, right = 28 - 8 - 3 = 17。不相等。left 變 11
- i = 3 (6)：left = 11, right = 28 - 11 - 6 = 11。相等
- 結果：回傳索引
- Time Complexity：$O(n)$。計算總和走一遍，搜尋中心索引走一遍
- Space Complexity：$O(1)$。只使用了 totalSum 和 leftSum 兩個變數

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 優化求和操作：初學者可能會在每個索引都用一個 for 迴圈去算左右和（導致 $O(n^2)$），但學會「總和 - 左側 - 自己 = 右側」的邏輯後，複雜度能降到 $O(n)$
    - 累積函式 std::accumulate：這是在 C++ 中處理數值加總最標準且簡潔的方法
    - 左側優先：因為迴圈是從 0 開始增加，這自然保證了如果有多個答案，我們會先遇到並回傳最左邊的中心索引

#### 總結 (Takeaway)
- 中心平衡點問題 SOP：
    1. 算出整體的總分（或總重量）
    2. 由左向右掃描
    3. 利用總分減去目前位置與左側累計，算出右側數值
    4. 比較兩側是否平衡
- 這種「空間換時間」或「數學公式簡化」的思考方式，是解決陣列區間問題的核心
