### 題目：[88. Merge Sorted Array](https://leetcode.com/problems/merge-sorted-array/)

#### 解題思路
給你兩個已經排好序（Sorted）的整數陣列 nums1 和 nums2，請將 nums2 合併到 nums1 中，使合併後的 nums1 也是排好序的。
- 限制條件：nums1 後方已經預留了足夠的空間（補 0）來存放 nums2，我們必須「原地（In-place）」修改 nums1。

#### 關鍵邏輯：
1. 從後往前合併 (Merge from the Back)：如果從前面開始合併，會需要移動 nums1 原有的數字，非常麻煩。因此我們從 nums1 的最末端（tail）開始填入數字。
2. 比較大小：每次比較 nums1[p1] 和 nums2[p2]，誰大誰就先放到 tail 的位置，並將該指標與 tail 向前移動。
3. 剩餘處理：如果 p2 還沒跑完（代表 nums2 還有比較小的數字沒放進去），則將剩餘的 nums2 全部填入 nums1。

#### 模擬運算：
- 第一步：比較 3 和 6。6 較大，放進 nums1[5]。tail 變 4, p2 變 1。
- 第二步：比較 3 和 5。5 較大，放進 nums1[4]。tail 變 3, p2 變 0。
- 第三步：比較 3 和 2。3 較大，放進 nums1[3]。tail 變 2, p1 變 1。
- 最後結果：持續進行直到所有數字定位。
- Time Complexity (時間複雜度)：$O(m + n)$。我們只需要掃描兩個陣列一遍。
- Space Complexity (空間複雜度)：$O(1)$。沒有使用額外的陣列，完全在 nums1 內部操作。

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 逆向思維：這題如果從頭開始合併（From head），會需要頻繁搬移元素；但從尾巴開始填（From tail），可以利用 nums1 後方的空位，效率最高。
    - 邊界判斷：使用 while (p1 >= 0 && p2 >= 0) 來確保指標不會超出範圍。
    - 為什麼不用處理 p1 剩餘？：因為如果 p2 先跑完，p1 剩下的數字本來就在 nums1 的正確位置上，所以不需要額外動作。

#### 總結 (Takeaway)
- 陣列合併問題 SOP：
    1. 確認是否有額外空間或是需要原地修改。
    2. 思考「從後往前」是否比「從前往後」更容易操作。
    3. 利用雙指標來管理兩個陣列的當前進度。
- 這種「倒著做」的技巧在很多陣列修改的題目中非常實用，能將複雜度從 $O(n^2)$ 降低到 $O(n)$
