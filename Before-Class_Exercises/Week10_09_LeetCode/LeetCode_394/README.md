### 題目：[394. Decode String](https://leetcode.com/problems/decode-string/description/)

#### 解題思路
給定一個經過編碼的字串，規則為 k[encoded_string]，表示括號內的 encoded_string 正好重複 k 次
- 目標：將字串完整解碼
- 核心挑戰：
    1. 巢狀結構：如 3[a2[c]]，必須先解開內層的 2[c] 得到 acc，再解開外層
    2. 多位數字：如 100[a]，數字部分可能不只一位，需要正確換算
- 策略：使用兩個堆疊，一個存 數字 (k)，一個存 解碼到一半的字串

#### 關鍵邏輯：
1. isdigit：
    - 當遇到數字字元時，透過 k = k * 10 + (c - '0') 將字元轉換為整數，這能有效處理多位數（如 10, 100）
2. 進入嵌套 (遇到 [ )：
    - 代表舊的任務尚未結束，新的重複任務即將開始
    - 將目前的倍數 k 壓入 counts 堆疊
    - 將目前累積的字串 currentStr 壓入 resultStack 堆疊暫存
    - 重置 k 與 currentStr 以處理括號內的內容
3. 完成嵌套 (遇到 ] ):
    - 代表一個局部的重複任務完成了
    - 取出 counts 堆疊頂部的倍數
    - 取出 resultStack 堆疊頂部的「前段字串」作為基礎
    - 將括號內解碼完的字串重複 $k$ 次，串接到「前段字串」後方
4. 一般字元：
    - 直接加進 currentStr 中

#### 模擬運算(3[a2[c]]為例)：
- 讀到 3：k = 3
- 讀到 [：counts 壓入 3，resultStack 壓入 ""。重置 k=0, currentStr=""
- 讀到 a：currentStr = "a"
- 讀到 2：k = 2
- 讀到 [：counts 壓入 2，resultStack 壓入 "a"。重置 k=0, currentStr=""
- 讀到 c：currentStr = "c"
- 讀到 ]：
    - 彈出倍數 2，彈出前段 "a"
    - 組合："a" + "c" * 2 = "acc"。此時 currentStr = "acc"
- 讀到 ]：
    - 彈出倍數 3，彈出前段 ""
    - 組合："" + "acc" * 3 = "accaccacc"
- Time Complexity：$O(L)$，其中 $L$ 是解碼後字串的長度
- Space Complexity：$O(D)$，其中 $D$ 是括號嵌套的最大深度

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 雙堆疊同步：學會用兩個堆疊分別管理數量與內容，這是解析層級化資料（如 JSON, XML, 或數學公式）的基礎
    - ASCII 轉換：使用 c - '0' 處理字元轉數字，並理解處理多位數的累乘邏輯
    - 字串拼接效能：在 C++ 中，字串拼接 += 的效能優於不斷重新分配空間，但在極大量資料下需注意字串拷貝的成本

#### 總結 (Takeaway)
- 嵌套編碼問題 SOP：
    1. 遇到數字：更新計數器
    2. 遇到 [：將當前狀態「快照」存入堆疊，清空工作區
    3. 遇到字元：更新工作區
    4. 遇到 ]：彈出堆疊頂部的快照，與目前工作區結果合併
