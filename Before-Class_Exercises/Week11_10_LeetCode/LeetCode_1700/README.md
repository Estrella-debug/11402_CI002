### 題目：[1700. Number of Students Unable to Eat Lunch](https://leetcode.com/problems/number-of-students-unable-to-eat-lunch/description/)

#### 解題思路
學校食堂提供兩種三明治（0 和 1），學生們排隊領取
- 領取規則：
    1. 隊伍最前面的學生如果喜歡最上面的三明治，就拿走三明治並離開隊伍
    2. 如果不喜歡，該學生會回到隊伍的最末尾重新排隊
- 目標：計算最終有多少學生無法吃到三明治
- 核心挑戰：當隊伍中剩下的所有學生都不喜歡最上面的那個三明治時，領取過程會永遠停止

#### 關鍵邏輯：
1. 資料結構模擬：
    - 學生 (Queue)：使用 std::queue 維持「先進先出」的排隊特性。不喜歡三明治的學生會 pop 出來再 push 回去
    - 三明治 (Stack)：使用 std::stack 維持「後進先出」的特性。注意：為了讓陣列的第一個三明治在最上面，壓入堆疊時需反向遍歷
2. 死鎖判定：
    - 引入一個計數器 count
    - 每當一名學生不喜歡三明治並回到隊尾，count++
    - 每當一名學生成功領到三明治，count 重置為 0
    - 停止條件：當 count 等於目前佇列的長度時，代表隊伍中每個人都已經輪流看過這個三明治且都不喜歡，此時程式終止
3. 結果回傳：
    - 迴圈結束後，佇列中剩餘的人數即為無法吃午餐的學生總數

#### 模擬運算(students = [1, 1, 0], sandwiches = [0, 1, 1]為例)：
- 初始狀態：Queue: [1, 1, 0], Stack: [0, 1, 1] (Top is 0), count = 0
- 輪次 1：學生 1 不喜歡三明治 0
    - Queue: [1, 0, 1], count = 1
- 輪次 2：學生 1 不喜歡三明治 0
    - Queue: [0, 1, 1], count = 2
- 輪次 3：學生 0 喜歡三明治 0
    - 拿走,Queue: [1, 1], Stack: [1, 1], count = 0
- 輪次 4：學生 1 喜歡三明治 1
    - 拿走,Queue: [1], Stack: [1], count = 0
- 輪次 5：學生 1 喜歡三明治 1
    - 拿走,Queue: [], Stack: []
- 結果：回傳 0
- Time Complexity：$O(N^2)$。在最差情況下（每個人都領到），學生可能會循環排隊多次，但總體操作次數受限於學生人數與三明治數量的乘積
- Space Complexity：$O(N)$。需要空間儲存佇列與堆疊

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 容器特性選擇：理解了為什麼學生適合用 queue（循環排隊）而三明治適合用 stack（只能拿最上面）
    - 計數器的妙用：不需要複雜的邏輯，只需記錄「連續不匹配次數」就能偵測出循環是否進入死胡同
    - 優化思考：實際上這題可以不用 queue 和 stack，只需統計學生喜歡 0 和 1 的總人數，再依序消耗三明治，直到某種三明治沒人要為止即可優化到 $O(N)$

#### 總結 (Takeaway)
- 循環隊列問題 SOP：
    1. 將動態變動的群體放入 queue
    2. 將靜態等待消耗的群體放入 stack
    3. 設定一個「完全輪替計數器」作為終止條件
    4. 依規則執行 pop、push 或 top 操作
