### 題目：[950. Reveal Cards In Increasing Order](https://leetcode.com/problems/reveal-cards-in-increasing-order/description/)

#### 解題思路
有一疊牌，翻牌規則如下：
1. 翻開最上面的牌，將其移除
2. 如果還有剩餘的牌，將下一張牌移到這疊牌的最底層
    - 目標：給定一組牌，找出一種初始排列方式，使得按照上述規則翻牌時，翻出的數字是由小到大遞增排列的
    - 核心策略：我們不需要真的去猜排列，而是直接 模擬索引的移動。既然翻出的牌要由小到大，我們就先將牌排序，然後透過佇列模擬「翻開」與「搬移」的索引順序，再將牌填入對應的位置

#### 關鍵邏輯：
1. Sorting：
    - 為了讓翻開的結果是遞增的，我們必須將原始牌組 deck 從小到大排序
2. Index Simulation：
    - 建立一個包含所有索引 0, 1, ..., n-1 的佇列 indexQueue
    - 這些索引代表最終結果陣列 result 中的「空位」
3. 模擬翻牌規則：
    - 遍歷排序後的每一張牌：
        - 取出位置：從 indexQueue 的前端取出一個索引，這個索引就是目前最小的牌應該放置的位置
        - 模擬移動：如果佇列不為空，按照題目規則，將下一個索引移動到佇列的末尾
4. 填入結果：
    - 將排序後的牌依照模擬出來的索引順序填入 result 陣列

#### 模擬運算(deck = [2, 13, 3, 11]為例)：
- Step 0 (排序與初始化)：deck = [2, 3, 11, 13], indexQueue = [0, 1, 2, 3]
- 處理 2：
    - 取出索引 0。result[0] = 2
    - 將索引 1 移到末尾。indexQueue = [2, 3, 1]
- 處理 3：
    - 取出索引 2。result[2] = 3
    - 將索引 3 移到末尾。indexQueue = [1, 3]
- 處理 11：
    - 取出索引 1。result[1] = 11
    - 將索引 3 移到末尾。indexQueue = [3]
- 處理 13：
    - 取出索引 3。result[3] = 13
- 結果：result = [2, 11, 3, 13]
- Time Complexity：$O(N \log N)$。排序需要 $O(N \log N)$，佇列模擬與填值需要 $O(N)$
- Space Complexity：$O(N)$。需要額外的佇列與結果陣列儲存空間

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 索引模擬法：面對複雜的移動規則時，直接模擬「位置（索引）」的變化比直接搬移「數值」更不容易出錯
    - 佇列的雙重操作：這題展示了佇列如何處理「間隔操作」（取一個、移一個）的邏輯
    - 逆向思維：題目描述的是翻牌過程，但我們透過模擬索引順序，將原本看似隨機的排列轉化為確定的填值過程

#### 總結 (Takeaway)
- 規則模擬問題 SOP：
    1. 將目標對象（數值）排序以符合輸出期待
    2. 使用資料結構（如 queue 或 deque）模擬物件的移動邏輯
    3. 在模擬過程中紀錄「應該被處理的位置」
    4. 依照紀錄的位置順序填入對應的數值
