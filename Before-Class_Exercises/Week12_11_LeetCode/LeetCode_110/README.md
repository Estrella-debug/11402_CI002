### 題目：[110. Balanced Binary Tree](https://leetcode.com/problems/balanced-binary-tree/description/)

#### 解題思路
給定一個二元樹，判斷它是否為 高度平衡
- 定義：一個二元樹每個節點的左右兩個子樹的高度差絕對值不超過 1
- 核心策略：使用遞迴由底向上檢查。如果直接對每個節點呼叫一次高度函式，會導致重複計算（複雜度 $O(N^2)$）。本解法透過「回傳高度或錯誤代碼」的技巧，將複雜度降至 $O(N)$

#### 關鍵邏輯：
1. Bottom-up Approach：
    - 先檢查左子樹高度，再檢查右子樹高度，最後處理當前節點。這確保了我們在判定當前節點前，已經知道下方所有子樹的狀態
2. 狀態標記：
    - 正常狀態：若子樹平衡，函式回傳該子樹的真實 高度
    - 不平衡狀態：若子樹已失去平衡，函式回傳 -1 作為標記
3. 剪枝優化：
    - 在遞迴過程中，一旦發現左子樹或右子樹回傳了 -1，代表該分支已經毀壞，不需要再進行任何高度計算，直接向上傳遞 -1 即可
4. 平衡判定公式：
    - 若左右子樹高度差 $abs(l - r) > 1$，則當前節點不平衡，回傳 -1
    - 否則回傳當前節點高度：$\max(l, r) + 1$

#### 模擬運算(不平衡樹為例)：
- 深層遞迴：到達節點 4，高度為 1
- 節點 3：左邊高度 1 (來自 4)，右邊高度 1 (來自 4)，平衡，回傳高度 2
- 節點 2 (左)：左邊高度 2 (來自 3)，右邊高度 1 (來自 3 的另一側)，平衡，回傳高度 3
- 根節點 1：左子樹高度 3，右子樹高度 1
- 判斷：$abs(3 - 1) = 2$，大於 1
- 結果：回傳 -1，判定為 False
- Time Complexity：$O(N)$。每個節點僅被存取一次
- Space Complexity：$O(H)$，其中 $H$ 是樹的高度。主要為遞迴時系統棧的開銷

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 多合一函式設計：學會讓一個遞迴函式同時承擔「計算結果」與「狀態回報」兩種責任，避免重複走訪節點
    - 遞迴終止條件：處理二元樹時，空節點 (!x) 回傳高度 0 是最穩定的基礎
    - 早停機制 (Early Exit)：透過 if (l == -1) return -1; 的寫法，讓錯誤能像滾雪球一樣快速傳回頂端，節省運算量

#### 總結 (Takeaway)
- 二元樹屬性檢查 SOP：
    1. 設計一個由底向上的遞迴函式
    2. 定義一個特殊值（如 -1 或特定旗標）來代表非法狀態
    3. 先遞迴左、右子樹，並立即檢查是否有非法狀態傳回
    4. 在當前節點根據子樹結果進行判定
    5. 最終在主程式中檢查根節點的回傳值
