### 題目：[1109. Corporate Flight Bookings](https://leetcode.com/problems/corporate-flight-bookings/description/)

#### 解題思路
有 $n$ 個航班（編號 1 到 $n$），以及一組訂票記錄 bookings。每筆記錄 [first, last, seats] 代表從 first 到 last 這些航班都要預訂 seats 個座位
- 目標：回傳一個長度為 $n$ 的陣列，表示每個航班最終預訂的總座位數
- 策略：如果直接對每個區間進行 for 迴圈修改，複雜度會達到 $O(Booking \times n)$，在數據量大時會超時。因此我們使用 差分陣列

#### 關鍵邏輯：
1. 初始化差分陣列：建立一個長度為 $n+1$ 的 diff 陣列，初始值皆為 0
2. 區間更新：
    - 對於每筆 [first, last, seats]：
        - 在起始位置 first-1 處增加座位：diff[first-1] += seats
        - 在結束位置的後一個位置 last 處減少座位：diff[last] -= seats（如果 last 還在範圍內）
    - 原理：我們只記錄「變化量」，而不是記錄每一格的數值
3. 還原前綴和：
    - 差分陣列的「前綴和」就是原始陣列
    - 遍歷 diff 陣列並不斷累加，即可得到每個航班最終的座位數

#### 模擬運算(bookings = [[1, 2, 10], [2, 3, 20]], n = 3 為例)：
- 初始 diff：[0, 0, 0, 0]
- 處理 [1, 2, 10]：diff[0] += 10, diff[2] -= 10 $\rightarrow$ [10, 0, -10, 0]
- 處理 [2, 3, 20]：diff[1] += 20, diff[3] -= 20 $\rightarrow$ [10, 20, -10, -20]
- 還原前綴和：
    - 航班 1: 10
    - 航班 2: 10 + 20 = 30
    - 航班 3: 30 + (-10) = 20
- 結果：[10, 30, 20]
- Time Complexity：$O(n + \text{bookings.size()})$。處理訂單與還原陣列都只需要一次遍歷
- Space Complexity：$O(n)$。需要額外空間儲存差分陣列

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 區間修改神技：差分陣列能將區間修改的複雜度從 $O(n)$ 降到 $O(1)$，這是處理大量區間操作的標配
    - 索引偏移 (Offset)：題目編號是從 1 開始，而 C++ 陣列是從 0 開始，因此處理 first-1 與 last 時需要格外小心
    - 前綴和與差分的關係：這兩者是互逆的運算，掌握了這層關係就能靈活處理區間加減問題

#### 總結 (Takeaway)
- 區間頻繁修改問題 SOP：
    1. 建立差分陣列
    2. 對每個區間執行「頭加、尾後減」
    3. 最後執行前綴和累加，還原出原始數值
- 差分陣列是解決「大量區間增減」問題的最終解法，在工程實務與演算法比賽中都非常實用
