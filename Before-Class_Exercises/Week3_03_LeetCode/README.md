# [169. Majority Element](https://leetcode.com/problems/majority-element/)

### 解題思路 (Logic)
這題的目標是找出陣列中出現次數大於 **⌊n / 2⌋** 的元素。題目保證這個元素在陣列中一定存在。

#### 直覺解法 (Hash Map)
- **思路**：使用 `unordered_map<int, int>` 來紀錄 `[數字, 出現次數]`。掃描陣列，當某個數字的次數超過 `n / 2` 時即為答案。
- **時間複雜度**：$O(n)$
- **空間複雜度**：$O(n)$ (需要額外空間存地圖)

#### 排序法 (Sorting)
- **思路**：既然「多數元素」的數量超過一半，排序後**陣列正中間的數字** `nums[n / 2]` 絕對就是答案。
- **時間複雜度**：$O(n \log n)$ (排序的成本)
- **空間複雜度**：$O(1)$ 或 $O(\log n)$

#### 最佳化解法 (Boyer-Moore Voting Algorithm)
- **思路**：這是面試最推崇的解法。將其想像成「投票抵銷」。
  - 設一個 `candidate` (候選人) 和 `count` (票數)。
  - 遇到相同的數字，`count + 1`；遇到不同的數字，`count - 1` (互相抵銷)。
  - 當 `count == 0` 時，更換當前的數字為新的 `candidate`。
  - 因為多數派的人數超過一半，戰到最後活下來的一定是它。
- **時間複雜度**：$O(n)$
- **空間複雜度**：$O(1)$ (最強大的地方：完全不耗費額外記憶體)

#### 語言差異與踩坑紀錄
- 數學特性：有時候不需要真的去計數。利用「超過一半」這個幾何特性，可以將空間複雜度從 $O(n)$ 降到 $O(1)$。
- C++ 排序工具：複習了 std::sort(nums.begin(), nums.end())，雖然這題有更快的 $O(n)$ 解法，但排序法在程式碼實作上最簡潔。

#### 總結
- 核心盲點：容易陷入「必須算出每個數字出現幾次」的傳統陷阱。
- 反射動作：只要看到題目關鍵字出現 「超過一半 (Majority)」，腦中第一秒就要想到 Boyer-Moore Voting Algorithm (投票法)。
