### 題目：[1046 – Last Stone Weight](https://leetcode.com/problems/last-stone-weight/description/)

#### 解題思路
有一堆石頭，每顆石頭都有一個重量。每次碰撞時，我們挑選兩顆「最重」的石頭進行碰撞：
- 若 $x == y$，兩顆石頭皆碎
- 若 $x \neq y$，碎掉較小的那顆，較大的那一顆剩下 $x - y$ 的重量
- 目標：最後剩下一顆石頭或完全碎裂時，回傳最後的重量
- 核心策略：為了高效地反覆取得「最重」的兩顆石頭，我們需要一個能自動維護排序的資料結構，即 最大堆疊 (Max-Heap)

#### 關鍵邏輯：
1. 最大堆疊初始化：
    - 利用 std::priority_queue<int> 將所有石頭重量塞入。在 C++ 中，priority_queue 預設就是一個最大堆疊，這保證了 pq.top() 永遠是當前集合中最重的石頭
2. Simulation:
    - 每次從堆疊中 pop 出最重的兩顆石頭 x 與 y
    - 計算碰撞後的結果。若剩餘重量 $> 0$，則 push 回堆疊
    - 這個過程持續進行，直到堆疊中剩下少於兩顆石頭
3. 邊界處理：
    - 若最後堆疊為空，代表所有石頭都完全抵消，回傳 0
    - 若堆疊不為空，回傳 pq.top() 即為最後剩下的石頭重量

#### 模擬運算(輸入 {2, 7, 4, 1, 8, 1}為例)：
- 堆疊狀態 (由大到小)：[8, 7, 4, 2, 1, 1]
- 第一輪碰撞：取出 8 與 7，剩餘 1。堆疊狀態：[4, 2, 1, 1, 1]
- 第二輪碰撞：取出 4 與 2，剩餘 2。堆疊狀態：[2, 1, 1, 1]
- 第三輪碰撞：取出 2 與 1，剩餘 1。堆疊狀態：[1, 1, 1]
- 第四輪碰撞：取出 1 與 1，兩者皆碎。堆疊狀態：[1]
- 最終結果：剩下 1
- Time Complexity：$O(N \log N)$。每個石頭最多被 push 和 pop 一次，而堆疊操作的時間複雜度為 $O(\log N)$
- Space Complexity：$O(N)$，用於儲存石頭重量的堆疊

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - Max-Heap 的自動排序：深刻理解了 priority_queue 如何幫助我們省去手動排序的繁瑣過程（否則每次都要重新排序 $O(N \log N)$，整體效率會降至 $O(N^2 \log N)$）
    - abs 的運用：雖然 x 必定大於等於 y，但在碰撞邏輯中使用 abs(x-y) 讓程式碼更具備容錯性與邏輯彈性
    - 效率思維：在這種需要反覆取得最值（Max/Min）的問題中，Heap 是遠比陣列或 Vector 更優異的選擇
#### 總結 (Takeaway)
- 堆疊應用問題 SOP：
    1. 確認是否需要反覆取得「最值」（最重、最大、最小）
    2. 將元素初始化放入 priority_queue
    3. 利用迴圈進行模擬，直到條件滿足
    4. 檢查堆疊狀態，回傳最終結果
