### 題目：[31. Next Permutation](https://leetcode.com/problems/next-permutation/)

#### 解題思路
給定一個整數陣列 nums，請找出其「下一個」更大的排列
- 規則：必須是字典序中的下一個。如果已經是最大的排列（例如 [3, 2, 1]），則將其重新排列為最小的排列（即升序 [1, 2, 3]）
- 限制：必須「原地（In-place）」修改，只能使用額外的常數空間

#### 關鍵邏輯：
1. 尋找轉折點 (Find the Pivot)：從右向左找第一個「遞減」的數字，即找到索引 i 使得 nums[i] < nums[i+1]。這代表 i 之後的數字已經是最大排列，我們必須變動 nums[i] 來取得下一個排列
2. 尋找交換對象 (Find Successor)：如果在左邊找到了這樣的 i，我們再次從右向左找，找到第一個比 nums[i] 大的數字 nums[j]，並交換它們
3. 反轉尾部 (Reverse Tail)：交換後，i 之後的序列依然是降序的（最大的）。為了讓整個數字最小，我們需要將 i+1 到末尾的部分進行「反轉」，使其變成升序（最小的）

#### 模擬運算(nums = [1, 2, 3]為例)：
- 第一步：從右往左看，發現 2 < 3，所以轉折點 i = 1 (數字是 2)
- 第二步：從右往左找比 2 大的數，發現 3 > 2，交換它們。陣列變成 [1, 3, 2]
- 第三步：反轉 i 之後的部分（原本是 2）。陣列維持 [1, 3, 2]
- 結果：回傳 [1, 3, 2]
- Time Complexity：$O(n)$。最多掃描兩次陣列加上一次反轉
- Space Complexity：$O(1)$。只在原陣列進行交換

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 邊界處理：如果整個陣列都是降序（找不到 i >= 0），代表它是最大排列。此時直接執行第三步的 reverse，陣列就會變成升序（最小排列），符合題目要求
    - 內建函式庫：在 C++ 中，其實有內建的 std::next_permutation 可以直接使用，但手寫一次邏輯能更深入理解字典序的本質
    - 反轉的意義：反轉之所以有效，是因為原本 i 之後的部分是降序排列，反轉後會變成升序，這保證了我們得到的「下一個排列」是所有大於原排列中最小的那一個

#### 總結 (Takeaway)
- 下一個排列 SOP：
    1. 找轉折點：由後往前找第一個 $nums[i] < nums[i+1]$
    2. 找大於者：由後往前找第一個比 $nums[i]$ 大的數並交換
    3. 反轉後段：將 $i$ 之後的部分由大到小反轉為由小到大
- 這種「觀察規律並分段處理」的方法，是處理排列與組合問題的標準流程
