### 題目：[150. Evaluate Reverse Polish Notation](https://leetcode.com/problems/evaluate-reverse-polish-notation/description/)

#### 解題思路
計算一個後置表達式（RPN）的結果
- RPN 特點：運算符號跟在操作數之後。例如 2 1 + 等於中置表達式的 2 + 1。其優點是不需要括號就能表示運算優先順序
- 核心策略：使用堆疊來儲存數字。每當遇到數字就推入；遇到運算符號就從堆疊取出最新的兩個數字進行運算，並將結果存回堆疊

#### 關鍵邏輯：
1. 數字處理：
    - 遍歷 tokens，若字串不是 +, -, *, /，則視為數字
    - 使用 stoi() 將字串轉換為整數後壓入堆疊
2. Operand Order：
    - 關鍵陷阱：堆疊是「後進先出」。當彈出兩個數字時，第一個彈出的是右操作數 ($x$)，第二個彈出的是左操作數 ($y$)
    - 對於減法和除法，順序極其重要：結果應為 y - x 或 y / x
3. 算術運算：
    - 根據運算符號執行對應的加、減、乘、除
    - C++ 的整數除法會自動向零取整（例如 13 / 5 = 2），符合題目規範
4. 獲取結果：
    - 表達式處理完畢後，堆疊頂端的最後一個數字即為最終答案
#### 模擬運算(["4", "13", "5", "/", "+"]為例)：
- 讀取 "4"：堆疊 [4]
- 讀取 "13"：堆疊 [4, 13]
- 讀取 "5"：堆疊 [4, 13, 5]
- 讀取 "/"：
    - 彈出 $x = 5, y = 13$
    - 計算 $13 / 5 = 2$
    - 堆疊 [4, 2]
- 讀取 "+"：
    - 彈出 $x = 2, y = 4$
    - 計算 $4 + 2 = 6$
    - 堆疊 [6]
- 結果：輸出 6
- Time Complexity：$O(n)$。每個 token 只會被處理一次，且堆疊操作均為 $O(1)$
- Space Complexity：$O(n)$。在最差情況下（如全是數字），堆疊會存儲所有元素

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 操作數順序：深刻理解了減法與除法在 RPN 中的左右順序，這是最容易出錯的地方
    - 類型轉換：熟練使用 stoi 處理字串轉整數
    - 計算機原理：RPN 是計算機底層（如編譯器、工程計算機）評估數學公式的標準方法
#### 總結 (Takeaway)
- RPN 運算問題 SOP：
    1. 建立一個整數堆疊
    2. 遍歷所有 Token
    3. 是數字：push 進棧
    4. 是符號：pop 兩次 $\rightarrow$ 注意左右順序 $\rightarrow$ 計算 $\rightarrow$ push 回棧
    5. 最終回傳 stack.top()
