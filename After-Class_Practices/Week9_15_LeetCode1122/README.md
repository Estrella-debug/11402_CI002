### 題目：[1122. Relative Sort Array](https://leetcode.com/problems/relative-sort-array/description/)

#### 解題思路
給定兩個陣列 arr1 和 arr2，依據特定的「相對順序」對 arr1 進行排序
- 排序規則：
    1. 優先放入 arr1 中有出現在 arr2 裡的元素，且順序必須與 arr2 完全一致
    2. 剩下的元素（未出現在 arr2 中的）則需按照 升序（從小到大） 排在末尾
- 核心策略：利用計數排序的思想，先統計 arr1 每個數字出現幾次，再依序取出

#### 關鍵邏輯：
1. Frequency Array:
    - 由於題目限制數值在 $0 \sim 1000$ 之間，我們宣告 int count[1001] 來記錄每個數字出現的頻率
    - 這種做法比 std::map 更快，因為它省去了樹狀結構或雜湊計算的開銷，直接透過索引存取只需 $O(1)$
2. Relative Order：
    - 遍歷 arr2，根據 arr2 出現的順序，將 count 陣列中對應的元素個數全部填入結果陣列 res，並將該計數歸零
3. Ascending Remainder：
    - 遍歷 count 陣列從索引 $0 \sim 1000$
    - 此時 count[i] > 0 的位置就是那些沒被 arr2 點名到的數字
    - 因為我們是從 $0$ 開始往後遍歷，這些剩餘數字會自然而然地以升序加入結果中
#### 模擬運算(arr1 = [2, 3, 1, 3], arr2 = [2, 1]為例)：
- Step 1 (統計)：count[1]=1, count[2]=1, count[3]=2
- Step 2 (依 arr2 提取)：
    - 先看 2：res = [2], count[2]=0
    - 再看 1：res = [2, 1], count[1]=0
- Step 3 (餘數掃描)：
    - 發現 count[3] 還有 $2$ 個
    - res = [2, 1, 3, 3]
- Time Complexity：$O(N + K)$。$N$ 是 arr1 的長度，$K$ 是數值的範圍（本題固定為 1001）
- Space Complexity：$O(K)$。需要一個固定大小的陣列來儲存頻率

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 固定範圍的優化：當題目給出明確的數值範圍時，陣列計數通常是比 std::sort 或 std::map 更強大的武器
    - 邏輯簡化：最後一步處理剩餘元素時，直接從 $0$ 遍歷到 $1000$ 同時解決了「尋找剩餘」與「升序排列」兩個需求
    - 記憶體預分配：在 C++ 中，如果知道結果陣列的大小，可以使用 res.reserve(arr1.size()) 來減少動態擴張帶來的效能損失
#### 總結 (Takeaway)
- 相對排序問題 SOP (範圍有限時)：
    1. 建立一個足以容納所有數值的頻率陣列
    2. 遍歷參考陣列，優先消耗頻率
    3. 循序遍歷頻率陣列，補齊剩餘元素
