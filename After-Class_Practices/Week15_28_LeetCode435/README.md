### 題目：[LeetCode: 435. Non-overlapping Intervals](https://leetcode.com/problems/non-overlapping-intervals/description/)

#### 解題思路
給定一組區間 intervals，計算為了讓剩餘區間互不重疊，最少需要移除多少個區間
- 目標：最小化移除數量，等同於 最大化互不重疊的區間數量
- 核心特性：為了讓剩餘區間最多，我們應該優先選擇「結束時間越早」的區間。因為結束得越早，留給後面區間的可用空間就越大
- 核心策略：貪心法。將區間按「結束時間」排序，並依序檢查每個區間是否與前一個已選區間重疊

#### 關鍵邏輯：
1. 排序策略：
    - 使用 std::sort 對區間依照 end 時間進行升序排列。這是貪心選擇的關鍵，確保我們總是先處理最快結束的任務
2. 重疊判斷：
    - 維護一個 prev_end 變數，記錄目前被接受的最後一個區間的結束時間
    - 當下一個區間的起始時間 intervals[i][0] 小於 prev_end 時，說明兩者重疊，我們必須移除當前這個區間（removals++）
3. 更新狀態：
    - 若不重疊，則保留該區間，並將 prev_end 更新為該區間的結束時間

#### 模擬運算([[1,2], [2,3], [1,3]]為例)：
1. 排序：[[1,2], [1,3], [2,3]] (按結束時間排序)
2. 初始：prev_end = 2 (第一個區間 [1,2])
3. 遍歷 [1,3]：
    - start(1) < prev_end(2)，判定為重疊，removals = 1
4. 遍歷 [2,3]：
    - start(2) >= prev_end(2)，不重疊，prev_end 更新為 3
- 結果：需要移除 1 個區間
- Time Complexity：$O(N \log N)$，主要耗時在排序
- Space Complexity：$O(1)$ 或 $O(N)$，視排序演算法的實作而定

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 結束時間排序的重要性：若按開始時間排序，我們難以判斷哪個區間會擋住更多後續區間；按結束時間排序則保證了「騰出更多空間」的最優選擇
    - 重疊定義：注意題目中的重疊是指 start < end，若 start == end 則通常視為不重疊（這點需視具體題目要求而定）
    - Lambda 表達式：使用 C++ 的 Lambda ([]...) 作為 sort 的自定義比較函式，是處理複雜結構體排序的現代標準寫法
#### 總結 (Takeaway)
- 區間最大化選擇 SOP：
    1. 依照「結束時間」進行升序排序
    2. 選取第一個區間，並記下其結束時間
    3. 掃描後續區間，若起始時間小於上一個結束時間，即發生重疊需移除
    4. 若起始時間大於等於結束時間，則接受新區間並更新結束時間
