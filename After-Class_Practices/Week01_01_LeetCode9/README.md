### 題目：[9. Palindrome Number ](https://leetcode.com/problems/palindrome-number/)

#### 解題思路
這題的核心在於判斷一個整數是否為迴文 (Palindrome)，也就是「正著讀」和「反著讀」都一樣。

最直覺的做法是把數字轉成字串，但題目通常會隱含挑戰：能不能不使用字串來完成？這時我們可以使用數學方法，將數字的每一位數取出並「反轉」後，再與原數字比較。

#### 關鍵邏輯：
1. 負數判定：所有的負數（如 -121）都不可能是迴文，因為反轉後會變成 121-。
2. 數字反轉：利用 while 迴圈配合取餘數 % 和除法 / 來重新建構數字。
3. 溢位處理：當反轉一個很大的整數時，結果可能會超過 int 的範圍。因此，將儲存反轉結果的變數 y 宣告為 long 型態。

#### 模擬運算 (以 121 為例)：
- 第一步：y = 0 * 10 + 1 = 1；x = 12
- 第二步：y = 1 * 10 + 2 = 12；x = 1
- 第三步：y = 12 * 10 + 1 = 121；x = 0
- 最後比較：原始值 121 是否等於反轉值 121？是的。
- 時間複雜度：$O(\log_{10}(n))$。數字有幾位數，迴圈就執行幾次。
- 空間複雜度：$O(1)$。只使用了少數幾個變數。

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 資料型態選擇：在 C++ 中，反轉數字時極易遇到溢位（Overflow）問題，使用 long 可以安全地存下反轉後的結果。
    - 暫存原始值：因為在 while 迴圈中我們會不斷修改 x 的值直到它變成 0，所以必須事先用一個變數 m 把原始的 x 存起來，最後才能進行比較。

#### 總結 (Takeaway)
- 處理數字翻轉的標準 SOP：
    1. 利用 x % 10 取得最後一位數。
    2. 利用 y = y * 10 + digit 將數字推入新變數。
    3. 利用 x /= 10 移除最後一位數。
- 這種數學解法比轉成字串更省記憶體，也是面試中更常被要求的解法。
