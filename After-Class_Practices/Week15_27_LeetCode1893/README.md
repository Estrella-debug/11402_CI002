### 題目：[LeetCode: 1893. Range Covered](https://leetcode.com/problems/check-if-all-the-integers-in-a-range-are-covered/description/)

#### 解題思路
給定一個二維陣列 ranges（每個子陣列代表一個覆蓋區間 $[start, end]$）以及兩個整數 left 和 right。判斷在 $[left, right]$ 區間內的所有整數是否都被這些區間至少覆蓋了一次
- 目標：回傳布林值，確認目標區間是否完全被覆蓋
- 核心特性：直接遍歷所有區間進行塗色會非常耗時，但題目範圍極小（數字上限 50）。即便範圍擴大，我們依然可以使用「差分陣列」技術
- 核心策略：利用差分陣列在區間起點加 1、終點後一位減 1，再透過一次前綴和掃描來還原每個位置的覆蓋次數

#### 關鍵邏輯：
1. 差分初始化：
    - 建立一個大小為 52 的陣列（涵蓋題目要求的 1~50 範圍）
    - 對於每個區間 [r[0], r[1]]：
        - diff[r[0]]++：表示從此處開始，覆蓋數加 1
        - diff[r[1] + 1]--：表示從此處之後，覆蓋數減 1（即區間終止）
2. Prefix Sum Reconstruction：
    - 遍歷陣列計算 current_coverage，這還原了每個點 $i$ 的實際覆蓋次數
3. 覆蓋檢查：
    - 在目標範圍 $[left, right]$ 內，若發現任何一個位置的 current_coverage <= 0，則代表該位置未被覆蓋，直接返回 false

#### 模擬運算(ranges = [[1, 2], [3, 4]], left = 1, right = 3為例)：
1. 差分操作：
    - [1, 2] $\rightarrow$ diff[1]++, diff[3]--
    - [3, 4] $\rightarrow$ diff[3]++, diff[5]--
    - 陣列狀態：diff[1]=1, diff[3]=0, diff[5]=-1
2. 計算前綴和 (1 到 3)：
    - i=1: cov = 0+1 = 1 ($>0$, OK)
    - i=2: cov = 1+0 = 1 ($>0$, OK)
    - i=3: cov = 1+0 = 1 ($>0$, OK)
- 結果：完全覆蓋，回傳 true
- Time Complexity：$O(N + R)$，其中 $N$ 為區間數量，$R$ 為數值範圍
- Space Complexity：$O(R)$，陣列空間固定為範圍大小

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 差分與前綴和的雙向關係：這是演算法中的「黃金組合」。差分擅長區間修改，前綴和擅長區間查詢
    - 邊界條件 r[1] + 1：這是最容易寫錯的地方，務必確保終點加 1 的位置在陣列長度範圍內，避免 Out of Bounds
    - 預期輸入的特性：若數字範圍極大（例如 $10^9$），差分陣列法會失敗（空間不足），那時則需要改用「線段合併」或「區間掃描線」演算法
#### 總結 (Takeaway)
- 差分陣列處理區間 SOP：
    1. 建立一個比數值範圍大 1 的陣列
    2. 對每個區間 $[L, R]$ 進行 $diff[L]++, diff[R+1]--$ 操作
    3. 對陣列計算前綴和
    4. 根據業務需求檢查結果（如：檢查該點是否大於 0）
