### 題目：[63 - Unique Paths II](https://leetcode.com/problems/unique-paths-ii/description/)

#### 解題思路
機器人同樣從左上角移動到右下角，但網格中隨機分佈著障礙物（以 1 表示，空地以 0 表示）
- 目標：計算機器人到達右下角有多少條唯一的路徑（不能經過障礙物）
- 核心特性：
    1. 若網格的某個位置 obstacleGrid[i][j] 為 1，則到達此位置的路徑數為 0
    2. 路徑計算依然遵循「由上方與左方格子相加」的原則
- 核心策略：使用空間優化的 DP，但在更新前必須檢查當前位置是否為障礙物，若是則強制將 DP 值設為 0
#### 關鍵邏輯：
1. 狀態定義與初始化：
    - 使用 vector<long long> dp(n, 0) 來維護當前處理行的狀態
    - long long 是為了防止路徑總數在大型網格下超過 int 的範圍
    - dp[0] = 1 是起點的預設值，但若起點就是障礙物，開頭的檢查邏輯會直接返回 0
2. Transition Formula:
    - 遍歷每一個網格單元 (i, j)：
        - 若是障礙物：dp[j] = 0（到達此處的路徑被截斷）
        - 若是空地且非第一列：dp[j] = dp[j] + dp[j-1]
3. 邊界安全性：
    - 程式碼開頭即檢查 obstacleGrid[0][0] == 1，這是一個非常好的防禦性寫法，能快速處理不可行的測資

#### 模擬運算( $2 \times 2$ 網格且 (0, 1) 為障礙物為例)：
- 網格：[[0, 1], [0, 0]]
- 初始化：dp = [1, 0] (第一行)
- 處理 Row 0：
    - j=0: dp[0] 保持 1
    - j=1: 是障礙物，dp[1] = 0
- 處理 Row 1：
    - j=0: 不是障礙物，dp[0] 保持 1 (從上方繼承)
    - j=1: 不是障礙物，dp[1] = dp[1](上) + dp[0](左) = 0 + 1 = 1
- 結果：輸出 dp[1] = 1
- Time Complexity：$O(m \times n)$
- Space Complexity：$O(n)$。僅需維護一維空間

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 狀態重置的細節：障礙物的處理不只是跳過，而是要將該位置的 dp 值清零，這是因為障礙物切斷了所有通過此點的路徑
    - 資料型態溢出：當網格很大時，路徑總數非常容易超過 2^31 - 1，因此使用 long long 是撰寫 DP 網格題的良好習慣
    - 原地修改 (In-place) 的利弊：雖然原地修改可以節省空間，但若邏輯不夠細膩，容易在「更新值」與「作為下一格的參考值」之間產生衝突，分開 dp 陣列有時會更穩健
#### 總結 (Takeaway)
- 帶障礙物的路徑問題 SOP：
    1. 檢查起點與終點是否即為障礙物
    2. 初始化 DP 陣列並處理第一行/列的累積（直到遇到第一個障礙物）
    3. 迴圈遍歷，遇到障礙物將 dp[j] 歸零
    4. 若非障礙物則套用轉移方程
