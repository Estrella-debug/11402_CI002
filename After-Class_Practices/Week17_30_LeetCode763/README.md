### 題目：[LeetCode: 763. Partition Labels](https://leetcode.com/problems/partition-labels/description/)

#### 解題思路
給定一個字串 s，將其分割成多個片段，使得每個字母最多只出現在一個片段中
- 目標：回傳每個片段的長度
- 核核心特性：為了滿足「同一個字母只能出現在一個片段」的要求，對於片段內的任意字母，我們必須將該片段的右邊界延伸至該字母最後一次出現的位置
- 核心策略：先透過預處理找出每個字母的最後出現位置，接著在遍歷過程中利用貪心法動態維護當前片段的「最遠邊界」
#### 關鍵邏輯：
1. Last Occurrence：
    - 建立一個大小為 26 的陣列 lastPos，記錄每個字母在字串中最後一次出現的索引
2. 動態邊界維護：
    - 在遍歷字串時，更新 end = max(end, lastPos[s[i]])。這代表只要區間內出現了某個字元，這個區間就必須延伸到該字元最後出現的地方
3. 切割時機：
    - 當遍歷指針 i 剛好追上 end 時，代表當前片段已經無法再擴充（因為再往後看，該區間內的所有字元都不會再出現了），此時即為最佳切割點

#### 模擬運算(s = "ababcbacadefegde"為例)：
1. 統計：a 最後在 8, b 最後在 5, c 最後在 7...
2. 遍歷：
    - i=0 (a)：end 更新為 lastPos['a'] = 8
    - i=1 (b)：end 更新為 max(8, 5) = 8
    - ... 當 i 達到 8 時：i == end 成立，切割。第一個區間長度為 8 - 0 + 1 = 9
3. 更新：下一個區間 start 設為 9
- Time Complexity：$O(N)$，只需掃描兩次字串
- Space Complexity：$O(1)$，僅需 26 個單位的陣列空間

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 貪心邊界思想：當問題要求「分割」或「覆蓋」且涉及範圍時，紀錄每個元素的最遠邊界通常是最佳策略
    - 雙指標的簡化：雖然 start 和 end 看起來像雙指標，但其實我們只需要一個 i 指針和一個 end 邊界變數就能完美解決，簡化了複雜度
    - ASCII 運算：使用 s[i] - 'a' 將字元映射到 0-25 的索引，是處理字串題目中最常見且高效的技巧
#### 總結 (Takeaway)
- 字串區間分割 SOP：
    1. 遍歷一次字串，利用陣列存儲每個字母最後出現的位置
    2. 再次遍歷字串，同時追蹤當前片段內所有字母的「最遠邊界」
    3. 當指針到達最遠邊界時，切割當前片段，並更新下一個片段的起始點
