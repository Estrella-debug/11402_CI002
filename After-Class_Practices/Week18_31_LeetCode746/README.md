### 題目：[ 746. Min Cost Climbing Stairs](https://leetcode.com/problems/min-cost-climbing-stairs/description/)

#### 解題思路
給定一個整數陣列 cost，其中 cost[i] 是從第 i 階樓梯向上爬所需的費用。每一步你可以選擇爬 1 階或 2 階。你可以從第 0 階或第 1 階開始。計算達到樓梯頂端（超出最後一階）所需的最低成本
- 目標：最小化總累計費用
- 核心特性：這同樣是一個「遞歸依賴」問題，但目標從「方法數」變成了「累計代價」
- 核心策略：狀態轉移方程為 $dp[i] = \min(dp[i-1] + \text{cost}[i-1], dp[i-2] + \text{cost}[i-2])$。我們同樣可以使用滾動變數將空間複雜度優化至 $O(1)$

#### 關鍵邏輯：
1. 狀態定義：
    - prev1: 到達前一階的最低成本
    - prev2: 到達前兩階的最低成本
2. 狀態轉移：
    - 對於每一階 i，到達這裡的成本是「從前一階跳上來」或「從前兩階跳上來」兩者中的最小值
    - 因此，current = min(prev1 + cost[i-1], prev2 + cost[i-2])
3. 邊界條件：
    - 我們可以從第 0 階或第 1 階開始，因此初始時到達這兩階的成本都視為 0

#### 模擬運算(cost = [10, 15, 20]為例)：
1. 初始：prev2 = 0, prev1 = 0
2. i=2：計算到達第 2 階的成本
    - current = min(0 + 15, 0 + 10) = 10
    - 更新：prev2 = 0, prev1 = 10
3. i=3 (目標頂層)：計算到達第 3 階的成本
    - current = min(10 + 20, 0 + 15) = 15
    - 更新：prev2 = 10, prev1 = 15
- 結果：15
- Time Complexity：$O(N)$，遍歷一次陣列
- Space Complexity：$O(1)$，使用滾動變數節省空間

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 索引對應：要注意 cost[i] 代表的是「從第 i 階出發」的代價，這與 dp 陣列紀錄「到達第 i 階」的代價不同
    - 動態規劃的廣度：這類問題證明了 DP 不僅能算「有多少種走法」，還能處理「最優路徑選擇」
    - 變數更新順序：在更新 prev2 與 prev1 時，必須先保存 prev1 的舊值，或者像程式碼中這樣正確安排賦值順序，避免覆蓋錯誤

#### 總結 (Takeaway)
- 路徑成本最小化 SOP：
    1. 定義到達前兩階的初始狀態 (費用為 0)
    2. 迴圈遍歷至 $n$ (包含頂層)，計算每個階層的最小進入成本
    3. 利用滾動變數記錄最近兩次的計算結果
    4. 最終回傳到達頂層的最小費用
