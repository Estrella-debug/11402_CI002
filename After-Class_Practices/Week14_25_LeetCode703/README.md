### 題目：[LeetCode: 703. - Kth Largest in a Stream](https://leetcode.com/problems/kth-largest-element-in-a-stream/description/)

#### 解題思路
在一個數據流（Stream）中，設計一個類別來實作 add(val) 方法，每次新增一個數字後，返回目前所有數字中第 $k$ 大的那個
- 目標：在動態新增數字的同時，保持 $O(\log k)$ 的查詢效能
- 核心特性：我們只需要知道「最大的 $k$ 個數」中有哪些，而其中「最小的那個」，就是我們要的「第 $k$ 大」
- 核心策略：維護一個大小固定為 $k$ 的 最小堆疊 (Min-Heap)

#### 關鍵邏輯：
1. 最小堆疊的維護：
    - 堆疊中始終保留目前數據流中最大的 $k$ 個元素
    - 因為是最小堆疊，堆疊頂端 (top()) 永遠是這 $k$ 個元素中「最小的那個」，也就是全域第 $k$ 大的數
2. add 操作流程：
    - 每次呼叫 add(val)，將新數字推入堆疊
    - 若堆疊大小超過 $k$，彈出堆疊頂端（最小值）
    - 此時堆疊內剩餘的 $k$ 個數字，就是數據流中最大的 $k$ 個，而頂端就是第 $k$ 大

#### 模擬運算(nums = [0, 1, 2, 4, 5, 7]為例)：
- 堆疊初始化：放入 4, 5, 8。堆疊：[4, 5, 8]，頂端為 4（第 3 大）
- add(2)：放入 2 後堆疊為 [2, 4, 5, 8]，超過大小 $k=3$，彈出 2。堆疊：[4, 5, 8]，頂端為 4
- add(3)：放入 3 後堆疊為 [3, 4, 5, 8]，超過大小 $k=3$，彈出 3。堆疊：[4, 5, 8]，頂端為 4
- Time Complexity：
    - 建構式：$O(N \log k)$，其中 $N$ 為初始陣列大小
    - add 方法：$O(\log k)$，因為堆疊大小固定為 $k$
- Space Complexity：$O(k)$，僅需維護大小為 $k$ 的堆疊

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - Heap 的選擇：解決「第 $k$ 大」問題，永遠要用「最小堆疊」；反之，若要求「第 $k$ 小」，則要用「最大堆疊」。這點千萬不能搞混
    - 動態數據處理：這是處理串流數據 (Stream Data) 的標準範本。當數據量極大無法全部排序時，堆疊是維持「局部有序」的最佳方案
    - C++ STL 語法：透過 std::greater<int> 輕鬆將預設的最大堆疊轉為最小堆疊，這是 C++ 開發者必備的技能
#### 總結 (Takeaway)
- K-th 問題 SOP：
    1. 建立大小為 $k$ 的優先佇列 (優先級與題目要求相反)
    2. 若堆疊未滿 $k$ 個，直接加入
    3. 若堆疊已滿 $k$ 個，比較新值與堆疊頂端，新值較大則替換掉頂端
    4. 堆疊頂端即為所求結果
