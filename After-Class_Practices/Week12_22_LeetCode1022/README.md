### 題目：[1022. Sum of Root To Leaf Binary Numbers](https://leetcode.com/problems/sum-of-root-to-leaf-binary-numbers/description/)

#### 解題思路
給定一棵二元樹，每個節點的值不是 $0$ 就是 $1$。從根節點到葉節點的每條路徑都代表一個二進位數字
- 目標：計算所有從根到葉路徑所代表的數字之總和
- 核心策略：使用 深度優先搜尋 (DFS)。當我們從父節點移動到子節點時，相當於將目前的二進位數值「左移一位」（即乘以 2），然後加上當前節點的值

#### 關鍵邏輯：
1. Binary Shift：
    - 在遞迴過程中傳遞一個變數 now
    - 進入子節點時，計算方式為：now * 2 + node->val。這符合二進位從高位往低位計算的邏輯
2. Base Case:
    - 當遇到 !(root->left) && !(root->right) 時，代表到達了路徑的終點（葉子節點）
    - 此時將累積的 now 直接加到全域變數 sum 中
3. 遞迴走訪：
    - 只要左子樹或右子樹存在，就繼續向下遞迴，並在參數中更新該分支目前的二進位數值
4. 狀態重置：
    - 在主函式 sumRootToLeaf 中將 sum 歸零，確保多次呼叫（例如在 main 測試不同範例）時結果不會累加出錯

#### 模擬運算(路徑 1 -> 0 -> 1 為例)：
- 根節點 (1)：now = 1
- 左子節點 (0)：now = 1 * 2 + 0 = 2 (二進位為 $10_2$)
- 葉節點 (1)：now = 2 * 2 + 1 = 5 (二進位為 $101_2$)
- 結算：將 5 加入總和
- Time Complexity：$O(N)$。每個節點恰好被走訪一次
- Space Complexity：$O(H)$，其中 $H$ 是樹的高度。這取決於遞迴時系統棧（System Stack）的深度

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 二進位即時計算：不需要先用 vector 儲存路徑最後再轉換，邊走訪邊計算 now = now * 2 + val 更加節省空間
    - 遞迴參數傳遞：透過參數傳遞 now 而非修改全域變數，可以自然地讓不同路徑擁有各自獨立的數值狀態，且無需手動「回溯」
    - 葉節點定義：在二元樹中，必須「左右皆為空」才算是一條路徑的終點，若只有單邊為空則仍需繼續往下
#### 總結 (Takeaway)
- 路徑數值累積問題 SOP：
    1. 選擇前序遍歷（由根向葉累積）
    2. 定義進位邏輯（二進位為 * 2，十進位為 * 10）
    3. 設定葉節點為終止條件並累加結果
    4. 注意處理根節點為空的邊界情況
