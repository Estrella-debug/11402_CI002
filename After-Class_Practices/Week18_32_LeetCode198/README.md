### 題目：[198. House Robber](https://leetcode.com/problems/house-robber/description/)

#### 解題思路
你是一位專業強盜，計畫搶劫一排房屋。每間房子都有一定的現金，唯一的限制是「不能搶劫兩間相鄰的房屋」，因為這會觸發警報
- 目標：計算在不觸發警報的情況下，所能搶劫到的最高金額
- 核心特性：當你考慮搶劫第 $i$ 間房子時，你面臨兩個選擇：
    1. 搶劫：那麼你必須加上第 $i-2$ 間房子的最高收益（因為不能搶第 $i-1$ 間）
    2. 不搶：那麼你目前的最高收益即為第 $i-1$ 間的最高收益
- 核心策略：狀態轉移方程為 dp[i] = max(dp[i-1], dp[i-2] + nums[i])

#### 關鍵邏輯：
1. 狀態定義：
    - prev1: 到達上一間房子時的最高收益
    - prev2: 到達上上間房子時的最高收益
2. 滾動更新：
    - 每一間房子都執行一次決策：current = max(prev1, prev2 + num)
    - 更新後將 prev2 設為舊的 prev1，prev1 設為新的 current
3. 切割時機：
    - 若沒有房子，金額為 0；若只有一間，則直接搶劫該間

#### 模擬運算(nums = [2, 7, 9, 3, 1]為例)：
1. 初始 prev2 = 0, prev1 = 0
2. 搶 2：current = max(0, 0+2) = 2；prev2=0, prev1=2
3. 搶 7：current = max(2, 0+7) = 7；prev2=2, prev1=7
4. 搶 9：current = max(7, 2+9) = 11；prev2=7, prev1=11
5. 搶 3：current = max(11, 7+3) = 11；prev2=11, prev1=11
6. 搶 1：current = max(11, 11+1) = 12；prev2=11, prev1=12
- 結果：最高收益 12
- Time Complexity：$O(N)$，只需遍歷一次房屋陣列
- Space Complexity：$O(1)$，僅使用兩個整數記錄狀態

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 貪心 vs 動態規劃：初學者常誤以為「選最大的」就是最佳解，但本題證明了「局部最佳選擇（當下看見大錢就搶）」並不保證「全域最大收益」，必須透過 DP 來考慮未來的可能性
    - 空間優化：此題完美展現了如何透過滾動變數將 $O(N)$ 的空間複雜度壓到 $O(1)$
    - 決策的不可逆性：在 DP 中，每個決策都建立在之前的狀態之上，這培養了將複雜問題拆解為「當前決策對未來影響」的能力

#### 總結 (Takeaway)
- 搶劫最佳化 SOP：
    1. 針對每間房子進行兩難決策：搶 (搭配上上間收益) 或 不搶 (延續上一間收益)
    2. 用兩個變數紀錄「上一間」與「上上間」的最高收益
    3. 遍歷每間房子並動態更新這兩個變數
    4. 最後的變數即為全域最大收益
