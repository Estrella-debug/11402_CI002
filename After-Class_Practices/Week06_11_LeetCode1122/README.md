### 題目：[1122. Relative Sort Array](https://leetcode.com/problems/relative-sort-array/description/)

#### 解題思路
給定兩個陣列 arr1 和 arr2，其中 arr2 中的元素各不相同，且 arr2 中的所有元素都出現在 arr1 中
- 排序規則：
    1. 讓 arr1 中元素的相對順序與 arr2 中的相對順序相同
    2. 未在 arr2 中出現的元素，必須按照 升序（從小到大） 放在 arr1 的末尾
- 策略：先統計所有數字出現的次數，再依照「指定順序」與「剩餘順序」依序填入結果

#### 關鍵邏輯：
1. 頻率統計 (Counting)：使用 std::map<int, int> counts
    - 優點：std::map 在 C++ 中會自動根據 Key (數值) 由小到大排序。這對我們處理「不在 arr2 中的剩餘元素」非常有幫助，因為遍歷 map 時它們自然就是由小到大的
2. 優先填入：
    - 遍歷 arr2 中的每個數字 x
    - 查找 counts[x]，將其對應數量的 x 全部放入結果陣列，並將次數歸零
3. 收尾填入：
    - 遍歷 map 中剩餘次數大於 0 的元素
    - 由於 map 的特性，這些元素會自動按升序被放入結果陣列末尾
#### 模擬運算：
- 輸入：arr1 = [2, 3, 1, 3, 2, 4, 6, 7, 9, 2, 19], arr2 = [2, 1, 4, 3, 9, 6]
- Step 1 (統計)：counts = {1:1, 2:3, 3:2, 4:1, 6:1, 7:1, 9:1, 19:1}
- Step 2 (依 arr2 填入)：
    - 按 2, 1, 4, 3, 9, 6 順序填入 $\rightarrow$ [2, 2, 2, 1, 4, 3, 3, 9, 6]
- Step 3 (依 map 剩餘填入)：
    - 剩下 7 和 19，按順序補上 $\rightarrow$ [..., 7, 19]
- Time Complexity：$O(N \log M + K)$。$N$ 是 arr1 長度，$M$ 是 arr1 中不重複數字個數（Map 的操作成本），$K$ 是 arr2 長度
- Space Complexity：$O(M)$。需要空間儲存頻率表

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - Map 的排序特性：這題利用了 std::map 的 Key 會自動排序的特質，省去了最後對剩餘元素手動呼叫 sort() 的步驟
    - 計數排序 (Counting Sort) 的變體：這題本質上是計數排序的思想，只是輸出的順序由另一個陣列來引導
    - 記憶體管理：雖然 map 方便，但若數字範圍很小（例如 0 到 1000），使用簡單的 vector<int> count(1001) 速度會更快，因為少了紅黑樹的維護成本
#### 總結 (Takeaway)
- 相對排序問題 SOP：
    1. 建立頻率分布表
    2. 依照參考陣列的順序，把對應數量的元素領出來
    3. 將剩下的元素按照特定規則（通常是升序）填完
- 這種「先統計、後重組」的模式是處理非標準排序規則最有效率的手段
