### 題目：[622. Design Circular Queue](https://leetcode.com/problems/design-circular-queue/description/)

#### 解題思路
設計一個 環形佇列。環形佇列相比普通佇列，最大的優點是它可以回收利用已經出隊的空間
- 物理結構：底層使用一個固定長度的陣列（或 vector）
- 邏輯結構：當指標移動到陣列末尾時，下一個位置會自動回到陣列開頭，形成一個環
- 核心策略：使用兩個指標 head 與 tail 來追蹤隊首與隊尾。關鍵在於利用 取餘數運算 (%) 來處理指標溢出並實現「繞回」的效果

#### 關鍵邏輯：
1. 取餘數運算 (%)：
    - 這是環形結構的靈魂。例如當指標 tail 到達索引 k-1 時，下一個位置是 (k-1 + 1) % k = 0，成功回到起點
2. 指標管理：
    - head：指向隊首元素
    - tail：指向最新加入的隊尾元素
    - size：獨立記錄目前佇列中的元素個數，這能讓 isEmpty 與 isFull 的判斷變得非常簡單
3. 邊界狀態重置：
    - 當佇列變為空時，將 head 與 tail 重置為 -1，這有助於簡化下一次入隊時的初始化判斷
4. 空間效率：
    - 所有操作（enQueue, deQueue, Front, Rear）的時間複雜度均為 $O(1)$，且不需要動態重新分配記憶體
#### 模擬運算(容量 3 為例)：
- 初始：capacity = 3, head = -1, tail = -1, size = 0
- 入隊 1, 2, 3：
    - data = [1, 2, 3], head = 0, tail = 2, size = 3 (已滿)
- DeQueue:
    - 移除 data[0]，head = (0 + 1) % 3 = 1
    - 現在 Front 是 data[1] = 2
- 入隊 4：
    - 空間已釋出，tail = (2 + 1) % 3 = 0
    - data[0] = 4，size = 3
    - 此時陣列雖然實體順序是 [4, 2, 3]，但邏輯順序是 2 -> 3 -> 4

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 取餘數的威力：深刻理解了 (i + 1) % capacity 如何將線性空間變成邏輯上的環
    - 指標初始化：將 head/tail 初始化為 -1 或是 0 需要搭配不同的邏輯，本程式碼使用 -1 並配合 size 計數，判斷邏輯非常直觀
    - 靜態空間管理：在嵌入式系統或對效能要求極高的場景中，固定大小的環形佇列比動態生長的 std::queue 更加穩定
#### 總結 (Takeaway)
- 環形資料結構設計 SOP：
    1. 預留固定大小的儲存空間
    2. 定義 head 指向開頭，tail 指向結尾
    3. 所有的指標移動必須配合 % capacity
    4. 額外使用 size 變數來簡化「空」與「滿」的邏輯判定
