### 題目：[LeetCode: 347. - Top K Frequent Elements](https://leetcode.com/problems/top-k-frequent-elements/description/)

#### 解題思路
給定一個整數陣列 nums 和一個整數 k，找出陣列中出現頻率最高的 k 個元素
- 目標：回傳這 k 個元素
- 核心特性：元素的出現次數（頻率）範圍被限定在 [0, nums.size()] 之間。這使得我們可以將「頻率」作為索引，直接將數字放入對應的「桶 (Bucket)」中
- 核心策略：使用 桶排序 (Bucket Sort)。這避免了對整個頻率表進行排序所需的 $O(N \log N)$ 時間，將複雜度優化至 $O(N)$

#### 關鍵邏輯：
1. 頻率統計：
    - 使用 unordered_map<int, int> 統計每個數字出現的次數
2. 桶結構分配：
    - 建立一個大小為 nums.size() + 1 的桶陣列 vector<vector<int>> buckets
    - 索引 i 代表出現頻率，buckets[i] 存放所有出現頻率剛好為 i 的數字列表
3. 逆向收集:
    - 從頻率最高的桶（陣列最後一個位置）開始向前遍歷，依序將桶內的數字加入結果中，直到收集滿 k 個數字為止

#### 模擬運算(nums = [1, 1, 1, 2, 2, 3], k = 2為例)：
- 統計頻率：{1:3, 2:2, 3:1}
- 分配至桶：
    - buckets[1] = {3}
    - buckets[2] = {2}
    - buckets[3] = {1}
- 收集結果：
    - 從 i=6 遞減檢查，發現 buckets[3] 有 {1}，加入結果
    - 繼續檢查 i=2，發現 buckets[2] 有 {2}，加入結果
    - 結果達到 k=2，返回 [1, 2]
- Time Complexity：$O(N)$。統計頻率與放入桶的動作皆為線性時間
- Space Complexity：$O(N)$，用於儲存雜湊表與桶陣列

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 桶排序的適用時機：只有當我們能明確預測資料的「數值範圍」或「頻率範圍」時，桶排序才會比基於比較的排序演算法（如堆疊、快排）更快
    - C++17 結構化綁定：程式碼中 for (auto const& [num, freq] : counts) 是 C++17 的語法，能讓遍歷 Map 的代碼更加簡潔易讀
    - 邊界條件：桶的數量必須設為 size + 1，因為最高頻率可能是 size 次，這點在設定陣列長度時容易遺漏
#### 總結 (Takeaway)
- 頻率 Top-K 問題 SOP：
    1. 使用 Hash Map 統計所有數字的頻率
    2. 建立一個「以頻率為索引」的陣列（桶）
    3. 將數字分配至對應頻率的桶中
    4. 從最高頻率開始逆向掃描桶陣列
