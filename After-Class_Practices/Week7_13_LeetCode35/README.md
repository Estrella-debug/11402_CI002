### 題目：[35. Search Insert Position](https://leetcode.com/problems/search-insert-position/description/)

#### 解題思路
給定一個已排序且無重複元素的整數陣列 nums，以及一個目標值 target
- 目標：
    1. 如果找到目標值，回傳其索引
    2. 如果找不到，回傳它被按順序插入後該有的索引位置
- 限制：必須使用時間複雜度為 $O(\log n)$ 的演算法

#### 關鍵邏輯：
1. 標準二元搜尋框架：設定 low = 0 與 high = nums.size() - 1
2. 安全中點計算：使用 mid = low + (high - low) / 2 避免大數相加導致的整數溢位
3. 插入位置的判定：
    - 當 nums[mid] == target，代表找到目標，直接回傳
    - 當迴圈結束（即 low > high）仍未找到目標時，low 的值剛好就是該數字應該插入的位置
    - 原因：在最後一次比較中，如果 target 較大，low 會移動到 mid + 1；如果 target 較小，low 會停在 mid 的位置，而這正是維持排序所需的正確索引
#### 模擬運算(nums = [1, 3, 5, 6], target = 2 為例)：
- 第一次迭代：low = 0, high = 3。mid = 1 (值為 3)
    - 3 > 2，往左找。high = 0
- 第二次迭代：low = 0, high = 0。mid = 0 (值為 1)
    - 1 < 2，往右找。low = 1
- 結束：low (1) > high (0)，跳出迴圈
- 結果：回傳 low = 1
- Time Complexity：$O(\log n)$。標準二元搜尋的效率
- Space Complexity：$O(1)$。僅使用常數級的索引變數

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 邊界返回值：這題最巧妙的地方在於理解為何迴圈結束後要回傳 low。這是二元搜尋在「尋找插入點」或「尋找第一個大於等於目標的數」時的標準特性
    - 無重複前提：因為題目保證無重複，所以我們不需要處理「多個相同數字該插在哪裡」的問題
    - 高效處理有序陣列：只要看到「排序過」與「對數時間複雜度」，二元搜尋永遠是第一個要跳進腦袋的工具
#### 總結 (Takeaway)
- 搜尋插入位置問題 SOP：
    1. 確認陣列已排序
    2. 執行標準二元搜尋
    3. 若匹配成功，回傳 mid
    4. 若搜尋失敗，回傳最後的 low 指標
