### 題目：[222. Count Complete Tree Nodes](https://leetcode.com/problems/count-complete-tree-nodes/description/)

#### 解題思路
給定一個二元樹（題目背景通常為完全二元樹），計算該樹中所有節點的數量
- 核心策略：雖然題目給出的是完全二元樹，但最直觀且通用的解法是利用 遞迴
- 拆解邏輯：整棵樹的節點數 = 根節點本身 (1) + 左子樹的節點總數 + 右子樹的節點總數

#### 關鍵邏輯：
1. Base Case：
    - 當遞迴遇到 nullptr 時，代表已經到達葉節點的下方，此時應回傳 0
2. Recurrence Relation
3. 走訪順序：
    - 本程式碼採用的是 後序遍歷 (Post-order) 的思維，即先獲取左右子樹的結果，最後加上根節點自己

#### 模擬運算(樹 [1, 2, 3] 為例)：
1. 呼叫 countNodes(1)：
    - 需要 1 + countNodes(2) + countNodes(3)
2. 計算 countNodes(2)：
    - 節點 2 是葉節點，其左右皆為 nullptr，回傳 $1 + 0 + 0 = 1$
3. 計算 countNodes(3)：
    - 節點 3 是葉節點，回傳 $1 + 0 + 0 = 1$
4. 回到根節點：
    - 總數為 $1 + 1 + 1 = 3$
- Time Complexity：$O(N)$。每個節點都會被訪問一次
- Space Complexity：$O(H)$，其中 $H$ 是樹的高度，代表遞迴呼叫時系統棧（Stack）的深度

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 遞迴思維的建立：掌握了將大問題（計算整棵樹）轉化為小問題（計算子樹）的拆解能力
    - 指標安全檢查：在進行 root->left 操作前，必須先判斷 root 是否為 nullptr，這是避免程式崩潰（Runtime Error）的基本功
    - 完全二元樹的優化潛力：雖然 $O(N)$ 解法可以通過，但在完全二元樹中，其實可以利用其結構特性（如二分搜尋）達成 $O(\log^2 N)$ 的更高效解法
#### 總結 (Takeaway)
- 二元樹計數問題 SOP：
    1. 處理空節點（回傳 0）
    2. 遞迴計算左邊
    3. 遞迴計算右邊
    4. 回傳「左 + 右 + 1」
