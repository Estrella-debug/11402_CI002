### 題目：[326. Power of Three](https://leetcode.com/problems/power-of-three/description/)

#### 解題思路
這題的目標是判斷一個整數 $n$ 是否為 3 的冪次方（Power of Three）。也就是說，是否存在一個整數 $x$，使得 $n = 3^x$
- 範例：$27$ 是 3 的冪次方 ($3^3$)；$21$ 雖然能被 3 整除，但不是 3 的冪次方
#### 關鍵邏輯：
1. 排除負數與零：根據冪次方的定義，$3^x$ 在整數範圍內永遠是大於 0 的。因此如果 $n \le 0$，直接回傳 false
2. 連續除法 (Successive Division)：
    - 只要 $n$ 能夠被 3 整除（即 n % 3 == 0），我們就不斷地將 $n$ 除以 3
    - 這個過程會一直持續到 $n$ 無法再被 3 整除為止。
3. 最終判定：
    - 如果 $n$ 原本就是 3 的冪次方，經過連續除法後，最後剩下的值一定會是 1
    - 如果最後剩下的值不是 1，代表它含有其他的質因數，則不是 3 的冪次方

#### 模擬運算($n = 27$ 為例)：
- 第一步：$27 > 0$，進入迴圈
- 第二步：$27 \% 3 == 0$，執行 $27 / 3 = 9$
- 第三步：$9 \% 3 == 0$，執行 $9 / 3 = 3$
- 第四步：$3 \% 3 == 0$，執行 $3 / 3 = 1$
- 第五步：$1 \% 3 \ne 0$，跳出迴圈
- S判定：$n$ 目前等於 1，回傳 true
- Time Complexity：$O(\log_3 n)$。每次循環數字都會縮小為原來的三分之一
- Space Complexity：$O(1)$。僅使用一個判斷變數

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - While 迴圈的運用：這題展示了如何利用 while 搭配 % (取餘數) 來檢查數字的組成因數。
    - 邊界檢查 (Edge Case)：很多新手會忘記處理 $n=0$ 或負數的情況，導致程式進入無窮迴圈或結果錯誤。
    - 進階思考：題目通常會問「能不能不使用迴圈？」。在 C++ 中，我們可以利用數學對數（Logarithm）或是利用 int 範圍內最大的 3 的冪次方（即 $3^{19} = 1,162,261,467$）來直接進行整除判定。

#### 總結 (Takeaway)
- 判定「某數之冪」問題 SOP：
    1. 處理小於等於 0 的例外狀況
    2. 利用 while 迴圈持續進行「取餘數檢查」與「除法縮小」
    3. 檢查最終留下的數字是否為 1
- 這種「不斷剝離因數」的做法是處理質因數分解與數字特性問題的基礎
