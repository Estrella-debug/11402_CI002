### 題目：[148. Sort List](https://leetcode.com/problems/sort-list/description/)

#### 解題思路
在 $O(n \log n)$ 時間複雜度和常數級額外空間（如果不考慮遞迴佔用的棧空間）下，對鏈結串列進行排序
- 策略：利用「分治法 (Divide and Conquer)」。先將串列從中間切開，分別排序後，再將兩個已排序的子串列合併

#### 關鍵邏輯：
1. 尋找中點：
    - 使用 快慢指標 (Fast and Slow Pointers)。快指標走兩步，慢指標走一步。當快指標到終點時，慢指標剛好在中點
    - 切斷連接：利用一個 pre 指標紀錄慢指標的前一個位置，將 pre->next 設為 nullptr，正式將串列拆成兩半
2. 遞迴拆解：
    - 不斷呼叫 sortList 直到串列只剩下一個節點或為空（這是遞迴終止條件）
3. 合併串列：
    - 建立一個 dummy 節點作為輔助
    - 比較兩個子串列的節點值，將較小的節點依序接在 tail->next 後面
    - 最後將剩餘還沒接完的節點直接掛載在末尾
#### 模擬運算(4 -> 2 -> 1 -> 3 為例)：
- 拆分階段：
    - 第一層：拆成 (4, 2) 與 (1, 3)
    - 第二層：拆成 (4), (2) 與 (1), (3)
- 合併階段：
    - 合併 (4), (2) $\rightarrow$ 得到 (2 -> 4)
    - 合併 (1), (3) $\rightarrow$ 得到 (1 -> 3)
    - 最終合併 (2, 4) 與 (1, 3) $\rightarrow$ 得到 (1 -> 2 -> 3 -> 4)
- 結果：輸出 [2]
- Time Complexity：$O(n \log n)$。分治法的高度為 $\log n$，每層合併需要 $O(n)$
- Space Complexity：$O(\log n)$。遞迴呼叫所使用的棧空間（Stack space）

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - 快慢指標的穩定性：這是處理 Linked List 找中點或環（Cycle）的標準工具
    - Dummy Node 的妙用：在合併串列時，使用一個虛擬頭節點 dummy 可以避免處理「新串列開頭是誰」的繁瑣判斷
    - 遞迴終止條件：if (!head || !head->next) 是這類問題最容易漏寫但也最重要的邊界保護
#### 總結 (Takeaway)
- Linked List 排序問題 SOP：
    1. 利用快慢指標找到中點
    2. 斷開中點，將問題拆解為兩個子問題
    3. 實作 merge 函式將兩條排序好的串列接起來
- 合併排序是 Linked List 最理想的排序方式，因為它不需要隨機存取（Random Access），非常符合鏈結串列的特性
