### 題目：[8. String to Integer (atoi) ](https://leetcode.com/problems/string-to-integer-atoi/)

#### 解題思路
這題的目標是把一個 字串 (string) 變成一個 整數 (integer)。我們需要遵守一些規則，例如處理空格、正負號以及超大數字。

#### 關鍵邏輯：
1. 跳過空格 (Clear Spaces)： 忽略字串開頭的所有空格。
2. 檢查正負號 (Check the Sign)： 查看是否有 + 或 -。
3. 跳過 0 (Skip Zeros)： 忽略數字開頭的 0。
4. 讀取數字 (Read Digits)： 持續讀取字元，直到遇到非數字或字串結束。
5. 溢位處理 (Check Range)： 如果數字太大，超過了 int 的範圍，必須回傳最大值或最小值。

#### 模擬運算 (以 -42 為例)：
- 第一步：跳過前兩個空格，指標 i 移向 '-'。
- 第二步：讀取 '-'，設定 sign = -1，i 移向 '4'。
- 第三步：讀取 '4'，計算 sum = 0 * 10 + 4 = 4。
- 第四步：讀取 '2'，計算 sum = 4 * 10 + 2 = 42。
- 最後計算：結果為 42 * -1 = -42，在整數範圍內，回傳 -42。
- 時間複雜度：$O(n)$，其中 $n$ 是字串長度。
- 空間複雜度：$O(1)$，只使用固定數量的變數。

#### 語言差異與踩坑紀錄
* **學到了什麼：**
    - ASCII 轉換技巧：字元變數字不需要複雜函數，直接用 s[i] - '0' 即可利用 ASCII 碼的價差取得數值。
    - 資料型態安全：在 C++ 中，為了防止在計算 sum 時就直接溢位，通常會先用 long 或 double 儲存中間值，最後再跟 INT_MAX 與 INT_MIN 做比較。
    - 提早中斷：如果字串長度極長（如 100 個數字），long 也會溢位。因此在迴圈中可以加入長度判定，一旦太長就直接判定為溢位。

#### 總結 (Takeaway)
- 處理數字翻轉的標準 SOP：
    1. Trim：處理開頭無效字元（空格、正負號）。
    2. Convert：利用 sum = sum * 10 + digit 累加數值。
    3. Clamp：將結果限制在 [INT_MIN, INT_MAX] 區間內。
- 這種「手動解析」字串的方法是面試的基本功，能體現你對資料邊界（Edge Cases）的敏感度。
